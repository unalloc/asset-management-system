# F-2.1.1 - Multiple Asset Types Support

## Requirement ID
F-2.1.1

## Function Name
Multiple Asset Types Support

## Function Description
The system shall support the registration and management of multiple asset types including but not limited to IT equipment, office furniture, vehicles, and production machinery.

## Business Rules

1. Asset types use a hierarchical classification structure
2. Each asset type has a specific attribute template
3. Asset types can be dynamically extended
4. Type changes require audit tracking

## Preconditions

- User has "Asset Registration" permission
- Asset type classification system is configured

## Postconditions

- New asset successfully registered in the system
- Asset type information is completely recorded

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Asset Type Code | Yes | Unique identifier for the type |
| Asset Type Name | Yes | Display name |
| Description | No | Type description |
| Parent Type ID | No | For hierarchy structure |
| Attribute Template ID | No | Associated template |

## Output Data

1. Asset type registration confirmation
2. Assigned unique type ID
3. Creation timestamp

## Validation Rules

1. Type code must be unique
2. Type name cannot be empty
3. If parent type is specified, it must exist and be valid
4. Code format: alphanumeric, maximum 20 characters

## Error Handling

| Error | Message |
|-------|---------|
| Duplicate type code | "Type code already exists" |
| Invalid parent type | "Parent type does not exist" |
| Required field missing | "Required field [field name] is missing" |

## Priority
High

## Related User Stories
US-001, US-002
