## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Tooling lifecycle management (Clause 6.4.4) |
| **ISO 55001 Physical Asset Management** | Yes | Usage-based maintenance for tooling |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Tooling-specific maintenance strategies |

**ISO 55000 Alignment Explanation:**

Tooling and fixtures management supports ISO 55000's asset lifecycle management by providing:
- Usage tracking for maintenance optimization
- Life-cycle monitoring for replacement planning
- Production-linked maintenance triggers
- Tool wear analysis for quality control

**Alignment Gaps:**
- No explicit support for ISO 22995 (tool management) standard
- Missing integration with tool CAD/design systems

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Tooling defined as asset type | `AssetType = 'TOOLING' OR Exists(ToolingType)` | VAL-001 | "Tooling asset type not defined" |
| PRE-002 | User has "Tooling Management" permission | `PermissionCheck(user, 'TOOLING_MGMT') == TRUE` | AUTH-001 | "User does not have Tooling Management permission" |
| PRE-003 | Tooling number is unique | `NOT Exists(Tooling WHERE ToolingNumber = Number)` | VAL-002 | "Tooling number already exists" |
| PRE-004 | Design life is positive | `DesignLife > 0` | VAL-003 | "Design life must be greater than 0" |

**Pre-condition Validation Matrix:**

```
IF ToolingTypeDefined()
   AND UserHasPermission('TOOLING_MGMT')
   AND ToolingNumberIsUnique(Number)
   AND DesignLife > 0
THEN proceed to tooling creation
ELSE throw appropriate error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Tooling Creation
  │
  ▼
Validate Tooling Type ──► [Not Defined] ──► VAL-001 Error
  │
  ▼ [Defined]
Validate Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Tooling Number ──► [Duplicate] ──► VAL-002 Error
  │
  ▼ [Unique]
Validate Design Life ──► [Not Positive] ──► VAL-003 Error
  │
  ▼ [Valid]
Create Tooling Asset
  │
  ▼
Initialize Usage Counter
  │
  ▼
Create Maintenance Plan
  │
  ▼
Log Tooling Creation
  │
  ▼
Return Success
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Number Uniqueness | ToolingNumber unique | Reject if duplicate | 1 |
| BR-002 | Design Life Validation | DesignLife > 0 | Reject if invalid | 1 |
| BR-003 | Usage Tracking | Track usage counts | Update counter on use | 1 |
| BR-004 | Maintenance Threshold | Trigger at threshold | Generate maintenance work order | 1 |
| BR-005 | Life Exhaustion Warning | Usage >= DesignLife * 0.9 | Warn user | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Design life exceeded | Warn, allow continued use | WRN-001 |
| EC-002 | Reset usage counter | Track resets with reason | Create history record |
| EC-003 | Transfer to new product | Update applicable product link | Update product reference |
| EC-004 | Tool replacement | Create replacement record, link to old | Preserve history |

### 3.5 State Transitions

| Current State | Event | Next State | Side Effects |
|---------------|-------|------------|---------------|
| NEW | First use | ACTIVE | Counter activated |
| ACTIVE | Maintenance | UNDER_MAINTENANCE | Work order created |
| ACTIVE | Design life reached | LIFE_EXHAUSTED | Warning triggered |
| UNDER_MAINTENANCE | Complete | ACTIVE | Counter continues |
| LIFE_EXHAUSTED | Rehabilitation | ACTIVE | New design life set |

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|

| BRV-003 | Threshold valid | Threshold <= DesignLife | Reject | VAL-005 |

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |

| VAL-003 | Error | Validation | Invalid design life | Design life must be greater than 0 | Enter positive value |
| VAL-004 | Error | Validation | Invalid type | Invalid tooling type | Select MOLD, FIXTURE, JIG, CUTTING, or GAUGE |
