# F-2.1.2 - Custom Asset Categories

## Requirement ID
F-2.1.2

## Function Name
Custom Asset Categories

## Function Description
Allow users to create and manage custom asset categories and subcategories based on enterprise-specific needs.

## Business Rules

1. Categories support unlimited hierarchical levels
2. Subcategories inherit attributes from parent category
3. Categories can be enabled/disabled
4. Category changes require version control

## Preconditions

- User has "Category Management" permission
- System initialization is complete

## Postconditions

- New category added to classification system
- Classification relationship correctly established

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Category Name | Yes | Display name |
| Category Code | Yes | Unique identifier |
| Parent Category ID | No | Parent category reference |
| Description | No | Category description |
| Sort Order | No | Display ordering |
| Enabled Status | No | Active/inactive state |

## Output Data

1. Category creation success response
2. Assigned category ID
3. Complete category path

## Validation Rules

1. Category code must be unique among siblings
2. Category name cannot be empty
3. Cannot create circular dependencies
4. Code format: alphanumeric with underscore, maximum 50 characters

## Error Handling

| Error | Message |
|-------|---------|
| Circular dependency | "Circular dependency detected in category relationship" |
| Duplicate code | "Category code already exists at this level" |
| Invalid parent | "Parent category does not exist or is disabled" |

## Priority
Medium
