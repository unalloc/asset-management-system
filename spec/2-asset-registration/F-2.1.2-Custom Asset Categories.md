## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Category hierarchy supports asset planning and classification (Clause 6.1.2) |
| **ISO 55001 Physical Asset Management** | Yes | Asset categorization aligns with Clause 6.4.3 asset register requirements |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Category structure follows documentation guidelines |

**ISO 55000 Alignment Explanation:**

Custom asset categories directly support ISO 55000's asset management planning by providing:
- Structured asset grouping for lifecycle planning (Clause 6.1.2)
- Hierarchical organization supporting risk-based asset grouping
- Foundation for category-specific maintenance strategies
- Enables financial reporting by asset class (IFRS/GAAP alignment)

**Alignment Gaps:**
- No explicit sustainability category classification
- Missing ESG/sustainability dimension in category structure

### 2.2 US/EU Regulatory Compliance

| Regulation | Applies | Compliance Status | Evidence/Mapping |
|------------|---------|-------------------|------------------|
| **GDPR** (EU) | Yes | Compliant | Category is non-personal master data |
| **SOX** (US) | Yes | Compliant | Category supports financial segment reporting |
| **OSHA** (US) | Partial | Gap | Category should link to OSHA safety classifications |
| **EPA** (US) | Partial | Gap | No WEEE/RCRA category mapping |
| **WEEE Directive** (EU) | Partial | Gap | Missing electronic waste categories |
| **IFRS 16** (EU/US) | Yes | Compliant | Category supports right-of-use asset classification |
| **US GAAP** (US) | Yes | Compliant | ASC 360 class mapping supported |
| **GAMP 5** (Pharma) | No | N/A | Not pharmaceutical-specific |

**Regulatory Compliance Details:**

- **SOX**: Categories map to financial reporting segments for fixed asset disclosure
- **GDPR**: No PII in category definitions; audit logs maintained separately
- **US GAAP/IFRS**: Category hierarchy supports PP&E depreciation class mapping

### 2.3 Gap Analysis

| Gap Category | Gap Description | Severity | Remediation Effort |
|--------------|-----------------|----------|-------------------|
| Environmental | No WEEE/RoHS/REACH category mapping | Medium | Add environmental category type |
| Safety | No OSHA equipment category linkage | Medium | Add safety classification |
| ESG | No EU Taxonomy category alignment | Low | Add sustainability dimension |

**Critical Gaps Requiring Immediate Attention:**
- None blocking initial deployment

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | User has "Category Management" permission | `PermissionCheck(user, 'CATEGORY_MANAGEMENT') == TRUE` | AUTH-001 | "User does not have Category Management permission" |
| PRE-002 | System initialization is complete | `SystemConfig.IsInitialized == TRUE` | CFG-001 | "System initialization not complete" |
| PRE-003 | Company context established | `Session.CompanyCode IS NOT NULL` | CTX-001 | "Company context required" |
| PRE-004 | If parent specified, parent exists | `ParentCategoryID == NULL OR EXISTS(Category WHERE CategoryID = ParentCategoryID)` | VAL-001 | "Parent category does not exist" |

**Pre-condition Validation Matrix:**

```
IF UserHasPermission('CATEGORY_MANAGEMENT')
   AND SystemConfig.IsInitialized == TRUE
   AND Session.CompanyCode IS NOT NULL
   AND (ParentCategoryID IS NULL OR ParentExists(ParentCategoryID))
THEN proceed to category creation
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Category added to classification system | CategoryID generated, audit log created | DB verification |
| POST-002 | Classification relationship established | Parent-child link created | Query parent relationship |
| POST-003 | Attribute inheritance configured | Child inherits parent attributes | Attribute verification |

**Post-condition Verification:**
- New Category record exists with unique ID
- AuditLog entry created with CREATE action
- Parent-child relationship established
- Attribute inheritance chain configured

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Category Create Request
  │
  ▼
Validate User Permissions ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Category Code ──► [Invalid] ──► VAL-002 Error
  │
  ▼ [Valid]
Check Duplicate at Same Level ──► [Exists] ──► VAL-003 Error
  │
  ▼ [Unique]
Validate Parent Category ──► [Invalid] ──► VAL-001 Error
  │
  ▼ [Valid]
Check Circular Dependency ──► [Would Cycle] ──► VAL-004 Error
  │
  ▼ [No Cycle]
Validate Category Name ──► [Empty] ──► VAL-005 Error
  │
  ▼ [Valid]
Create Category Record
  │
  ▼
Configure Attribute Inheritance
  │
  ▼
Publish CategoryCreated Event
  │
  ▼
Return Success Response
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Code Uniqueness per Parent | CategoryCode must be unique among siblings | Reject if duplicate at same level | 1 |
| BR-002 | Parent Validation | ParentCategoryID must exist | Reject if invalid | 1 |
| BR-003 | No Circular Dependency | Parent chain cannot loop | Reject if circular | 1 |
| BR-004 | Attribute Inheritance | Child inherits parent attributes | Auto-merge + override capability | 2 |
| BR-005 | Version Control | Any category field change | Create new version | 1 |
| BR-006 | Enabled/Disabled Status | Disabled categories unusable | Reject asset creation with disabled category | 1 |
| BR-007 | Maximum Hierarchy Depth | Maximum 10 levels | Reject if exceeded | 1 |

**Calculation Logic (if applicable):**
N/A - No calculations required

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Move category to new parent | Update hierarchy, preserve assets | Re-validate depth, update children paths |
| EC-002 | Parent is disabled | Allow but warn | WRN-001: Parent is inactive |
| EC-003 | Delete category with children | Block or cascade delete option | Require children handling |
| EC-004 | Maximum depth exceeded | Reject with error | VAL-006: Max depth is 10 levels |
| EC-005 | Code with special characters | Normalize or reject | VAL-002: Invalid characters |
| EC-006 | Rename category with assets | Allow, assets reference unchanged | No asset update required |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| ParentExists=TRUE | ParentActive=TRUE | DepthValid=TRUE | Proceed |
| ParentExists=TRUE | ParentActive=FALSE | DepthValid=TRUE | Warning + proceed |
| ParentExists=FALSE | - | - | VAL-001: Invalid parent |
| DepthValid=FALSE | - | - | VAL-006: Max depth exceeded |
| CreatesCircular=TRUE | - | - | VAL-004: Circular dependency |

### 3.5 State Transitions

**State Diagram:**

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   DRAFT     │──────►│   ACTIVE    │──────►│   DEPRECATED│
```

## 6. Enhanced Validation Rules

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |
|------------|-----------|--------|-------|----------|-----------------|------------|---------------|
| CategoryCode | VARCHAR | [A-Z0-9_]{1,50} | Uppercase alphanumeric + underscore | YES | Pattern: `^[A-Z0-9_]{1,50}$` | VAL-002 | "Code must be 1-50 uppercase alphanumeric characters" |
| CategoryName | VARCHAR | N/A | 1-200 characters | YES | Length: `LEN >= 1 AND LEN <= 200` | VAL-005 | "Category name is required" |
| Description | VARCHAR | N/A | 0-500 characters | NO | Length: `LEN <= 500` | VAL-007 | "Description cannot exceed 500 characters" |
| ParentCategoryID | UUID | N/A | Valid UUID or NULL | NO | If NOT NULL: `EXISTS(SELECT 1 FROM Category WHERE CategoryID = ParentCategoryID)` | VAL-001 | "Parent category does not exist" |
| SortOrder | INT | N/A | 0-9999 | NO | Range check | N/A | System default |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Code uniqueness per parent | Same-level siblings must have unique codes | Reject duplicate | VAL-003 |
| BRV-002 | No circular dependency | Parent chain cannot loop back | Reject | VAL-004 |
| BRV-003 | Maximum depth | Cannot exceed 10 levels | Reject | VAL-006 |
| BRV-004 | Parent same company | Parent must be in same company | Reject | VAL-008 |

**Cross-Field Validation Logic:**

```
IF (ParentCategoryID IS NOT NULL)
   AND NOT IsValidHierarchyPath(CategoryID, ParentCategoryID, 10)
THEN throw VAL-004 Error: "Circular dependency detected"
```

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|
| VAL-001 | Error | Validation | Invalid parent category | Parent category does not exist | Select a valid parent or leave empty |
| VAL-002 | Error | Validation | Invalid code format | Code must be 1-50 uppercase alphanumeric | Use only letters, numbers, underscores |
| VAL-003 | Error | Business Rule | Duplicate code | Code already exists at this level | Choose a unique code |
| VAL-004 | Error | Business Rule | Circular dependency | Circular dependency in category hierarchy | Restructure the hierarchy |
| VAL-005 | Error | Validation | Name required | Category name is required | Enter a category name |
| VAL-006 | Error | Business Rule | Max depth exceeded | Maximum hierarchy depth is 10 levels | Reorganize the category structure |
| VAL-007 | Error | Validation | Description too long | Description cannot exceed 500 characters | Shorten the description |
| VAL-008 | Error | Business Rule | Cross-company parent | Parent must be in same company | Select parent from same company |
| VAL-009 | Warning | Business Rule | Has child categories | Category has child categories | Handle children before delete |
| AUTH-001 | Error | Authorization | Permission denied | User lacks Category Management permission | Contact administrator |
| CFG-001 | Error | Configuration | Not initialized | System initialization not complete | Run system setup |
| CTX-001 | Error | Context | No company | Company context required | Select a company |
