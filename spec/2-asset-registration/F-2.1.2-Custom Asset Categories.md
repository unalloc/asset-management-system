# F-2.1.2 - Custom Asset Categories

**Requirement ID:** F-2.1.2  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** Medium

---

## Requirement Description

Allow users to create and manage custom asset categories and subcategories based on enterprise-specific needs. Categories are organized hierarchically and support attribute inheritance from parent to child categories.

---

## Business Rules

1. Category codes must be unique among siblings at the same hierarchy level
2. Parent category must exist if one is specified
3. Circular dependencies in category hierarchy are not allowed
4. Maximum hierarchy depth is 10 levels
5. Child categories automatically inherit attributes from parent
6. Disabled categories cannot be used for new asset classification
7. Categories are scoped to the legal entity (company) level

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Category Code | Yes | Unique identifier code for the category |
| Category Name | Yes | Display name for the category |
| Parent Category | No | Reference to parent category for hierarchy |
| Description | No | Detailed description of the category |
| Sort Order | No | Display ordering within siblings |
| Enabled Status | Yes | Whether category is active |

---

## Output Data

| Data | Description |
|------|-------------|
| Category Record | Created category with hierarchy position |
| Attribute Inheritance | Configuration of inherited attributes |
| Version History | Record of all changes to the category |

---

## Validation Rules

1. Category code must be 1-50 uppercase alphanumeric characters or underscores
2. Category name is required and must be 1-200 characters
3. Description cannot exceed 500 characters
4. Parent category must exist if specified
5. Cannot create circular reference in hierarchy
6. Cannot exceed maximum hierarchy depth of 10 levels

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full control across all companies |
| Company Admin | Create, modify, delete categories within own company |
| Division Manager | View categories within division |
| Plant Manager | View categories within plant |
| Department User | View own department categories (read-only) |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Company Admin can manage categories; other roles have view-only access
- **Isolation**: Each company can only view and manage its own categories

---

## Compliance Requirements

- **SOX**: Categories map to financial reporting segments for fixed asset disclosure
- **GDPR**: No personal data in category definitions
- **US GAAP/IFRS**: Category hierarchy supports PP&E depreciation class mapping
