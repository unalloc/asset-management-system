# F-2.1.13 - Criticality Classification

## Requirement ID
F-2.1.13

## Function Name
Criticality Classification

## Function Description
Classify assets by A/B/C criticality based on production impact.

## Business Rules

1. Class A: Production critical, failure causes production stoppage
2. Class B: Production important, failure affects capacity
3. Class C: Non-critical, failure has limited impact
4. Dynamic criticality assessment

## Preconditions

- Asset is registered
- Criticality assessment criteria defined
- User has "Criticality Management" permission

## Postconditions

- Asset criticality classification completed
- Related strategies applied (maintenance, spare parts, etc.)

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Asset ID | Yes | Target asset |
| Criticality Level | Yes | A/B/C |
| Assessment Basis | Yes | Justification for rating |
| Impact Scope Description | No | Affected areas/processes |
| Assessment Valid Until | No | Expiration date |

## Output Data

1. Criticality classification confirmation
2. Classification record ID
3. Related strategy application results

## Validation Rules

1. Asset must exist
2. Criticality level must be valid (A, B, or C)
3. Assessment basis cannot be empty

## Error Handling

| Error | Message |
|-------|---------|
| Invalid level | "Criticality level must be A, B, or C" |
| Asset not found | "Asset ID does not exist" |
| Assessment basis missing | "Assessment basis is required" |

## Priority
High (for manufacturing enterprises)

## Related User Stories
US-041, US-042
