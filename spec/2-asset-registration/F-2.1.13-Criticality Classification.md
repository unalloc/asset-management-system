## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Risk-based asset management (Clause 6.1.2) |
| **ISO 55001 Physical Asset Management** | Yes | Criticality assessment per Clause 6.4.6 |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Risk-based prioritization aligned |

**ISO 55000 Alignment Explanation:**

Criticality classification directly supports ISO 55000's risk-based asset management by providing:
- Asset prioritization for maintenance planning
- Spare parts determination based on criticality
- Maintenance frequency based on criticality level
- Resource allocation based on asset importance

**Alignment Gaps:**
- No explicit FMEA (Failure Mode and Effects Analysis) integration
- Missing support for RCM (Reliability-Centered Maintenance) criticality

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset is registered | `AssetExists(AssetID) == TRUE` | VAL-001 | "Asset does not exist" |
| PRE-002 | Criticality criteria defined | `CriteriaDefined() == TRUE` | CFG-001 | "Criticality criteria not defined" |
| PRE-003 | User has "Criticality Management" permission | `PermissionCheck(user, 'CRITICALITY_MGMT') == TRUE` | AUTH-001 | "User does not have Criticality Management permission" |
| PRE-004 | Assessment basis provided | `AssessmentBasis IS NOT NULL AND LENGTH > 0` | VAL-002 | "Assessment basis is required" |

**Pre-condition Validation Matrix:**

```
IF AssetExists(AssetID)
   AND CriteriaDefined()
   AND UserHasPermission('CRITICALITY_MGMT')
   AND AssessmentBasis IS NOT NULL
THEN proceed to classification
ELSE throw appropriate error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Criticality Classification
  │
  ▼
Validate Asset ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Criteria Defined ──► [Not Defined] ──► CFG-001 Error
  │
  ▼ [Defined]
Validate Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Assessment Basis ──► [Missing] ──► VAL-002 Error
  │
  ▼ [Provided]
Evaluate Criticality Level
  │
  ├── Calculate based on criteria
  ├── Impact scope
  └── Failure probability
  │
  ▼
Assign Criticality Level (A/B/C)
  │
  ▼
Apply Maintenance Strategy
  │
  ▼
Apply Spare Parts Strategy
  │
  ▼
Log Classification
  │
  ▼
Return Classification Result
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Level A - Critical | Failure causes production stoppage | Preventive maintenance | 1 |
| BR-001 | Level B - Important | Failure affects capacity | Condition-based | 1 |
| BR-001 | Level C - Non-critical | Failure has limited impact | Run-to-failure | 1 |
| BR-002 | Dynamic Assessment | Reassess when criteria change | Trigger reassessment | 2 |
| BR-003 | Expiry Tracking | Track assessment validity | Warn when expiring | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Change criticality level | Reassess and update | Preserve history |
| EC-002 | Assessment expires | Warn, require reassessment | WRN-001 |
| EC-003 | Asset decommissioned | Clear criticality | Archive record |
| EC-004 | Multiple criteria conflict | Use highest criticality | Conservative approach |

### 3.5 State Transitions

| Current State | Event | Next State | Side Effects |
|---------------|-------|------------|---------------|
| PENDING | Assess | ACTIVE | Strategies applied |
| ACTIVE | Reassess | ACTIVE | History record |
| ACTIVE | Decommision | ARCHIVED | Clear active classification |
| ACTIVE | Expire | EXPIRED | Warning generated |
