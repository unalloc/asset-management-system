# F-2.1.13 - Criticality Classification

**Requirement ID:** F-2.1.13  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** High (for manufacturing enterprises)

---

## Requirement Description

Classify assets by A/B/C criticality based on production impact to enable risk-based maintenance prioritization. Critical assets receive more frequent maintenance and higher priority for spare parts allocation.

---

## Business Rules

1. Criticality levels are A (Critical), B (Important), and C (Non-critical)
2. Level A assets: Failure causes production stoppage, require preventive maintenance
3. Level B assets: Failure affects capacity, use condition-based maintenance
4. Level C assets: Failure has limited impact, can use run-to-failure strategy
5. Criticality assessments have a validity period and must be renewed
6. Changes to criticality level preserve history
7. Maintenance and spare parts strategies are automatically applied based on criticality

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset ID | Yes | Asset being classified |
| Criticality Level | Yes | A, B, or C classification |
| Assessment Basis | Yes | Justification for the classification |
| Impact Scope | No | Description of failure impact |
| Valid Until | No | Expiration date of assessment |
| Assessed By | Yes | Person conducting assessment |

---

## Output Data

| Data | Description |
|------|-------------|
| Criticality Record | Current classification with history |
| Maintenance Strategy | Auto-generated strategy based on level |
| Criticality History | Record of all changes |

---

## Validation Rules

1. Asset must exist in the system
2. Criticality level must be A, B, or C
3. Assessment basis is required and must be 1-1000 characters
4. Valid until date must be after assessment date
5. Criticality criteria must be defined before classification

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full criticality management across companies |
| Company Admin | Full management within company |
| Plant Manager | Manage criticality within plant |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Criticality classifications scoped to company
- **Isolation**: Classifications isolated per company
