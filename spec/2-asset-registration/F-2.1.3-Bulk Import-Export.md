## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Supports efficient asset data migration and consolidation (Clause 6.3) |
| **ISO 55001 Physical Asset Management** | Yes | Enables bulk asset register population (Clause 6.4.3.2) |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Data migration follows documented procedures |

**ISO 55000 Alignment Explanation:**

Bulk import/export supports ISO 55000's support requirements by providing:
- Efficient initial asset data population during implementation
- Data migration capabilities for system transitions
- Backup/export for disaster recovery scenarios
- Audit data extraction for compliance reporting

**Alignment Gaps:**
- No explicit data validation against ISO 55000 asset data requirements
- Missing validation for regulatory field requirements (OSHA, EPA)

### 2.2 US/EU Regulatory Compliance

| Regulation | Applies | Compliance Status | Evidence/Mapping |
|------------|---------|-------------------|------------------|
| **GDPR** (EU) | Yes | Compliant | Bulk operations respect data privacy; PII fields masked in export |
| **SOX** (US) | Yes | Compliant | Audit trail on bulk operations; approval workflow |
| **OSHA** (US) | Partial | Gap | No validation for OSHA-required fields in bulk import |
| **EPA** (US) | Partial | Gap | No validation for EPA environmental fields |
| **WEEE Directive** (EU) | Partial | Gap | No WEEE category validation in bulk import |
| **IFRS 16** (EU/US) | Yes | Compliant | IFRS/GAAP depreciation fields supported |
| **US GAAP** (US) | Yes | Compliant | ASC 360 asset fields supported |
| **GAMP 5** (Pharma) | Partial | Gap | GMP data integrity requirements need explicit support |

**Regulatory Compliance Details:**

- **SOX**: Bulk operations logged with full audit trail; segregation of duties maintained
- **GDPR**: PII fields (assigned-to, maintained-by) excluded from bulk export unless authorized
- **GAMP 5**: Electronic signature requirements for GMP-critical data imports

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | User has "Bulk Operations" permission | `PermissionCheck(user, 'BULK_OPERATIONS') == TRUE` | AUTH-001 | "User does not have Bulk Operations permission" |
| PRE-002 | File format is supported | `FileFormat IN ('CSV', 'XLSX', 'XLS', 'JSON')` | VAL-001 | "Unsupported file format" |
| PRE-003 | File size within limit | `FileSize <= 100MB` | VAL-002 | "File size exceeds 100MB limit" |
| PRE-004 | Company context established | `Session.CompanyCode IS NOT NULL` | CTX-001 | "Company context required" |
| PRE-005 | Import template matches file version | `TemplateVersion == File.Version` | VAL-003 | "Template version mismatch" |

**Pre-condition Validation Matrix:**

```
IF UserHasPermission('BULK_OPERATIONS')
   AND FileFormatIsValid(File)
   AND FileSize <= 100MB
   AND Session.CompanyCode IS NOT NULL
   AND TemplateVersionMatches(File)
THEN proceed to file processing
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Asset data successfully imported | Records created/updated in DB | DB record count verification |
| POST-002 | Import result report generated | Report record exists | Report table query |
| POST-003 | Audit log entry created | AuditLog record with BULK_IMPORT action | Audit log verification |

**Post-condition Verification:**
- Database records created/updated as expected
- Import report generated with success/failure statistics
- AuditLog entry for each bulk operation
- Integration events published for downstream systems

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Bulk Import Request
  │
  ▼
Validate File Format ──► [Invalid] ──► VAL-001 Error (abort)
  │
  ▼ [Valid]
Validate File Size ──► [Exceeds 100MB] ──► VAL-002 Error (abort)
  │
  ▼ [Valid]
Parse File Content
  │
  ▼
Validate Template Version ──► [Mismatch] ──► VAL-003 Warning (can proceed)
  │
  ▼
Field Mapping Configuration
  │
  ▼
Row-by-Row Validation Loop
  │
  ├──► [Validation Error] ──► Log to Error Report ──► Continue or Abort
  │
  ▼
Business Rule Validation Loop
  │
  ├──► [Business Rule Failure] ──► Log to Error Report ──► Continue or Abort
  │
  ▼
Transaction Processing (per mode)
  │
  ├── Mode=CREATE: Insert records
  ├── Mode=UPDATE: Update existing records
  └── Mode=REPLACE: Delete + Insert
  │
  ▼
Generate Import Report
  │
  ▼
Publish BulkImportCompleted Event
  │
  ▼
Return Result Summary
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | File Format Validation | File extension and MIME type must match | Reject if mismatch | 1 |
| BR-002 | Required Field Validation | All required fields present and non-empty | Reject row if missing | 1 |
| BR-003 | Data Type Validation | Values match expected data types | Reject row if invalid | 1 |
| BR-004 | Duplicate Detection | Asset code/ID already exists | Apply mode logic (skip/update/error) | 1 |
| BR-005 | Cross-Reference Validation | Referenced entities exist | Reject row if invalid reference | 1 |
| BR-006 | Import Mode Handling | Mode determines behavior | CREATE/UPDATE/REPLACE | 1 |
| BR-007 | Batch Size Limiting | Process in batches of 1000 | Prevents memory overflow | 1 |
| BR-008 | Transaction Boundaries | Each batch is a transaction | Rollback on failure | 1 |

**Calculation Logic (if applicable):**
N/A - No financial calculations

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Empty file | Reject with error | VAL-004: File contains no data |
| EC-002 | File with only headers | Reject with error | VAL-005: No data rows found |
| EC-003 | Partial failure (some rows fail) | Continue with Error Handling Strategy | Skip/Report/Abort based on config |
| EC-004 | Large file (>50MB) | Process in chunks | Memory-efficient streaming |
| EC-005 | Duplicate asset codes in file | Apply duplicate resolution rules | First wins or reject |
| EC-006 | Circular reference in hierarchy import | Detect and reject row | VAL-006: Circular dependency |
| EC-007 | Invalid characters in data | Sanitize or reject | Based on field validation rules |
| EC-008 | Encoding issues | Detect and convert | Default UTF-8, fallback to detected |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| ErrorHandling=SKIP | RowValidation=FAIL | BusinessRule=FAIL | Skip row, log error |
| ErrorHandling=REPORT | RowValidation=FAIL | BusinessRule=FAIL | Skip row, log error, continue |
| ErrorHandling=ABORT | RowValidation=FAIL | - | Rollback transaction |
| Mode=REPLACE | AssetExists=TRUE | - | Delete then insert |
| Mode=UPDATE | AssetExists=FALSE | - | Skip or create based on config |

### 3.5 State Transitions

**Import Job State Diagram:**

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   PENDING   │──────►│  VALIDATING │──────►│ PROCESSING │
└─────────────┘       └─────────────┘       └─────────────┘
                                               │         │
                            ┌──────────────────┘         │
                            ▼                            ▼
                     ┌─────────────┐             ┌─────────────┐
                     │  COMPLETED  │             │   FAILED   │
                     └─────────────┘             └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| PENDING | Start validation | VALIDATING | File valid | Log start time |
| VALIDATING | Pass validation | PROCESSING | Template matches | Init batch counter |
| VALIDATING | Fail validation | FAILED | Critical error | Log validation errors |
| PROCESSING | All batches complete | COMPLETED | No critical errors | Generate report |
| PROCESSING | Batch failure | FAILED | ErrorHandling=ABORT | Rollback |
| ANY | Cancel requested | FAILED | User cancels | Rollback pending |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Required field presence | All required columns present | Reject file | VAL-008 |
| BRV-002 | Data type consistency | Values match column types | Reject row | VAL-009 |
| BRV-003 | Reference integrity | FK values exist | Reject row | VAL-010 |
| BRV-004 | Duplicate in file | Same key in multiple rows | Warn or reject | VAL-011 |
| BRV-005 | Value range check | Numeric/date in valid range | Reject row | VAL-012 |
| BRV-006 | Pattern validation | Code format matches pattern | Reject row | VAL-013 |

**Cross-Field Validation Logic:**

```
FOR EACH row in ImportFile
   IF row.AssetCode IS NOT NULL
      AND row.AssetTypeCode IS NOT NULL
      AND NOT Exists(AssetType WHERE TypeCode = row.AssetTypeCode AND CompanyCode = Session.CompanyCode)
   THEN reject row with VAL-010 Error
```

| VAL-004 | Error | File | Empty file | No data found | Add data to file |
