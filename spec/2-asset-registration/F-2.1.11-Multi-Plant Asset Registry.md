## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Multi-site asset management (Clause 6.1.2) |
| **ISO 55001 Physical Asset Management** | Yes | Plant-level asset control (Clause 6.4) |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Multi-site organization guidance |

**ISO 55000 Alignment Explanation:**

Multi-plant asset registry directly supports ISO 55000's organizational requirements by providing:
- Plant-level asset data isolation
- Cross-plant asset visibility for group reporting
- Plant-specific asset configurations
- Inter-plant asset transfer tracking

**Alignment Gaps:**
- No explicit support for ISO 55001:2022 multi-site certification
- Missing global asset registry consolidation framework

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Multi-plant structure defined | `PlantExists(PlantID) == TRUE` | VAL-001 | "Plant does not exist" |
| PRE-002 | User has "Multi-Plant Management" permission | `PermissionCheck(user, 'MULTI_PLANT_MGMT') == TRUE` | AUTH-001 | "User does not have Multi-Plant Management permission" |
| PRE-003 | Plant is active | `Plant.Status = 'ACTIVE'` | VAL-002 | "Plant is not active" |
| PRE-004 | Local asset number unique | `NOT Exists(Asset WHERE LocalNumber = Number AND PlantID = PlantID)` | VAL-003 | "Asset number already exists in this plant" |

**Pre-condition Validation Matrix:**

```
IF PlantExists(PlantID)
   AND Plant.IsActive == TRUE
   AND UserHasPermission('MULTI_PLANT_MGMT')
   AND LocalNumberIsUnique(Number, PlantID)
THEN proceed to asset registration
ELSE throw appropriate error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Multi-Plant Asset Registration
  │
  ▼
Validate Plant Exists ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Plant Active ──► [Not Active] ──► VAL-002 Error
  │
  ▼ [Active]
Validate Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Local Number ──► [Duplicate] ──► VAL-003 Error
  │
  ▼ [Unique]
Register Asset with Plant
  │
  ▼
Create Global Asset ID
  │
  ▼
Create Plant-Specific Record
  │
  ▼
Apply Plant Attributes
  │
  ▼
Log Asset Registration
  │
  ▼
Return Success with Both IDs
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Plant Active | Plant must be active | Reject if inactive | 1 |
| BR-002 | Local Number Unique | Within plant, number must be unique | Reject if duplicate | 1 |
| BR-003 | Plant Attributes | Apply plant-specific attributes | Auto-apply | 1 |
| BR-004 | Cross-Plant View | Group users can view all plants | Filter by permission | 1 |
| BR-005 | Plant Responsible | Track plant responsible person | Required field | 1 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Inter-plant transfer | Move asset to new plant | Create transfer record |
| EC-002 | Plant closure | Mark assets for transfer or disposal | Require disposition plan |
| EC-003 | Duplicate global ID | Global ID already exists | VAL-004: Duplicate global asset |
| EC-004 | Plant hierarchy change | Reassign to new parent plant | Update plant reference |

### 3.5 State Transitions

| Current State | Event | Next State | Side Effects |
|---------------|-------|------------|---------------|
| REGISTERED | Plant assignment | ACTIVE | Plant ID linked |
| ACTIVE | Transfer | TRANSFERRED | New plant ID |
| ACTIVE | Disposal | DISPOSED | Plant ID cleared |
| TRANSFERRED | Receive at new plant | ACTIVE | New plant linked |
