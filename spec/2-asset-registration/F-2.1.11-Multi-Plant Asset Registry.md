# F-2.1.11 - Multi-Plant Asset Registry

**Requirement ID:** F-2.1.11  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** High (for enterprise groups)

---

## Requirement Description

Support asset registration and management in multi-plant environments with plant-specific attributes. Each plant maintains its own asset registry while maintaining a global asset ID for cross-plant visibility and group reporting.

---

## Business Rules

1. Assets are registered at a specific plant with a local asset number
2. Each asset receives a global asset ID for cross-plant tracking
3. Plants must be active before assets can be registered
4. Local asset numbers must be unique within each plant
5. Plant-specific attributes can be applied to assets
6. Inter-plant transfers move assets between plants with full history
7. Group users can view assets across all plants within their permission scope

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Plant ID | Yes | Plant where asset is being registered |
| Local Asset Number | No | Plant-specific asset identifier |
| Global Asset ID | System | System-generated global identifier |
| Plant Responsible Person | No | Person responsible for asset at plant |
| Plant Specific Attributes | No | JSON structure with plant-specific data |

---

## Output Data

| Data | Description |
|------|-------------|
| Asset Record | Created with global and plant-specific IDs |
| Plant Assignment | Current plant association |
| Asset History | Complete registration and transfer history |

---

## Validation Rules

1. Plant must exist and be active
2. Local asset number must be unique within the plant (if provided)
3. Plant responsible person must exist if specified
4. Plant-specific attributes must be valid JSON if provided

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full cross-company and multi-plant access |
| Company Admin | Full access within company |
| Division Manager | View/modify within division |
| Plant Manager | Register and manage assets within plant |
| Plant User | View assets at own plant |

---

## Multi-Company Data Scope

- **Data Level**: Plant level
- **Access**: Assets visible within organizational hierarchy
- **Isolation**: Plant data isolated with access controls

---

## Compliance Requirements

- **GDPR**: Plant-level data residency compliance
- **SOX**: Multi-entity consolidation and elimination entries
- **OSHA/EPA**: Plant-specific regulatory compliance tracking
- **GAMP 5**: Site-specific validation and qualification
