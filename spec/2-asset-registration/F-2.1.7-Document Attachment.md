# F-2.1.7 - Document Attachment

**Requirement ID:** F-2.1.7  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** Medium

---

## Requirement Description

Allow attaching related documents to assets such as warranties, manuals, certificates, and safety data sheets. Documents are stored with version control and full audit trail, supporting regulatory compliance requirements.

---

## Business Rules

1. Documents must be in an allowed file format (PDF, Office documents, images)
2. File size is limited to 50MB per document
3. Each document must have a title and document type
4. Document types include WARRANTY, MANUAL, CERTIFICATE, SDS, PERMIT, DRAWING, PHOTO, and OTHER
5. Duplicate files are detected via file hash comparison
6. All documents are virus-scanned before storage
7. Documents can be superseded by newer versions while preserving history
8. Document access follows asset permission rules

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset ID | Yes | Asset to attach document to |
| Document Title | Yes | Display name for the document |
| Document Type | Yes | Type of document (WARRANTY, MANUAL, etc.) |
| Description | No | Additional description |
| File | Yes | Document file to upload |
| Effective Date | No | Date when document becomes effective |
| Expiry Date | No | Date when document expires |

---

## Output Data

| Data | Description |
|------|-------------|
| Document Record | Created document with metadata |
| Version History | Record of all versions |
| Thumbnail | Generated thumbnail for preview (images/PDFs) |
| Audit Log | Complete trail of document operations |

---

## Validation Rules

1. Asset must exist before attaching documents
2. File format must be supported and match extension
3. File size must not exceed 50MB
4. Document title is required and must be 1-200 characters
5. Document type must be one of the valid types
6. Expiry date must be after effective date if both provided
7. Files are virus-scanned before storage

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full document management across all companies |
| Company Admin | Upload, view, delete documents within company |
| Plant Manager | Manage documents within plant scope |
| Department User | Upload and view own documents |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Documents are scoped to the company of the asset
- **Isolation**: Document storage and metadata isolated by company

---

## Compliance Requirements

- **SOX**: Financial documents have complete audit trail and retention
- **OSHA**: Safety Data Sheets (SDS) and safety manuals linked to assets
- **GAMP 5**: Document version control supports 21 CFR Part 11 compliance
- **GDPR**: Personal data in documents protected by access control
