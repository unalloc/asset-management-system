## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Document control per Clause 7.5 |
| **ISO 55001 Physical Asset Management** | Yes | Document linkage for asset information (Clause 6.4.3) |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Document management aligned with Clause 7.5 |

**ISO 55000 Alignment Explanation:**

Document attachment supports ISO 55000's document control requirements by providing:
- Centralized document storage linked to assets
- Version control for asset-related documentation
- Access control for sensitive documents
- Retention management for regulatory compliance

**Alignment Gaps:**
- No explicit document collaboration workflow
- Missing support for ISO 9001 document control

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset is registered | `AssetExists(AssetID) == TRUE` | VAL-001 | "Asset does not exist" |
| PRE-002 | Document files are prepared | `FileSize <= 50MB AND FileFormat IN allowed list` | VAL-002 | "File exceeds limit or unsupported format" |
| PRE-003 | User has "Document Management" permission | `PermissionCheck(user, 'DOCUMENT_MANAGEMENT') == TRUE` | AUTH-001 | "User does not have Document Management permission" |
| PRE-004 | Required fields complete | `DocumentTitle IS NOT NULL AND DocumentType IS NOT NULL` | VAL-003 | "Required field missing" |
| PRE-005 | Document type is valid | `DocumentType IN ('WARRANTY', 'MANUAL', 'CERTIFICATE', 'SDS', 'PERMIT', 'DRAWING', 'PHOTO', 'OTHER')` | VAL-004 | "Invalid document type" |

**Pre-condition Validation Matrix:**

```
IF AssetExists(AssetID)
   AND FileIsValid(File)
   AND UserHasPermission('DOCUMENT_MANAGEMENT')
   AND DocumentTitle IS NOT NULL
   AND DocumentType IS VALID
THEN proceed to document upload
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Document uploaded and associated | DocumentID created, file stored | Database + storage verification |
| POST-002 | Document metadata recorded | Metadata record complete | Database query |
| POST-003 | Notification sent (if required) | Notification delivered | Notification log |

**Post-condition Verification:**
- Document record exists in database
- File stored in document storage
- AuditLog entry created

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Document Upload Request
  │
  ▼
Validate Asset Exists ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate File Format ──► [Unsupported] ──► VAL-002 Error
  │
  ▼ [Supported]
Validate File Size ──► [Exceeds 50MB] ──► VAL-002 Error
  │
  ▼ [Valid]
Validate User Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Required Fields ──► [Missing] ──► VAL-003 Error
  │
  ▼ [Complete]
Validate Document Type ──► [Invalid] ──► VAL-004 Error
  │
  ▼ [Valid]
Store Document File
  │
  ├── Generate unique storage path
  ├── Calculate file hash (SHA-256)
  └── Store file
  │
  ▼
Create Document Record
  │
  ▼
Generate Thumbnail (if applicable)
  │
  ▼
Log Document Upload
  │
  ▼
Publish DocumentAttached Event
  │
  ▼
Return Success Response
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | File Size Limit | FileSize <= 50MB | Reject if exceeded | 1 |
| BR-002 | Format Validation | MIME type matches extension | Reject if mismatch | 1 |
| BR-003 | Document Versioning | New version creates revision | Update existing record | 1 |
| BR-004 | Access Control | Document access follows asset permissions | Enforce permission | 1 |
| BR-005 | Thumbnail Generation | Auto-generate for images/PDFs | Generate thumbnail | 2 |
| BR-006 | Virus Scan | Scan file before storage | Quarantine if infected | 1 |
| BR-007 | Duplicate Detection | Hash collision check | Warn if duplicate | 2 |

**Calculation Logic (if applicable):**
N/A - No calculations required

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Duplicate file upload | Compare hash, warn user | Option to link existing |
| EC-002 | Large PDF (near 50MB) | Allow but warn about performance | Warning message |
| EC-003 | Corrupt file | Reject with error | VAL-005: Invalid file |
| EC-004 | Concurrent upload same document | Serialize uploads | Queue management |
| EC-005 | Document type change | Allow update, preserve history | Update metadata |
| EC-006 | Replace existing document | Create new version | Keep version history |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| FileValid=TRUE | HashExists=TRUE | - | Warn duplicate |
| FileValid=TRUE | HashExists=FALSE | - | Store new |
| FileSize > 50MB | - | - | Reject VAL-002 |
| FileCorrupt=TRUE | - | - | Reject VAL-005 |

### 3.5 State Transitions

**Document State Diagram:**

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│  UPLOADING  │──────►│   ACTIVE    │──────►│  ARCHIVED   │
└─────────────┘       └─────────────┘       └─────────────┘
                            │                      │
                            ▼                      │
                     ┌─────────────┐               │
                     │  SUPERSEDED │◄──────────────┘
                     └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |

| SUPERSEDED | Reactivate | ACTIVE | User request | Remove superseded |
