# F-2.1.4 - Barcode/QR Code Generation

**Requirement ID:** F-2.1.4  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** High

---

## Requirement Description

Automatically generate asset barcode or QR code labels and support printing functionality. Labels include machine-readable codes and human-readable asset identification information.

---

## Business Rules

1. Assets must be registered and active to generate new labels
2. Supported barcode formats include CODE128, CODE39, QR, DATAMATRIX, and PDF417
3. Barcode content encodes company code and asset code following GS1-128 standard
4. Print quantity is limited to 100 labels per request
5. All print operations are logged for audit purposes
6. Labels can be reprinted for inactive assets but new labels require active status

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset ID | Yes | Asset requiring label generation |
| Barcode Format | Yes | Type of barcode (CODE128, CODE39, QR, etc.) |
| Label Template | No | Template for label layout |
| Print Quantity | Yes | Number of labels to print (1-100) |

---

## Output Data

| Data | Description |
|------|-------------|
| Barcode Image | Generated barcode/QR code image |
| Print Job | Print job sent to printer |
| Print Log | Record of the print operation |
| Label History | Complete history of label generations |

---

## Validation Rules

1. Asset must exist and be in active status for new label generation
2. Barcode format must be one of the supported types
3. Print quantity must be between 1 and 100
4. Label template must exist if specified
5. For CODE128 format, encoded data must not exceed 48 characters

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full control across all companies |
| Company Admin | Full label operations within own company |
| Plant Manager | Print labels within plant scope |
| Department User | Print own assigned asset labels (read-only) |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Users can only print labels within their company
- **Isolation**: Templates and print logs are isolated per company

---

## Compliance Requirements

- **SOX**: All label generation operations are logged for audit trail
- **GDPR**: Barcode content contains only non-personal asset identifiers
