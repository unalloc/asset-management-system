## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Supports unique asset identification (Clause 6.4.3.1) |
| **ISO 55001 Physical Asset Management** | Yes | Barcode identification for physical asset tracking |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Automated ID supports asset tracking procedures |

**ISO 55000 Alignment Explanation:**

Barcode/QR code generation supports ISO 55000's asset identification requirements by providing:
- Automated unique identification for each physical asset
- Machine-readable labels enabling rapid asset location
- Integration point for mobile asset management workflows
- Foundation for physical inventory counts

**Alignment Gaps:**
- No explicit GS1 barcode standard compliance mentioned
- Missing support for ISO/IEC 15434 syntax for asset labels

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset is registered and active | `AssetExists(ID) AND Asset.Status = 'ACTIVE'` | VAL-001 | "Asset must exist and be active" |
| PRE-002 | User has "Label Printing" permission | `PermissionCheck(user, 'LABEL_PRINTING') == TRUE` | AUTH-001 | "User does not have Label Printing permission" |
| PRE-003 | Barcode format is supported | `BarcodeFormat IN ('CODE128', 'CODE39', 'QR', 'DATAMATRIX', 'PDF417')` | VAL-002 | "Unsupported barcode format" |
| PRE-004 | Label template exists | `TemplateID IS NULL OR Exists(Template WHERE TemplateID = TemplateID)` | VAL-003 | "Label template not found" |
| PRE-005 | Print quantity valid | `1 <= PrintQuantity <= 100` | VAL-004 | "Print quantity must be 1-100" |

**Pre-condition Validation Matrix:**

```
IF AssetIsValid(AssetID)
   AND UserHasPermission('LABEL_PRINTING')
   AND BarcodeFormatIsValid(Format)
   AND TemplateExists(TemplateID)
   AND PrintQuantityInRange(1, 100)
THEN proceed to label generation
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Asset label generated | Barcode image created and stored | File verification |
| POST-002 | Print task completed | Print job sent to printer | Print queue status |
| POST-003 | Label history recorded | PrintLog record created | Database verification |

**Post-condition Verification:**
- Barcode image file generated in storage
- Print task entry created in print queue
- Audit log entry for label generation

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Label Generation Request
  │
  ▼
Validate Asset Status ──► [Invalid] ──► VAL-001 Error
  │
  ▼ [Valid]
Validate Permissions ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Barcode Format ──► [Invalid] ──► VAL-002 Error
  │
  ▼ [Valid]
Validate Template ──► [Not Found] ──► VAL-003 Error
  │
  ▼ [Found]
Validate Print Quantity ──► [Invalid] ──► VAL-004 Error
  │
  ▼ [Valid]
Generate Barcode Content
  │
  ├── Code = Asset.CompanyCode + Asset.AssetCode
  ├── HumanReadable = Asset.AssetName
  └── AdditionalData = Asset.CustomFields
  │
  ▼
Generate Barcode Image
  │
  ▼
Apply Label Template Layout
  │
  ▼
Create Print Job(s)
  │
  ▼
Log Print History
  │
  ▼
Publish LabelGenerated Event
  │
  ▼
Return Barcode Image + Print Status
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Asset Status Check | Asset must be ACTIVE | Reject if not active | 1 |
| BR-002 | Barcode Content | Encode CompanyCode + AssetCode | Format per GS1-128 | 1 |
| BR-003 | Human Readable | Include asset name below barcode | Always | 2 |
| BR-004 | Print Quantity | Limit to 100 per request | Reject if exceeded | 1 |
| BR-005 | Print History | Log all print operations | Record with user/time | 1 |
| BR-006 | Reprint Allowed | Same label can be reprinted | No restrictions | 3 |

**Calculation Logic (if applicable):**
N/A - No calculations required

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Asset is inactive | Allow reprint of existing label | Warning only |
| EC-002 | Printer offline | Queue print job for retry | Retry mechanism |
| EC-003 | Very long asset code | Truncate or use QR | Use QR for long codes |
| EC-004 | Bulk label request | Process as batch | Queue management |
| EC-005 | Label template with custom fields | Include selected custom fields | Template expansion |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| AssetActive=TRUE | - | - | Generate label |
| AssetActive=FALSE | Operation=REPRINT | - | Allow with warning |
| AssetActive=FALSE | Operation=NEW | - | Reject |
| PrintQuantity > 100 | - | - | Reject VAL-004 |
| PrinterOffline=TRUE | Retry configured | - | Queue and retry |

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |

| BarcodeFormat | ENUM | N/A | CODE128/CODE39/QR/DATAMATRIX/PDF417 | YES | Value in list | VAL-002 | "Unsupported barcode format" |
| PrintQuantity | INT | N/A | 1-100 | YES | Range check | VAL-004 | "Print quantity must be 1-100" |
| TemplateID | UUID | N/A | Valid UUID or NULL | NO | If NOT NULL: `EXISTS(Template)` | VAL-003 | "Label template not found" |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Asset must be active | Asset.Status = 'ACTIVE' | Reject for new prints; warn for reprints | VAL-001 |
| BRV-002 | Barcode code length | Encoded data <= 48 chars (CODE128) | Use QR for longer data | Auto-switch to QR |
| BRV-003 | Template compatible | Template.BarcodeFormat matches request | Reject | VAL-005 |

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|
| VAL-001 | Error | Validation | Invalid asset | Asset does not exist or not active | Register asset first |
| VAL-002 | Error | Validation | Invalid format | Barcode format not supported | Use CODE128, CODE39, QR, DATAMATRIX, or PDF417 |
| VAL-003 | Error | Validation | Template not found | Label template does not exist | Select a valid template |
| VAL-004 | Error | Validation | Invalid quantity | Print quantity out of range | Enter 1-100 |
| VAL-005 | Error | Validation | Format mismatch | Template does not support requested format | Select compatible template |
| AUTH-001 | Error | Authorization | Permission denied | User lacks Label Printing permission | Contact administrator |
