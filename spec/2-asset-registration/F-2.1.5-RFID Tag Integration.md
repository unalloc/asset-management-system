**ISO 55000 Alignment Explanation:**

RFID integration supports ISO 55000's asset tracking requirements by providing:

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | RFID reader is connected | `RFIDReader.IsConnected == TRUE` | VAL-001 | "RFID device not connected" |
| PRE-002 | Tag hardware is available | `TagTypeSupported(TagType) == TRUE` | VAL-002 | "Tag type not supported" |
| PRE-003 | User has "RFID Management" permission | `PermissionCheck(user, 'RFID_MANAGEMENT') == TRUE` | AUTH-001 | "User does not have RFID Management permission" |
| PRE-004 | Asset exists | `AssetExists(AssetID) == TRUE` | VAL-003 | "Asset does not exist" |
| PRE-005 | Tag UID is valid | `TagUIDFormatValid(TagUID) == TRUE` | VAL-004 | "Invalid tag UID format" |
| PRE-006 | Tag not already assigned | `NOT Exists(Asset WHERE RFIDTagUID = TagUID AND AssetID != AssetID)` | VAL-005 | "Tag is already used by another asset" |

**Pre-condition Validation Matrix:**

```
IF RFIDReader.IsConnected == TRUE
   AND TagTypeSupported(TagType) == TRUE
   AND UserHasPermission('RFID_MANAGEMENT')
   AND AssetExists(AssetID) == TRUE
   AND TagUIDFormatValid(TagUID) == TRUE
   AND TagNotAssignedToOther(TagUID, AssetID)
THEN proceed to RFID association
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | RFID tag associated with asset | Asset.RFIDTagUID updated | Database verification |
| POST-002 | Tag data written | Tag memory contains expected data | Read-back verification |
| POST-003 | RFID operation logged | AuditLog entry created | Audit log query |

**Post-condition Verification:**
- Asset record updated with RFIDTagUID
- Tag read-back confirms data written
- AuditLog entry created with all details

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
RFID Tag Association Request
  │
  ▼
Check Reader Connection ──► [Not Connected] ──► VAL-001 Error
  │
  ▼ [Connected]
Check Tag Type Support ──► [Unsupported] ──► VAL-002 Error
  │
  ▼ [Supported]
Check User Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Check Asset Exists ──► [No] ──► VAL-003 Error
  │
  ▼ [Exists]
Validate Tag UID Format ──► [Invalid] ──► VAL-004 Error
  │
  ▼ [Valid]
Check Tag Availability ──► [Already Assigned] ──► VAL-005 Error
  │
  ▼ [Available]
Write Tag Data
  │
  ├── Encode: CompanyCode + AssetCode + AssetID
  ├── Write to tag memory
  └── Verify write
  │
  ▼
Update Asset Record
  │
  ▼
Log RFID Operation
  │
  ▼
Publish RFIDTagAssociated Event
  │
  ▼
Return Success Response
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Reader Connection | RFID reader must be connected | Reject if not | 1 |
| BR-002 | Tag One-to-One | One tag per asset, one asset per tag | Reject if already assigned | 1 |
| BR-003 | Tag Status Management | Track active/inactive/lost status | Update status field | 1 |
| BR-004 | Tag Data Encoding | GS1 Tag Data Standard format | Encode per standard | 1 |
| BR-005 | Batch Initialization | Support batch tag programming | Process in batches | 2 |
| BR-006 | Lost Tag Handling | Mark tag as lost, reassign new | Update status, create new | 2 |

**Calculation Logic (if applicable):**
N/A - No calculations required

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Tag read failure | Retry read 3 times, then error | VAL-006: Read failed |
| EC-002 | Tag write failure | Retry write 3 times, then abort | VAL-007: Write failed |
| EC-003 | Lost tag | Mark as LOST, allow new tag assignment | Update status, create history |
| EC-004 | Tag reuse after disposal | Validate asset status before reuse | VAL-008: Asset not active |
| EC-005 | Reader disconnection during operation | Rollback pending operation | VAL-001: Connection lost |
| EC-006 | Multiple tags in field | Read all, prompt user to select | Present tag list |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| TagExists=FALSE | ReadRetry < 3 | - | Retry read |
| TagExists=FALSE | ReadRetry >= 3 | - | VAL-006 Error |
| WriteFails=TRUE | WriteRetry < 3 | - | Retry write |
| WriteFails=TRUE | WriteRetry >= 3 | - | VAL-007 Error |
| AssetStatus != ACTIVE | Operation=ASSOCIATE | - | VAL-008 Error |

### 3.5 State Transitions

**RFID Tag State Diagram:**

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   BLANK     │──────►│   ACTIVE    │──────►│   INACTIVE  │
└─────────────┘       └─────────────┘       └─────────────┘
                              │                     │
                              │                     │
                              ▼                     │
                       ┌─────────────┐              │
                       │    LOST     │◄─────────────┘
                       └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| BLANK | Write asset data | ACTIVE | Write successful | Asset.RFIDTagUID updated |
| ACTIVE | Mark lost | LOST | User confirmation | History record |
| ACTIVE | Dissociate | INACTIVE | User confirmation | Asset.RFIDTagUID cleared |
| LOST | Assign new tag | ACTIVE | New tag written | History record |
| INACTIVE | Reactivate | ACTIVE | Tag re-initialization | Write new data |

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |
|------------|-----------|--------|-------|----------|-----------------|------------|---------------|
| TagUID | VARCHAR | Hex or decimal | 8-96 bits depending on type | YES | Pattern match per type | VAL-004 | "Invalid tag UID format" |
| TagType | ENUM | N/A | LF/HF/UHF | YES | Value in list | VAL-002 | "Unsupported tag type" |
| AssetID | UUID | N/A | Valid UUID | YES | `EXISTS(Asset)` | VAL-003 | "Asset does not exist" |
| WriteData | VARCHAR | N/A | Max 256 bytes | NO | Length check | VAL-009 | "Data too long for tag" |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Tag one-to-one | TagUID not assigned to another asset | Reject | VAL-005 |
| BRV-002 | Asset one-to-one | Asset.RFIDTagUID is NULL or matches | Reject if different tag | VAL-010 |
| BRV-003 | Tag capacity | Write data fits tag memory | Reject if overflow | VAL-009 |
| BRV-004 | Tag status active | Tag status != LOST | Warn if LOST | WRN-001 |

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|
| VAL-001 | Error | Hardware | Reader not connected | RFID device not connected | Check reader connection |
| VAL-002 | Error | Validation | Unsupported type | Tag type not supported | Use LF, HF, or UHF tags |
| VAL-003 | Error | Validation | Asset not found | Asset does not exist | Register asset first |
| VAL-004 | Error | Validation | Invalid UID | Invalid tag UID format | Check tag encoding |
| VAL-005 | Error | Business Rule | Tag assigned | Tag is already used by another asset | Use available tag or dissociate |
| VAL-006 | Error | Hardware | Read failed | Tag read operation failed | Retry or replace tag |
| VAL-007 | Error | Hardware | Write failed | Tag write operation failed | Retry or replace tag |
| VAL-008 | Error | Business Rule | Asset not active | Asset must be active | Check asset status |
| VAL-009 | Error | Validation | Data overflow | Data too long for tag memory | Reduce write data |
| VAL-010 | Error | Business Rule | Asset has tag | Asset already has different tag assigned | Dissociate current tag first |
| AUTH-001 | Error | Authorization | Permission denied | User lacks RFID Management permission | Contact administrator |

### 6.4 Warning Conditions
