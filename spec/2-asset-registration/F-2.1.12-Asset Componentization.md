# F-2.1.12 - Asset Componentization

**Requirement ID:** F-2.1.12  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** Medium

---

## Requirement Description

Decompose complex assets into components for independent tracking and detailed financial tracking. Components are tracked as separate assets but grouped under a main asset, enabling component-level depreciation, maintenance, and replacement tracking.

---

## Business Rules

1. Each component must be an independent asset in the system
2. Value proportions must sum to exactly 100% across all components
3. Components cannot be assigned to multiple main assets
4. Component roles include CORE, AUXILIARY, and SPARE
5. Component depreciation is allocated based on value proportions
6. Component replacements preserve full history
7. When the main asset is disposed, all components are also disposed

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Main Asset ID | Yes | Parent asset being componentized |
| Component Asset IDs | Yes | List of component assets |
| Value Proportions | Yes | Percentage of value for each component |
| Component Role | No | Role type (CORE, AUXILIARY, SPARE) |
| Replacement Strategy | No | Strategy for handling replacements |

---

## Output Data

| Data | Description |
|------|-------------|
| Component Configuration | Created configuration linking components |
| Financial Allocations | Depreciation allocated per component |
| Replacement History | Record of all component replacements |

---

## Validation Rules

1. Main asset must exist
2. All component assets must exist
3. Value proportions must each be between 0 and 100
4. Sum of all proportions must equal 100%
5. Components must not already be assigned to another main asset
6. All assets must be in the same company

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full component management across companies |
| Company Admin | Full management within company |
| Plant Manager | Manage within plant scope |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Component configurations scoped to company
- **Isolation**: Configurations isolated per company

---

## Compliance Requirements

- **US GAAP/IFRS**: Component depreciation per IAS 16.43 and ASC 360
- **SOX**: Component-level asset tracking for financial controls
- **WEEE**: Component-level end-of-life tracking for environmental compliance
