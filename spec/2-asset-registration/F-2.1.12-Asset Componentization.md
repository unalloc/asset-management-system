## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Asset decomposition per Clause 6.4.3 |
| **ISO 55001 Physical Asset Management** | Yes | Component lifecycle management |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Component tracking aligned |

**ISO 55000 Alignment Explanation:**

Asset componentization supports ISO 55000's asset management by providing:
- Detailed asset breakdown for maintenance planning
- Independent financial tracking per component
- Component replacement history
- Value proportion calculation for depreciation

**Alignment Gaps:**
- No explicit support for ISO 14240 asset breakdown structures
- Missing integration with FMEA for component criticality

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Main asset is registered | `AssetExists(MainAssetID) == TRUE` | VAL-001 | "Main asset does not exist" |
| PRE-002 | Components defined as independent assets | `All(ComponentIDs exist)` | VAL-002 | "One or more component assets do not exist" |
| PRE-003 | User has "Component Management" permission | `PermissionCheck(user, 'COMPONENT_MGMT') == TRUE` | AUTH-001 | "User does not have Component Management permission" |
| PRE-004 | Value proportions sum to 100% | `SUM(Proportions) = 100` | VAL-003 | "Value proportions must sum to 100%" |
| PRE-005 | Components not already assigned | `Components not assigned to other main assets` | VAL-004 | "Component is already used by another asset" |

**Pre-condition Validation Matrix:**

```
IF AssetExists(MainAssetID)
   AND AllComponentsExist(ComponentIDs)
   AND UserHasPermission('COMPONENT_MGMT')
   AND ProportionsSumTo100(ComponentIDs, Proportions)
   AND AllComponentsAvailable(ComponentIDs)
THEN proceed to componentization
ELSE throw appropriate error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Asset Componentization
  │
  ▼
Validate Main Asset ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Components ──► [Not Found] ──► VAL-002 Error
  │
  ▼ [Found]
Validate Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Proportions ──► [Sum != 100] ──► VAL-003 Error
  │
  ▼ [Valid]
Validate Component Availability ──► [Assigned] ──► VAL-004 Error
  │
  ▼ [Available]
Create Component Configuration
  │
  ▼
Link Components to Main Asset
  │
  ▼
Calculate Financial Allocations
  │
  ▼
Log Componentization
  │
  ▼
Return Success
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Value Sum | Proportions must sum to 100% | Reject if not | 1 |
| BR-002 | Component Availability | Components must not be assigned | Reject if assigned | 1 |
| BR-003 | Proportion Range | Each proportion must be > 0 and <= 100 | Reject if invalid | 1 |
| BR-004 | Component Independence | Components tracked separately | Create separate records | 1 |
| BR-005 | Replacement Tracking | Track component replacements | Create history | 2 |
| BR-006 | Depreciation Allocation | Allocate depreciation by proportion | Calculate per component | 1 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Proportions don't sum to 100 | Reject with error | VAL-003 |
| EC-002 | Add component to existing config | Update proportions | Recalculate all |
| EC-003 | Remove component | Redistribute proportions or reject | Require new proportions |
| EC-004 | Component replacement | Keep history, assign new component | Create replacement record |
| EC-005 | Main asset disposal | Dispose all components | Cascade disposal |

### 3.5 State Transitions

| Current State | Event | Next State | Side Effects |
|---------------|-------|------------|---------------|
| PENDING | Approve | ACTIVE | Allocations active |
| ACTIVE | Add component | ACTIVE | Recalculate proportions |
| ACTIVE | Remove component | ACTIVE | Recalculate proportions |
| ACTIVE | Replace component | ACTIVE | History record |
| ACTIVE | Dispose | DISPOSED | All components disposed |
