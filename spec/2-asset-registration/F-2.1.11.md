# F-2.1.11 - Multi-Plant Asset Registry

## Requirement ID
F-2.1.11

## Function Name
Multi-Plant Asset Registry

## Function Description
Support asset registration and management in multi-plant environments with plant-specific attributes.

## Business Rules

1. Assets belong to specific plants
2. Plant-specific attribute extension
3. Cross-plant asset views
4. Inter-plant asset transfer tracking

## Preconditions

- Multi-plant structure defined
- User has "Multi-Plant Management" permission

## Postconditions

- Asset registered under specified plant
- Plant attributes associated

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Asset Basic Information | Yes | Core asset data |
| Owning Plant ID | Yes | Plant assignment |
| Plant-Specific Attribute Values | No | Custom attributes |
| Local Asset Number | No | Plant-local identifier |
| Plant Responsible Person | No | Contact person |

## Output Data

1. Asset registration confirmation
2. Global asset ID
3. Plant asset ID
4. Plant attribute record ID

## Validation Rules

1. Plant must exist and be active
2. Local asset number unique within plant
3. Plant attributes conform to definition

## Error Handling

| Error | Message |
|-------|---------|
| Plant not found | "Specified plant does not exist" |
| Local number duplicate | "Asset number already exists in this plant" |
| Attribute validation failed | "Plant attribute value is invalid" |

## Priority
High (for enterprise groups)

## Related User Stories
US-047, US-062
