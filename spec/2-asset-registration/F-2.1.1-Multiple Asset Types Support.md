# F-2.1.1 - Multiple Asset Types Support

**Requirement ID:** F-2.1.1  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** High

---

## Requirement Description

The system shall support the registration and management of multiple asset types including but not limited to IT equipment, office furniture, vehicles, and production machinery. Asset types are classified hierarchically and each type can have specific attributes and templates.

---

## Business Rules

1. Each asset type must have a unique type code within the company
2. Asset types can be organized in a parent-child hierarchy
3. Child asset types inherit attributes from their parent type
4. Any change to an asset type creates a new version, preserving history
5. Disabled asset types cannot be used for new asset creation
6. Asset types are scoped to the legal entity (company) level

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset Type Code | Yes | Unique identifier code for the asset type |
| Asset Type Name | Yes | Display name for the asset type |
| Description | No | Detailed description of the asset type |
| Parent Type | No | Reference to parent asset type for hierarchy |
| Attribute Template | No | Template defining type-specific attributes |
| Status | Yes | Current status (Draft, Active, Inactive, Deprecated) |

---

## Output Data

| Data | Description |
|------|-------------|
| Asset Type Record | Created asset type with all attributes |
| Version History | Record of all changes made to the asset type |
| Audit Log | Complete trail of who created/modified the type |

---

## Validation Rules

1. Type code must be 1-20 uppercase alphanumeric characters or underscores
2. Type name is required and must be 1-100 characters
3. Description cannot exceed 500 characters
4. Parent type must exist if specified
5. Attribute template must exist if specified
6. No circular references allowed in type hierarchy
7. Cannot disable a type that has active assets without confirmation

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full control across all companies |
| Company Admin | Create, modify, delete asset types within own company |
| Division Manager | View asset types within division |
| Plant Manager | View asset types within plant |
| Department User | No access |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Company Admin can manage types; other roles have view-only within their scope
- **Isolation**: Each company can only view and manage its own asset types

---

## Compliance Requirements

- **SOX**: Asset type classification maps to financial reporting segments
- **GDPR**: No personal data processed in asset type definitions
- **US GAAP/IFRS**: Asset types support PP&E classification requirements
