### 1.1 Requirement Description

The system shall support the registration and management of multiple asset types including but not limited to IT equipment, office furniture, vehicles, and production machinery.

## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Asset type classification supports asset management planning (Clause 6.1) |
| **ISO 55001 Physical Asset Management** | Yes | Supports unique identification and asset register completeness per Clause 6.4.3 |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Hierarchical classification aligns with asset data structure guidelines |

**ISO 55000 Alignment Explanation:**

The asset type classification system directly supports ISO 55000's asset management planning requirements by providing:
- Structured asset identification system (Clause 4.2 - Asset management policy)
- Hierarchical organization enabling risk-based asset grouping (Clause 6.1.2)
- Foundation for asset lifecycle management (Clause 6.4.4)
- Support for maintenance optimization through type-specific templates (Clause 6.4.6)

**Alignment Gaps:**
- No explicit criticality classification linkage within asset types
- Missing sustainability/environmental aspect classification per ISO 14001 alignment

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | User has "Asset Registration" permission | `PermissionCheck(user, 'ASSET_REGISTRATION') == TRUE` | AUTH-001 | "User does not have Asset Registration permission" |
| PRE-002 | Asset type classification system is configured | `SELECT COUNT(*) FROM AssetTypeConfig > 0` | CFG-001 | "Asset type classification system not configured" |
| PRE-003 | Company context established | `Session.CompanyCode IS NOT NULL` | CTX-001 | "Company context required" |

**Pre-condition Validation Matrix:**

```
IF UserHasPermission('ASSET_REGISTRATION')
   AND AssetTypeConfigExists()
   AND Session.CompanyCode IS NOT NULL
THEN proceed to asset type creation
ELSE throw appropriate authorization/config error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | New asset type successfully registered | AssetTypeID generated, audit log created | DB query for new record + audit trail |
| POST-002 | Asset type information completely recorded | All required attributes persisted | Record count verification |
| POST-003 | Downstream systems notified | Integration event published | Event queue verification |

**Post-condition Verification:**

- New AssetType record exists in database with unique ID
- AuditLog entry created with CREATE action
- AssetTypeCreatedEvent published to integration bus

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Asset Type Create Request
  │
  ▼
Validate User Permissions ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Type Code ──► [Invalid] ──► VAL-001 Error
  │
  ▼ [Valid]
Check Duplicate ──► [Exists] ──► VAL-002 Error
  │
  ▼ [Unique]
Validate Parent Type ──► [Invalid] ──► VAL-003 Error
  │
  ▼ [Valid or N/A]
Validate Attribute Template ──► [Invalid] ──► VAL-004 Error
  │
  ▼ [Valid]
Create Asset Type Record
  │
  ▼
Publish AssetTypeCreated Event
  │
  ▼
Return Success Response
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Type Code Uniqueness | AssetTypeCode must be unique per company | Reject if duplicate | 1 |
| BR-002 | Parent Type Validation | If ParentTypeID specified | Validate parent exists and is same company | 1 |
| BR-003 | Attribute Inheritance | Child type inherits parent attributes | Auto-merge with child-specific attributes | 2 |
| BR-004 | Type Versioning | Any type field change | Create new version, preserve history | 1 |
| BR-005 | Type Status Management | Disabled types cannot be used | Reject asset creation with disabled type | 1 |

**Calculation Logic (if applicable):**
N/A - No calculations required for this requirement

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Parent type is disabled | Allow if child type being created | Warning: parent is inactive |
| EC-002 | Circular reference attempt | Reject with clear error | VAL-005: Circular dependency detected |
| EC-003 | Type has existing assets | Allow disable but warn | Require confirmation before disabling |
| EC-004 | Type code with special characters | Normalize or reject | VAL-006: Invalid character in type code |
| EC-005 | Duplicate type name in different company | Allow (company-scoped) | No error; type codes are company-unique |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| ParentExists=TRUE | ParentActive=TRUE | ChildInheritsAttr=TRUE | Proceed with inheritance |
| ParentExists=TRUE | ParentActive=FALSE | ChildInheritsAttr=TRUE | Warning + proceed |
| ParentExists=FALSE | - | - | VAL-003: Invalid parent |
| ParentExists=TRUE | CreatesCircular=TRUE | - | VAL-005: Circular dependency |

## 6. Enhanced Validation Rules

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |
|------------|-----------|--------|-------|----------|-----------------|------------|---------------|
| AssetTypeCode | VARCHAR | [A-Z0-9_]{1,20} | Uppercase alphanumeric + underscore | YES | Pattern: `^[A-Z0-9_]{1,20}$` | VAL-001 | "Type code must be 1-20 uppercase alphanumeric characters or underscores" |
| AssetTypeName | VARCHAR | N/A | 1-100 characters | YES | Length: `LEN >= 1 AND LEN <= 100` | VAL-002 | "Type name is required and must be 1-100 characters" |
| Description | VARCHAR | N/A | 0-500 characters | NO | Length: `LEN <= 500` | VAL-003 | "Description cannot exceed 500 characters" |
| ParentTypeID | UUID | N/A | Valid UUID or NULL | NO | If NOT NULL: `EXISTS(SELECT 1 FROM AssetType WHERE AssetTypeID = ParentTypeID)` | VAL-004 | "Parent type does not exist" |
| AttributeTemplateID | UUID | N/A | Valid UUID or NULL | NO | If NOT NULL: `EXISTS(SELECT 1 FROM AttributeTemplate WHERE TemplateID = AttributeTemplateID)` | VAL-005 | "Attribute template does not exist" |
| Status | ENUM | DRAFT/ACTIVE/INACTIVE/DEPRECATED | Valid status value | YES | Value IN ('DRAFT', 'ACTIVE', 'INACTIVE', 'DEPRECATED') | N/A | System-generated |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Type code uniqueness per company | CompanyCode + AssetTypeCode must be unique | Reject duplicate | VAL-006 |
| BRV-002 | No circular hierarchy | Parent chain cannot loop back to current type | Reject | VAL-007 |
| BRV-003 | Parent same company | Parent type must be in same company | Reject | VAL-008 |
| BRV-004 | Cannot delete type with assets | Type has no active asset children | Block or warn | VAL-009 |

**Cross-Field Validation Logic:**

```
IF (ParentTypeID IS NOT NULL)
   AND (ParentTypeID != AssetTypeID)
   AND NOT (IsValidHierarchyPath(AssetTypeID, ParentTypeID))  // No circular reference
THEN throw VAL-007 Error
```

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|
| VAL-001 | Error | Validation | Invalid type code format | Type code must be 1-20 uppercase alphanumeric characters | Enter type code using only letters, numbers, and underscores |
| VAL-002 | Error | Validation | Invalid type name | Type name is required | Enter a display name for the asset type |
| VAL-003 | Error | Validation | Description too long | Description cannot exceed 500 characters | Shorten the description |
| VAL-004 | Error | Validation | Invalid parent type | Parent type does not exist | Select a valid parent type or leave empty |
| VAL-005 | Error | Validation | Invalid template | Attribute template does not exist | Select a valid template or leave empty |
| VAL-006 | Error | Business Rule | Duplicate type code | Type code already exists in this company | Enter a unique type code |
| VAL-007 | Error | Business Rule | Circular dependency | Circular dependency detected in type hierarchy | Reorganize the type hierarchy |
| VAL-008 | Error | Business Rule | Cross-company parent | Parent type must be in same company | Select a parent type from the same company |
| VAL-009 | Warning | Business Rule | Type has assets | This type is used by existing assets | Consider reassigning assets before disabling |
| AUTH-001 | Error | Authorization | Permission denied | User does not have required permission | Contact administrator for Asset Registration permission |
| CFG-001 | Error | Configuration | Not configured | Asset type classification system not initialized | Run system initialization |
| CTX-001 | Error | Context | No company context | Company context required for this operation | Select a company before creating asset types |
