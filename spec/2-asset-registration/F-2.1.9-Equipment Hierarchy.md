# F-2.1.9 - Equipment Hierarchy

**Requirement ID:** F-2.1.9  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** High (for manufacturing enterprises)

---

## Requirement Description

Manage the hierarchical structure of production equipment following the ISA-95 standard: Plant → Production Line → Work Center → Machine. This hierarchy supports maintenance planning, production scheduling, and metrics aggregation across manufacturing operations.

---

## Business Rules

1. Equipment hierarchy has exactly 4 levels: PLANT, PRODUCTION_LINE, WORK_CENTER, MACHINE
2. Each node must have a unique code within its parent
3. Nodes can only be created under an existing plant
4. All nodes under the same plant must follow the strict 4-level structure
5. Nodes can be deactivated (INACTIVE) but remain in the hierarchy
6. All hierarchy changes are logged for audit purposes
7. Metrics roll up from lower levels to parent nodes

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Node Type | Yes | Level in hierarchy (PLANT, LINE, WORK_CENTER, MACHINE) |
| Node Code | Yes | Unique identifier for the node |
| Node Name | Yes | Display name |
| Parent Node | Conditional | Required for levels below PLANT |
| Plant ID | Yes | Plant this node belongs to |
| Level Attributes | No | Type-specific attributes as JSON |
| Location Info | No | Geographic/location information |

---

## Output Data

| Data | Description |
|------|-------------|
| Hierarchy Node | Created node with hierarchy position |
| Hierarchy Path | Full path from root to this node |
| Metrics | Aggregated metrics from child nodes |
| Audit History | Record of all changes |

---

## Validation Rules

1. Plant must exist before creating hierarchy nodes
2. Node type must be one of the valid types
3. Parent node must exist if not creating a top-level node
4. Node code must be unique within the parent
5. Maximum hierarchy depth of 4 levels must be maintained
6. Parent-child type combinations must be valid

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full hierarchy management across all companies |
| Company Admin | Create plant/line nodes; manage within company |
| Plant Manager | Create work center/machine nodes within plant |
| Work Center Lead | View hierarchy for own work center |

---

## Multi-Company Data Scope

- **Data Level**: Plant level
- **Access**: Nodes scoped to plant; hierarchy visible within company
- **Isolation**: Nodes cannot cross plant boundaries

---

## Compliance Requirements

- **OSHA**: Safety requirements tracked at each hierarchy level
- **EPA**: Environmental permits tracked at plant level
- **GAMP 5**: GMP-critical classification at machine level
- **SOX**: Complete audit trail for hierarchy changes
