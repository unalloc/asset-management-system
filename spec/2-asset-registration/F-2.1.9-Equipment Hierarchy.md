## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Hierarchical asset structure per Clause 6.4.3 |
| **ISO 55001 Physical Asset Management** | Yes | Functional location structure aligned |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | ISO 55002 supports functional hierarchies |

**ISO 55000 Alignment Explanation:**

Equipment hierarchy directly supports ISO 55000's asset management structure by providing:
- Functional location hierarchy for manufacturing assets
- Top-down decomposition from plant to machine level
- Metrics aggregation between hierarchy levels
- Impact analysis for maintenance activities

**Alignment Gaps:**
- No explicit support for ISO 22400-2 OEE hierarchy
- Missing integration with ISO 14001 environmental aspects

### 2.2 US/EU Regulatory Compliance

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Plant and organizational structure defined | `PlantExists(PlantID) == TRUE` | VAL-001 | "Plant does not exist" |
| PRE-002 | User has "Equipment Hierarchy Management" permission | `PermissionCheck(user, 'EQUIPMENT_HIERARCHY') == TRUE` | AUTH-001 | "User does not have Equipment Hierarchy Management permission" |
| PRE-003 | Node type is valid | `NodeType IN ('PLANT', 'PRODUCTION_LINE', 'WORK_CENTER', 'MACHINE')` | VAL-002 | "Invalid hierarchy type" |
| PRE-004 | Parent exists if not top-level | `ParentID IS NULL OR NodeExists(ParentID) == TRUE` | VAL-003 | "Parent node does not exist" |

**Pre-condition Validation Matrix:**

```
IF PlantExists(PlantID)
   AND UserHasPermission('EQUIPMENT_HIERARCHY')
   AND NodeTypeIsValid(Type)
   AND (ParentID IS NULL OR NodeExists(ParentID))
THEN proceed to node creation
ELSE throw appropriate error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Equipment Node Creation
  │
  ▼
Validate Plant Exists ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Permission ──► [No] ──► AUTH-001 Error
  │
  ▼ [Yes]
Validate Node Type ──► [Invalid] ──► VAL-002 Error
  │
  ▼ [Valid]
Validate Parent Node ──► [Not Found] ──► VAL-003 Error
  │
  ▼ [Found or Top-Level]
Validate Node Code Unique ──► [Duplicate] ──► VAL-004 Error
  │
  ▼ [Unique]
Validate Hierarchy Depth ──► [Exceeds 4] ──► VAL-005 Error
  │
  ▼ [Valid]
Create Node Record
  │
  ▼
Update Hierarchy Path
  │
  ▼
Log Node Creation
  │
  ▼
Return Success
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Strict Hierarchy | Only 4 levels allowed | Reject if exceeded | 1 |
| BR-002 | Level Validation | NodeType must match expected level | Reject if mismatched | 1 |
| BR-003 | Code Unique per Parent | NodeCode unique within parent | Reject if duplicate | 1 |
| BR-004 | Parent Same Plant | All nodes under same plant | Reject if different | 1 |
| BR-005 | Metrics Aggregation | Metrics roll up to parent | Calculate on child change | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Delete node with children | Warn, block or cascade | Require children handling |
| EC-002 | Move subtree to new parent | Validate new hierarchy | Reject if cycle |
| EC-003 | Change node type | Allow with warning | WRN-001 |

### 3.5 State Transitions

| Current State | Event | Next State | Side Effects |
|---------------|-------|------------|---------------|
| DRAFT | Save | DRAFT | - |
| DRAFT | Activate | ACTIVE | Timestamp |
| ACTIVE | Decommission | INACTIVE | Children inherit or block |
| ACTIVE | Relocate | ACTIVE | Parent change |

## 6. Enhanced Validation Rules

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |
|------------|-----------|--------|-------|----------|-----------------|------------|---------------|
| NodeType | ENUM | N/A | PLANT/LINE/WORK_CENTER/MACHINE | YES | Value in list | VAL-002 | "Invalid hierarchy type" |
| NodeCode | VARCHAR | [A-Z0-9_]{1,30} | Uppercase | YES | Pattern + uniqueness | VAL-004 | "Code already exists or invalid" |
| NodeName | VARCHAR | N/A | 1-100 chars | YES | Length check | VAL-006 | "Node name required" |
| ParentNodeID | UUID | N/A | Valid UUID or NULL | Conditional | If not top-level: must exist | VAL-003 | "Parent node does not exist" |
| PlantID | UUID | N/A | Valid UUID | YES | Exists check | VAL-001 | "Plant does not exist" |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | Max depth | Level <= 4 | Reject | VAL-005 |
| BRV-002 | Code unique | Code unique within parent | Reject | VAL-004 |
| BRV-003 | Parent type valid | Parent type is one level above | Reject | VAL-007 |

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|
| VAL-001 | Error | Validation | Plant not found | Plant does not exist | Select valid plant |
| VAL-002 | Error | Validation | Invalid type | Unsupported hierarchy type | Use PLANT, LINE, WORK_CENTER, MACHINE |
| VAL-003 | Error | Validation | Parent not found | Parent node does not exist | Select valid parent |
| VAL-004 | Error | Business Rule | Duplicate code | Node code already exists | Choose unique code |
| VAL-005 | Error | Business Rule | Max depth | Hierarchy structure does not conform | Max 4 levels |
| VAL-006 | Error | Validation | Name required | Node name required | Enter node name |
| VAL-007 | Error | Validation | Type mismatch | Invalid parent-child type combination | Check hierarchy rules |
