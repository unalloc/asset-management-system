# F-2.1.6 - Asset Hierarchy

**Requirement ID:** F-2.1.6  
**Module:** Asset Registration & Cataloging (2.1)  
**Priority:** Medium

---

## Requirement Description

Support parent-child hierarchical relationships between assets for managing components and assemblies. The hierarchy enables organizations to track multi-level asset structures, component relationships, and support maintenance planning at both individual and system levels.

---

## Business Rules

1. Each asset can have only one parent asset (single-parent hierarchy)
2. Circular dependencies are not allowed in the hierarchy
3. Relationship types include COMPONENT, ACCESSORY, CONTAINER, and SPARES
4. Child assets can be reassigned to different parents
5. Deleting a parent asset requires handling of child assets
6. All hierarchy changes are logged for audit purposes
7. Hierarchy depth is configurable with warning thresholds

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Parent Asset ID | Yes | Asset that will be the parent |
| Child Asset ID | Yes | Asset that will be the child |
| Relationship Type | Yes | Type of relationship (COMPONENT, ACCESSORY, etc.) |
| Relationship Attributes | No | Additional metadata for the relationship |

---

## Output Data

| Data | Description |
|------|-------------|
| Relationship Record | Created relationship between parent and child |
| Hierarchy History | Record of all changes to the hierarchy |
| Audit Log | Complete trail of hierarchy modifications |

---

## Validation Rules

1. Both parent and child assets must exist
2. An asset cannot be its own parent
3. Relationship type must be valid (COMPONENT, ACCESSORY, CONTAINER, SPARES)
4. Child must not already have a different parent (unless reassignment allowed)
5. Creating the relationship must not result in circular dependency
6. Parent and child assets must be in the same company

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full hierarchy management across all companies |
| Company Admin | Create, modify, delete relationships within own company |
| Plant Manager | Manage relationships within plant scope |
| Department User | View hierarchy (read-only) |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Hierarchy relationships are scoped within a company
- **Isolation**: Cross-company hierarchy relationships are not allowed

---

## Compliance Requirements

- **SOX**: Hierarchy changes tracked for asset ownership and control audit
- **US GAAP/IFRS**: Component hierarchy supports IAS 16/ASC 360 component depreciation accounting
