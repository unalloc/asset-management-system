## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Hierarchy supports asset portfolio organization (Clause 6.4) |
| **ISO 55001 Physical Asset Management** | Yes | Component relationships per Clause 6.4.3.2 |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Hierarchy documentation aligned |

**ISO 55000 Alignment Explanation:**

Asset hierarchy directly supports ISO 55000's asset management system requirements by providing:
- Logical grouping of related assets (Clause 6.4.3)
- Support for system-of-systems asset decomposition
- Enable parent-child financial and operational tracking
- Foundation for spares determination and maintenance planning

**Alignment Gaps:**
- No explicit FMEA (Failure Mode and Effects Analysis) linkage in hierarchy
- Missing support for RCM (Reliability-Centered Maintenance) dependency modeling

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Related assets are registered | `AssetExists(ParentID) AND AssetExists(ChildID)` | VAL-001 | "One or more assets do not exist" |
| PRE-002 | User has "Relationship Management" permission | `PermissionCheck(user, 'RELATIONSHIP_MANAGEMENT') == TRUE` | AUTH-001 | "User does not have Relationship Management permission" |
| PRE-003 | Parent and child are different | `ParentID != ChildID` | VAL-002 | "Asset cannot be its own parent" |
| PRE-004 | Relationship type is valid | `RelationshipType IN ('COMPONENT', 'ACCESSORY', 'CONTAINER', 'SPARES')` | VAL-003 | "Invalid relationship type" |
| PRE-005 | Child not already has parent | `ChildAsset.ParentAssetID IS NULL OR ChildAsset.ParentAssetID = ParentID` | VAL-004 | "Child asset already has a parent" |

**Pre-condition Validation Matrix:**

```
IF AssetExists(ParentID) AND AssetExists(ChildID)
   AND UserHasPermission('RELATIONSHIP_MANAGEMENT')
   AND ParentID != ChildID
   AND RelationshipTypeIsValid(Type)
   AND ChildHasNoOtherParent(ChildID, ParentID)
THEN proceed to hierarchy creation
ELSE throw appropriate error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Hierarchy relationship established | AssetRelationship record created | Database verification |
| POST-002 | Relationship change recorded | AuditLog entry created | Audit log query |
| POST-003 | Hierarchy structure updated | Parent-child link queryable | Hierarchy query |

**Post-condition Verification:**
- AssetRelationship record exists with correct parent-child link
- AuditLog entry created for relationship creation
- Hierarchy can be traversed correctly

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Hierarchy Relationship Request
  │
  ▼
Validate Parent Asset ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Child Asset ──► [Not Found] ──► VAL-001 Error
  │
  ▼ [Found]
Validate Not Self-Reference ──► [Same] ──► VAL-002 Error
  │
  ▼ [Different]
Validate Relationship Type ──► [Invalid] ──► VAL-003 Error
  │
  ▼ [Valid]
Check Child Availability ──► [Has Parent] ──► VAL-004 Error
  │
  ▼ [Available]
Check for Circular Dependency ──► [Would Cycle] ──► VAL-005 Error
  │
  ▼ [No Cycle]
Create Relationship Record
  │
  ▼
Update Child Asset Parent Reference
  │
  ▼
Log Relationship Change
  │
  ▼
Publish AssetHierarchyChanged Event
  │
  ▼
Return Success Response
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | No Self-Reference | ParentID != ChildID | Reject | 1 |
| BR-002 | No Circular Dependency | Parent chain cannot loop back | Reject | 1 |
| BR-003 | Child Single Parent | Each child has only one parent | Reject if has other parent | 1 |
| BR-004 | Multi-Level Support | Unlimited hierarchy depth (configurable) | Allow | 2 |
| BR-005 | Virtual Nodes | Support grouping assets without physical parent | Allow virtual parent | 2 |
| BR-006 | Relationship Types | Support COMPONENT/ACCESSORY/CONTAINER/SPARES | Validate type | 1 |
| BR-007 | Hierarchy Change Audit | All changes logged | Create audit record | 1 |

**Calculation Logic (if applicable):**
N/A - No calculations required

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Re-parenting child | Remove old parent, assign new | Delete old, create new |
| EC-002 | Delete parent with children | Warn, require child handling | Option: cascade delete or reassign |
| EC-003 | Deep hierarchy (>10 levels) | Warn or reject based on config | Configuration option |
| EC-004 | Move subtree to new parent | Validate entire chain for cycles | Reject if cycle would result |
| EC-005 | Parent is child of child | Circular dependency detected | Reject VAL-005 |
| EC-006 | Component replacement | Keep history, update component link | Create replacement record |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| ParentExists=TRUE | ChildExists=TRUE | NoCycle=TRUE | Create |
| ParentExists=TRUE | ChildExists=TRUE | Cycle=TRUE | Reject VAL-005 |
| ChildHasParent=TRUE | ReassignAllowed=TRUE | - | Remove old, create new |
| ChildHasParent=TRUE | ReassignAllowed=FALSE | - | Reject VAL-004 |

### 3.5 State Transitions

**Relationship State Diagram:**

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│  PENDING    │──────►│   ACTIVE    │──────►│  TERMINATED │
└─────────────┘       └─────────────┘       └─────────────┘
                            │                      │
                            ▼                      │
                     ┌─────────────┐               │
                     │  INACTIVE   │◄──────────────┘
                     └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| PENDING | Approve | ACTIVE | Approval given | Timestamp update |
| PENDING | Reject | TERMINATED | Rejection reason | No link created |
| ACTIVE | Replace | INACTIVE | New component assigned | History preserved |
| ACTIVE | Remove | TERMINATED | User confirmation | Link removed |
| INACTIVE | Reactivate | ACTIVE | Re-association | History preserved |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-001 | No self-reference | ParentAssetID != ChildAssetID | Reject | VAL-002 |

### 6.3 Error Codes

| Error Code | Severity | Category | Short Description | Long Description | User Guidance |
|------------|----------|----------|-------------------|------------------|---------------|

| VAL-002 | Error | Validation | Self reference | Asset cannot be its own parent | Select different parent |
| VAL-003 | Error | Validation | Invalid type | Invalid relationship type | Use COMPONENT, ACCESSORY, CONTAINER, or SPARES |
| VAL-004 | Error | Business Rule | Has parent | Child asset already has a parent | Remove existing relationship first |
| VAL-005 | Error | Business Rule | Circular | Circular relationship detected | Restructure the hierarchy |
| VAL-006 | Error | Validation | Invalid attributes | Relationship attributes invalid | Check JSON format |
| VAL-007 | Error | Business Rule | Cross-company | Assets must be in same company | Use assets from same company |
