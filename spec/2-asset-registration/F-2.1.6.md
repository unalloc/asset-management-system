# F-2.1.6 - Asset Hierarchy

## Requirement ID
F-2.1.6

## Function Name
Asset Hierarchy

## Function Description
Support parent-child hierarchical relationships between assets for managing components and assemblies.

## Business Rules

1. Support multi-level hierarchy
2. Child assets can be managed independently
3. Hierarchy changes require audit
4. Support virtual asset nodes

## Preconditions

- Related assets are registered
- User has "Relationship Management" permission

## Postconditions

- Asset hierarchy relationship established
- Relationship change recorded

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Parent Asset ID | Yes | Parent asset |
| Child Asset ID List | Yes | Child assets |
| Relationship Type | No | Component/Accessory/etc |
| Relationship Attributes | No | Additional attributes |

## Output Data

1. Relationship establishment confirmation
2. Hierarchy structure view
3. Change record ID

## Validation Rules

1. Cannot create circular dependencies
2. Assets must exist
3. Relationship type must be valid
4. Child asset cannot belong to multiple parents

## Error Handling

| Error | Message |
|-------|---------|
| Circular dependency | "Circular relationship detected" |
| Asset not found | "Asset ID does not exist" |
| Conflict relationship | "Asset already has a parent asset" |

## Priority
Medium
