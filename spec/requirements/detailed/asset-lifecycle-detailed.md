# 详细需求规格 - 资产生命周期管理 (2.6)

## 概述
本模块负责资产从采购到处置的全生命周期管理，包括采购工作流集成、资产接收检验、资本化过程、部署调试、借用归还、库存盘点、资产处置、保修索赔、租赁管理等功能，实现资产从"摇篮到坟墓"的全程跟踪。

## 功能详细规格

### 2.6.1 采购工作流集成
**功能ID**: F-2.6.1  
**功能名称**: 采购流程集成  
**功能描述**: 系统应与采购系统集成，支持从采购申请到采购订单、收货、资产创建的完整工作流。

**业务规则**:
1. 支持与主流ERP系统（SAP, Oracle, 用友, 金蝶）的采购模块集成
2. 采购申请可自动转换为资产采购需求
3. 采购订单收货后自动触发资产接收流程
4. 采购价格自动作为资产原值基础
5. 支持采购合同与资产关联管理
6. 多供应商比价和采购历史跟踪

**前置条件**:
- 采购系统接口已配置
- 用户具有"采购集成"权限
- 资产分类体系已定义

**后置条件**:
- 采购数据同步到资产系统
- 资产记录与采购订单关联
- 财务信息自动传递

**输入数据**:
1. 采购申请ID
2. 采购订单号
3. 供应商信息
4. 采购物品描述
5. 采购数量
6. 单价和总价
7. 预计交货日期
8. 采购合同条款

**输出数据**:
1. 资产采购需求记录
2. 采购-资产关联映射
3. 同步状态报告
4. 异常处理日志

**验证规则**:
1. 采购订单号必须唯一
2. 供应商必须存在于供应商主数据
3. 采购物品必须可映射到资产类型
4. 价格必须为正数
5. 交货日期不能早于当前日期

**错误处理**:
1. 接口连接失败：重试机制，失败通知
2. 数据映射失败：人工干预流程
3. 重复采购订单：返回警告并建议合并
4. 必填字段缺失：拒绝同步并返回错误详情

**优先级**: 高  
**相关用户故事**: US-2.6.1, US-2.6.2

---

### 2.6.2 资产接收和检验
**功能ID**: F-2.6.2  
**功能名称**: 资产接收检验  
**功能描述**: 提供资产接收、质量检验、验收确认的标准流程，确保接收资产符合采购规格和质量要求。

**业务规则**:
1. 支持批量接收和单个接收两种模式
2. 接收时必须进行基本质量检查（外观、数量、型号）
3. 支持拒收、部分接收、有条件接收
4. 检验结果决定资产状态（合格、待修、退货）
5. 检验人员需有相应资质认证
6. 检验记录必须保存并关联资产档案
7. 支持附件上传（检验报告、照片、证书）

**前置条件**:
- 采购订单已存在且未完全收货
- 用户具有"资产接收"权限
- 检验标准已配置

**后置条件**:
- 资产接收状态更新
- 检验记录保存
- 供应商绩效数据更新

**输入数据**:
1. 采购订单号
2. 实际到货数量
3. 检验结果（合格/不合格）
4. 检验人员
5. 检验日期
6. 检验备注
7. 附件文件
8. 拒收原因（如适用）

**输出数据**:
1. 资产接收记录
2. 检验报告
3. 供应商绩效评分更新
4. 采购订单收货状态更新
5. 不合格品处理建议

**验证规则**:
1. 到货数量不能超过采购订单未收货数量
2. 检验日期不能早于采购订单日期
3. 检验人员必须具有有效资质
4. 不合格资产必须填写原因
5. 必填字段完整

**错误处理**:
1. 采购订单不存在：返回错误"采购订单无效"
2. 数量超限：提示最大可接收数量
3. 资质无效：阻止检验提交并通知管理员
4. 附件过大：提示压缩或分拆

**优先级**: 高  
**相关用户故事**: US-2.6.3, US-2.6.4

### 2.6.3 资本化过程
**功能ID**: F-2.6.3  
**功能名称**: 在建工程转固定资产  
**功能描述**: 管理在建工程（CIP）到固定资产的转换过程，包括成本累计、资本化条件判断、会计凭证生成等。

**业务规则**:
1. 支持自动和手动资本化触发
2. 资本化条件：资产达到预定可使用状态
3. 资本化金额包括所有相关成本（采购价、运输费、安装费、调试费等）
4. 自动生成会计凭证（借：固定资产，贷：在建工程）
5. 支持部分资本化（分阶段投入使用）
6. 资本化日期决定折旧开始日期
7. 资本化审批工作流
8. 资本化差异分析报告

**前置条件**:
- 资产接收已完成
- 在建工程记录存在
- 用户具有"资本化"权限
- 会计期间未关闭

**后置条件**:
- 资产状态变更为"在用"
- 折旧计算开始
- 会计凭证生成
- 在建工程金额清零

**输入数据**:
1. 资产ID或CIP项目号
2. 资本化日期
3. 资本化金额
4. 资本化原因
5. 审批人员
6. 会计期间
7. 成本分配明细

**输出数据**:
1. 资本化记录
2. 会计凭证号
3. 资产状态更新
4. 折旧计划生成
5. 资本化完成报告

**验证规则**:
1. 资本化日期必须在当前会计期间内
2. 资本化金额不能超过CIP累计成本
3. 资产必须处于"接收完成"或"在建"状态
4. 会计期间必须开放
5. 审批流程必须完成

**错误处理**:
1. CIP成本不足：提示成本不足并建议调整
2. 会计期间关闭：返回错误"会计期间已关闭"
3. 审批未完成：阻止资本化操作
4. 折旧规则未配置：提示先配置折旧政策

**优先级**: 高  
**相关用户故事**: US-2.6.5, US-2.6.6

---

### 2.6.4 资产部署和调试
**功能ID**: F-2.6.4  
**功能名称**: 资产部署调试  
**功能描述**: 管理资产安装、调试、验收、投入使用的全过程，包括位置分配、责任人指定、调试记录等。

**业务规则**:
1. 支持部署计划制定和跟踪
2. 调试过程分阶段管理（安装、调试、试运行、正式运行）
3. 调试结果决定资产是否可投入使用
4. 部署涉及多部门协作（IT、设施、生产、安全）
5. 调试记录必须包括技术参数、性能测试结果
6. 支持移动端现场调试记录
7. 调试不合格资产转入维修或退货流程

**前置条件**:
- 资产已资本化或可直接投入使用
- 部署位置可用
- 调试人员具备相应资质
- 安全许可已获得

**后置条件**:
- 资产位置更新
- 责任人指定
- 资产状态变更为"在用"
- 维护计划激活

**输入数据**:
1. 资产ID
2. 部署位置（工厂、部门、具体位置）
3. 部署日期
4. 责任人
5. 调试阶段结果
6. 调试人员
7. 调试工具和设备信息
8. 性能测试数据
9. 安全检查结果

**输出数据**:
1. 部署记录
2. 调试报告
3. 资产位置和责任人更新
4. 维护计划激活通知
5. 投入使用确认

**验证规则**:
1. 部署位置必须有效且可用
2. 责任人必须为有效员工
3. 调试人员必须具备相应资质
4. 安全检查必须通过才能投入使用
5. 性能测试数据必须在允许范围内

**错误处理**:
1. 位置冲突：提示位置已被占用
2. 资质不符：阻止调试记录提交
3. 安全检查失败：转入整改流程
4. 性能不达标：建议维修或调整

**优先级**: 高  
**相关用户故事**: US-2.6.7, US-2.6.8

### 2.6.5 借用/归还系统
**功能ID**: F-2.6.5  
**功能名称**: 资产借用归还  
**功能描述**: 提供资产借用、归还、预留的全流程管理，支持临时资产分配和跟踪。

**业务规则**:
1. 借用需申请审批，根据资产价值和敏感度设置不同审批级别
2. 支持预定借用（提前预约）和即时借用
3. 借用期限可设置，超期自动提醒
4. 归还时需检查资产状况，记录损坏或丢失
5. 支持批量借用和归还
6. 借用历史完整记录，可追溯
7. 与员工门禁/身份系统集成，自动关联借用人
8. 支持移动端借用申请和审批

**前置条件**:
- 资产状态为"可用"或"在库"
- 借用人有有效员工账号
- 借用审批流程已定义
- 资产未处于维修或预留状态

**后置条件**:
- 资产状态更新为"借用中"
- 借用记录创建
- 审批记录保存
- 借用通知发送

**输入数据**:
1. 资产ID或资产列表
2. 借用人ID
3. 借用开始日期和时间
4. 预计归还日期和时间
5. 借用目的
6. 审批人ID（如需要）
7. 借用地点
8. 特殊要求

**输出数据**:
1. 借用申请单号
2. 审批状态
3. 借用确认
4. 借用提醒（开始前）
5. 归还提醒（到期前）
6. 借用历史记录

**验证规则**:
1. 借用开始时间不能早于当前时间
2. 借用结束时间必须晚于开始时间
3. 资产必须在指定地点可用
4. 借用人必须有有效借用权限
5. 借用期限不能超过最大允许期限

**错误处理**:
1. 资产不可用：提示可用替代资产或建议其他时间
2. 借用人无权限：拒绝申请并通知管理员
3. 审批人无效：提示选择有效审批人
4. 借用冲突：提示冲突资产和时间

**优先级**: 高  
**相关用户故事**: US-2.6.9, US-2.6.10

### 2.6.6 借用管理
**功能ID**: F-2.6.6  
**功能名称**: 临时资产借用管理  
**功能描述**: 管理临时资产借用、延长借用、提前归还、损坏赔偿等全流程，支持借用成本计算和绩效分析。

**业务规则**:
1. 借用可延长，需重新审批（如超过原定期限50%）
2. 提前归还可自动处理，释放资产
3. 借用损坏或丢失需启动赔偿流程
4. 借用成本可按时间、资产价值、部门预算计算
5. 借用绩效分析：借用频率、平均借用时长、准时归还率
6. 支持跨部门借用成本结算
7. 借用黑名单管理：多次违规借用者限制借用权限
8. 紧急借用流程：简化审批，事后补手续

**前置条件**:
- 借用申请已批准
- 资产已分配给借用人
- 借用成本计算规则已定义

**后置条件**:
- 借用状态更新
- 借用成本记录
- 借用绩效数据更新
- 资产可用性更新

**输入数据**:
1. 借用申请单号
2. 实际借用开始时间
3. 实际归还时间
4. 资产状况检查结果
5. 损坏描述（如适用）
6. 赔偿金额（如适用）
7. 延长借用申请（如需要）
8. 借用成本分配部门

**输出数据**:
1. 借用完成记录
2. 损坏赔偿单
3. 借用成本计算报告
4. 借用绩效指标
5. 借用黑名单更新
6. 资产状态更新通知

**验证规则**:
1. 实际借用时间必须在借用期限内（除非延长批准）
2. 损坏描述必须附照片或证据
3. 赔偿金额必须基于预定义赔偿标准
4. 借用成本必须分配到有效成本中心
5. 归还时资产必须经过状况检查

**错误处理**:
1. 借用记录不存在：返回错误"借用记录无效"
2. 损坏证据不足：要求补充证据或转入调查流程
3. 赔偿标准未定义：提示管理员配置标准
4. 成本中心无效：提示选择有效成本中心

**优先级**: 中  
**相关用户故事**: US-2.6.11, US-2.6.12

### 2.6.7 实物库存盘点
**功能ID**: F-2.6.7  
**功能名称**: 实物库存盘点  
**功能描述**: 支持定期或临时的实物资产盘点，包括盘点计划制定、盘点任务分配、移动端盘点、差异分析和调整处理。

**业务规则**:
1. 支持年度全面盘点和月度/季度循环盘点
2. 盘点计划可自定义：按地点、资产类型、部门等
3. 移动端盘点支持：扫描条码/RFID、拍照、备注
4. 盘点差异自动识别：盘盈、盘亏、位置错误
5. 差异处理工作流：调查、审批、调整
6. 盘点进度实时监控
7. 支持暂停和恢复盘点
8. 盘点报告自动生成

**前置条件**:
- 盘点计划已创建并批准
- 盘点人员已分配并培训
- 移动设备已配置盘点应用
- 资产标签（条码/RFID）可读

**后置条件**:
- 盘点结果记录
- 差异处理工作流启动
- 资产记录更新（如需要）
- 盘点报告生成

**输入数据**:
1. 盘点计划ID
2. 盘点区域/范围
3. 盘点人员ID
4. 盘点开始时间
5. 资产实际位置
6. 资产状况
7. 差异备注
8. 照片证据

**输出数据**:
1. 盘点任务分配
2. 实时盘点进度
3. 差异清单
4. 盘点完成报告
5. 调整建议
6. 审计跟踪记录

**验证规则**:
1. 盘点人员必须在授权人员列表中
2. 盘点时间必须在计划时间范围内
3. 资产必须属于盘点范围
4. 差异必须经过确认才能提交
5. 照片证据必须清晰可辨

**错误处理**:
1. 标签无法读取：手动输入资产编号并标记问题
2. 网络连接中断：离线模式支持，自动同步
3. 盘点范围错误：提示盘点人员确认范围
4. 人员无权限：拒绝盘点提交

**优先级**: 高  
**相关用户故事**: US-2.6.13, US-2.6.14

### 2.6.8 循环盘点计划
**功能ID**: F-2.6.8  
**功能名称**: 循环盘点计划  
**功能描述**: 支持基于ABC分类、风险等级或其他规则的循环盘点计划，实现持续库存准确性和差异预防。

**业务规则**:
1. ABC分类：A类资产（高价值）每月盘点，B类每季度，C类每半年
2. 风险调整：高风险资产（易盗、易损）增加盘点频率
3. 历史差异调整：过去有差异的资产增加盘点频率
4. 盘点计划自动生成，可手动调整
5. 支持例外规则：特定资产可排除或特别处理
6. 盘点结果影响未来盘点频率（差异减少则降低频率）
7. 移动端提醒：盘点任务推送
8. 盘点绩效分析：盘点准确率、完成率、差异趋势

**前置条件**:
- 资产ABC分类已定义
- 风险等级已评估
- 盘点历史数据存在
- 盘点人员可用

**后置条件**:
- 循环盘点计划生成
- 盘点任务分配
- 盘点频率优化
- 绩效报告更新

**输入数据**:
1. ABC分类规则（价值阈值）
2. 风险等级定义
3. 基础盘点频率设置
4. 例外资产列表
5. 历史差异权重
6. 盘点人员容量
7. 季节性调整因素

**输出数据**:
1. 循环盘点计划表
2. 月度/季度盘点任务
3. 盘点频率优化建议
4. 绩效分析报告
5. 差异趋势图表
6. 审计合规报告

**验证规则**:
1. ABC分类必须覆盖所有资产
2. 盘点频率必须合理（不低于法规要求）
3. 盘点人员工作量必须均衡
4. 例外规则必须有正当理由
5. 历史数据必须完整可靠

**错误处理**:
1. 分类规则冲突：提示调整规则优先级
2. 盘点容量不足：建议调整频率或增加人员
3. 历史数据缺失：使用默认频率并标记
4. 规则逻辑错误：验证规则并提示修正

**优先级**: 中  
**相关用户故事**: US-2.6.15, US-2.6.16

### 2.6.9 资产处置工作流
**功能ID**: F-2.6.9  
**功能名称**: 资产处置流程

**验证规则**:
1. 处置方式必须适用于资产类型

**错误处理**:
1. 资产不可处置：返回原因（如仍在保修期、有未结工单）
2. 审批人无效：提示选择有效审批人
3. 合规检查失败：提示整改要求

4. ROI 分析：基于资产产生的效益（生产产出、服务收入）

6. 对标分析：同类资产成本对比、行业基准比较
7. 预测模型：基于历史数据预测未来生命周期成本
8. 决策支持：采购决策、更换决策、外包决策

**前置条件**:
- 资产成本数据可获取
- 成本分类规则已定义
- 历史数据完整

**后置条件**:
- 生命周期成本报告生成
- TCO/ROI 计算完成
- 成本趋势分析可用
- 决策建议生成

**输入数据**:
1. 资产ID
2. 采购成本数据
3. 运营成本（能源、耗材、人工）
4. 维护成本（预防性、纠正性）
5. 处置成本/收入
6. 资产产出/效益数据
7. 对标数据
8. 预测参数

**输出数据**:
1. 生命周期成本报告
2. TCO 分析
3. ROI 计算
4. 成本趋势图表
5. 对标分析结果
6. 预测模型输出
7. 决策建议

**验证规则**:
1. 成本数据必须完整准确
2. 成本分类必须一致
3. 效益数据必须可量化
4. 对标数据必须来源可靠
5. 预测模型必须经过验证

**错误处理**:
1. 数据缺失：使用估算值并标记不确定性
2. 成本分类错误：重新分类或调整规则
3. 效益难以量化：使用替代指标或定性分析
4. 模型误差：人工调整或使用替代模型

**优先级**: 高  
**相关用户故事**: US-2.6.31, US-2.6.32

### 2.6.17 环境处置合规
**功能ID**: F-2.6.17  
**功能名称**: 环保合规处置  
**功能描述**: 确保资产处置符合环保法规，特别是危险物质、电子废物、回收要求的合规管理。

**业务规则**:
1. 物质清单：识别资产中的危险物质（铅、汞、镉、六价铬等）
2. 法规库：存储各地环保法规（中国 RoHS, WEEE, EU RoHS, REACH）
3. 合规检查：处置前自动检查合规要求
4. 处理商认证：仅使用认证处理商，验证资质
5. 处理证明：要求并归档处理证明文件
6. 报告生成：合规报告、物质流向报告
7. 审计准备：支持环保审计，提供证据链
8. 违规预警：法规变更预警、处理商资质到期预警

**前置条件**:
- 资产包含危险物质或受管制材料
- 环保法规数据库最新
- 认证处理商名录可用
- 用户具有"环保合规"权限

**后置条件**:
- 合规检查通过
- 处理证明归档
- 合规报告生成
- 审计跟踪记录

**输入数据**:
1. 资产ID
2. 物质清单
3. 处置地点（国家/地区）
4. 处置方式
5. 处理商信息
6. 法规要求
7. 处理证明文件
8. 审计要求

**输出数据**:
1. 合规检查报告
2. 处理商资质验证结果
3. 处理证明记录
4. 环保合规报告
5. 物质流向报告
6. 审计证据包

**验证规则**:
1. 危险物质必须识别完整
2. 处理商必须具有相应资质
3. 处理证明必须有效（日期、签名、内容）
4. 法规引用必须准确
5. 报告必须满足监管要求

**错误处理**:
1. 物质清单不全：暂停处置，补充信息
2. 处理商资质失效：选择替代处理商
3. 证明文件不合格：要求重新提供
4. 法规冲突：法律部门介入解释

**优先级**: 高  
**相关用户故事**: US-2.6.33, US-2.6.34

### 2.6.18 资产残值实现
**功能ID**: F-2.6.18  
**功能名称**: 残值实现跟踪  
**功能描述**: 跟踪资产处置后残值的实际实现情况，包括销售收入、回收收入、节省的成本，与预期残值对比分析。

**业务规则**:
1. 预期残值：基于折旧模型、市场数据预测的残值
2. 实际残值：处置实际获得的收入或节省的成本
3. 差异分析：比较预期 vs 实际，分析原因
4. 残值优化：基于历史数据优化未来残值预测模型
5. 收入确认：处置收入会计处理、税务影响
6. 成本节省：通过回收、再利用节省的采购成本
7. 绩效指标：残值实现率（实际/预期）
8. 持续改进：残值实现经验反馈到采购和处置决策

**前置条件**:
- 资产处置完成
- 处置收入数据可用
- 预期残值记录存在
- 财务处理完成

**后置条件**:
- 残值实现记录保存
- 差异分析完成
- 预测模型更新
- 绩效指标计算

**输入数据**:
1. 资产ID
2. 处置方式
3. 预期残值
4. 实际收入/节省
5. 处置成本
6. 市场条件
7. 处置时间
8. 买方/处理商信息

**输出数据**:
1. 残值实现报告
2. 差异分析
3. 绩效指标
4. 预测模型调整建议
5. 会计凭证引用
6. 经验教训记录

**验证规则**:
1. 实际收入必须可验证（合同、发票、银行记录）
2. 预期残值必须基于合理假设
3. 成本节省必须合理量化
4. 差异分析必须客观公正
5. 绩效指标计算必须一致

**错误处理**:
1. 收入数据不完整：估算并标记为不完整数据
2. 预期残值缺失：使用同类资产平均值
3. 成本节省难以量化：定性描述为主
4. 差异原因不明：进一步调查或标记为未知

**优先级**: 中  
**相关用户故事**: US-2.6.35, US-2.6.36
