# 详细需求规格 - 国内合规与本地化 (2.9)

## 概述
本模块支持国内合规要求和本地化功能，包括财政部固定资产卡片格式、中国会计折旧方法、税会差异调整、加速折旧备案管理、国有资产管理合规、增值税发票系统集成、金税系统集成、银企直连集成、国内审计要求、中文界面本地化、微信小程序移动盘点、国内档案管理标准、国内HR和社保系统集成、中国数据安全法合规、国内商业实践支持等功能，满足跨国企业在中国运营的合规需求。

## 功能详细规格

### 2.9.1 财政部固定资产卡片格式
**功能ID**: F-2.9.1  
**功能名称**: 财政部固定资产卡片管理  
**功能描述**: 支持符合中国财政部《固定资产管理办法》要求的固定资产卡片格式，包括卡片设计、字段映射、打印输出和电子存档。

**业务规则**:
1. 卡片格式：符合财政部《固定资产卡片（样式）》标准格式
2. 必填字段：资产编号、资产名称、规格型号、计量单位、数量、原值、使用部门、使用人、存放地点、启用日期
3. 可选字段：生产厂家、出厂编号、使用年限、预计净残值率、折旧方法、累计折旧、净值
4. 卡片打印：支持标准A4纸打印，支持套打模式
5. 电子卡片：支持PDF格式导出，带数字签名
6. 历史记录：卡片信息变更历史跟踪
7. 多语言支持：卡片标题支持中英文对照
8. 自定义字段：允许添加企业自定义字段，不影响标准格式

**前置条件**:
- 资产基本信息已录入系统
- 财政部卡片格式模板可用
- 用户具有"固定资产卡片管理"权限
- 打印设备配置完成

**后置条件**:
- 固定资产卡片记录创建
- 卡片格式验证通过
- 打印/导出功能可用
- 电子存档完成

**输入数据**:
1. 资产编号
2. 资产名称
3. 规格型号
4. 计量单位
5. 数量
6. 原值（人民币）
7. 使用部门
8. 使用人
9. 存放地点
10. 启用日期
11. 生产厂家（可选）
12. 出厂编号（可选）
13. 使用年限（可选）
14. 预计净残值率（可选）
15. 折旧方法（可选）
16. 自定义字段（可选）

**输出数据**:
1. 固定资产卡片记录
2. 卡片打印文件（PDF/纸质）
3. 卡片格式验证报告
4. 电子存档记录
5. 变更历史记录

**验证规则**:
1. 资产编号必须唯一
2. 资产名称不能为空
3. 原值必须大于0
4. 启用日期不能晚于当前日期
5. 使用部门必须存在于组织架构中
6. 计量单位必须为标准单位

**错误处理**:
1. 格式错误：提示不符合财政部标准格式的具体字段
2. 数据缺失：提示必填字段未填写
3. 打印失败：提示打印设备状态或配置问题
4. 保存失败：提示数据库连接或权限问题

**优先级**: 高  
**相关用户故事**: US-070

### 2.9.2 中国会计折旧方法
**功能ID**: F-2.9.2  
**功能名称**: 中国会计折旧方法管理  
**功能描述**: 支持中国会计准则规定的折旧方法，包括直线法、双倍余额递减法、年数总和法，支持折旧计算、调整和追溯。

**业务规则**:
1. 折旧方法：支持直线法、双倍余额递减法、年数总和法
2. 计算方法：每月自动计算折旧，支持手工调整
3. 折旧期间：从启用日期次月开始计提，至资产处置或提足折旧
4. 净残值：支持预计净残值率或固定净残值额
5. 折旧调整：支持折旧政策变更的追溯调整
6. 折旧暂停：支持资产停用期间的折旧暂停
7. 加速折旧：支持符合税收优惠的加速折旧计算
8. 折旧报告：生成月度折旧计提表、折旧明细表

**前置条件**:
- 资产原值、使用年限、净残值信息完整
- 折旧方法政策定义明确
- 会计期间配置正确
- 用户具有"折旧计算"权限

**后置条件**:
- 月度折旧自动计算完成
- 折旧凭证生成
- 折旧报告可查询
- 资产净值更新

**输入数据**:
1. 资产ID
2. 折旧方法（直线法、双倍余额递减法、年数总和法）
3. 使用年限（月）
4. 预计净残值率（%）
5. 启用日期
6. 折旧政策变更日期（如适用）
7. 加速折旧标识（如适用）
8. 停用期间（如适用）

**输出数据**:
1. 月度折旧计算表
2. 折旧计提凭证
3. 资产折旧明细表
4. 折旧政策变更记录
5. 折旧调整分录

**验证规则**:
1. 使用年限必须大于0
2. 预计净残值率必须在0-100%之间
3. 启用日期必须有效
4. 折旧方法必须为三种之一
5. 资产状态必须为"在用"或"停用"

**错误处理**:
1. 计算错误：提示折旧计算参数错误
2. 期间错误：提示会计期间未打开或已关闭
3. 凭证错误：提示凭证生成失败原因
4. 追溯错误：提示追溯调整超出允许范围

**优先级**: 高  
**相关用户故事**: US-070

### 2.9.3 税会差异调整和管理
**功能ID**: F-2.9.3  
**功能名称**: 税会差异调整管理  
**功能描述**: 管理会计折旧与税务折旧之间的差异，支持差异计算、调整、纳税申报和差异台账管理。

**业务规则**:
1. 差异类型：折旧年限差异、折旧方法差异、净残值差异、减值准备差异
2. 差异计算：自动计算会计折旧与税务折旧差异
3. 纳税调整：生成应纳税所得额调整表
4. 差异台账：维护税会差异台账，跟踪差异变化
5. 递延所得税：计算递延所得税资产/负债
6. 申报支持：生成企业所得税申报附表
7. 差异分析：提供税会差异趋势分析
8. 政策变更：支持税收政策变更的差异重算

**前置条件**:
- 会计折旧数据完整
- 税务折旧政策定义明确
- 纳税期间配置正确
- 用户具有"税会差异管理"权限

**后置条件**:
- 税会差异计算完成
- 纳税调整表生成
- 差异台账更新
- 递延所得税计算完成

**输入数据**:
1. 资产ID
2. 会计折旧数据（方法、年限、净残值）
3. 税务折旧数据（方法、年限、净残值）
4. 纳税期间
5. 税率信息
6. 前期差异余额
7. 政策变更信息（如适用）

**输出数据**:
1. 税会差异计算表
2. 应纳税所得额调整表
3. 税会差异台账
4. 递延所得税计算表
5. 企业所得税申报附表
6. 差异分析报告

**验证规则**:
1. 会计与税务数据必须完整
2. 纳税期间必须有效
3. 税率必须符合税法规定
4. 前期差异必须平衡
5. 政策变更必须有生效日期

**错误处理**:
1. 数据不一致：提示会计与税务数据不匹配
2. 计算错误：提示差异计算逻辑错误
3. 期间错误：提示纳税期间配置问题
4. 平衡错误：提示差异台账不平衡

**优先级**: 高  
**相关用户故事**: US-071

### 2.9.4 加速折旧备案管理
**功能ID**: F-2.9.4  
**功能名称**: 加速折旧备案管理  
**功能描述**: 管理加速折旧的备案申请、审批、执行和报告，支持税收优惠政策的合规利用。

**业务规则**:
1. 备案类型：固定资产加速折旧、技术进步设备加速折旧、特定行业设备加速折旧
2. 申请流程：线上申请→部门审核→税务备案→执行跟踪
3. 优惠条件：自动校验资产是否符合加速折旧条件
4. 备案文档：生成备案申请文档和附件
5. 执行监控：监控加速折旧执行情况，确保合规
6. 优惠统计：统计加速折旧带来的税收优惠金额
7. 到期提醒：提前提醒加速折旧到期时间
8. 变更管理：支持备案信息变更和重新备案

**前置条件**:
- 资产信息完整，符合加速折旧条件
- 税务备案流程定义明确
- 用户具有"加速折旧管理"权限
- 税收优惠政策库可用

**后置条件**:
- 加速折旧备案申请提交
- 备案状态跟踪
- 加速折旧执行
- 优惠金额统计完成

**输入数据**:
1. 资产ID列表
2. 加速折旧类型
3. 加速折旧比例/年限
4. 申请原因
5. 证明材料附件
6. 申请日期
7. 期望生效日期
8. 备案机关信息

**输出数据**:
1. 加速折旧备案申请表
2. 备案状态跟踪记录
3. 加速折旧执行计划
4. 税收优惠统计报告
5. 到期提醒通知
6. 备案变更记录

**验证规则**:
1. 资产必须符合加速折旧条件
2. 申请日期不能晚于当前日期
3. 证明材料必须上传
4. 加速折旧比例必须在税法允许范围内
5. 资产必须未处于加速折旧期间

**错误处理**:
1. 条件不符：提示资产不符合加速折旧条件的具体原因
2. 材料不全：提示缺失的证明材料
3. 申请冲突：提示与现有备案冲突
4. 审批失败：提示备案审批失败原因

**优先级**: 中  
**相关用户故事**: US-072

4. 处置审批流程必须合规

**功能名称**: 增值税发票系统集成  
**功能描述**: 与增值税发票系统集成，支持资产采购发票的自动获取、验证、匹配和税务处理。

4. 发票匹配：发票与采购订单、资产卡片自动匹配
5. 税务处理：进项税额计算、认证、抵扣
6. 发票归档：电子发票归档管理
7. 异常处理：发票异常（作废、红冲）的同步处理
8. 统计分析：发票数据分析报告

**前置条件**:
- 增值税发票系统接口可用
- 资产采购信息完整
- 用户具有"发票管理"权限
- 税务数字证书配置正确

**后置条件**:
- 发票数据同步完成
- 发票验证通过
- 进项税额计算完成
- 发票归档完成

**输入数据**:
1. 发票代码
2. 发票号码
3. 开票日期
4. 销售方信息
5. 购买方信息
6. 货物/服务信息
7. 金额信息（不含税金额、税额、合计金额）
8. 发票状态

**输出数据**:
1. 发票同步记录
2. 发票验证报告
3. 发票匹配结果

3. 销售方纳税人识别号必须有效
4. 税额计算必须准确
5. 发票状态必须正常（未作废、未红冲）

**错误处理**:
1. 接口错误：提示发票系统连接失败
2. 验证失败：提示发票验证失败原因
3. 匹配失败：提示发票与资产匹配失败
4. 税务错误：提示进项税额计算错误

**优先级**: 高

**功能ID**: F-2.9.8
