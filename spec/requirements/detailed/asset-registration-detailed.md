# 详细需求规格 - 资产注册与分类 (2.1)

## 概述
本模块负责资产的初始注册、分类、编码和信息管理，是资产管理系统的数据基础。

## 功能详细规格

### 2.1.1 支持多种资产类型
**功能ID**: F-2.1.1  
**功能名称**: 多资产类型支持  
**功能描述**: 系统应支持注册和管理多种类型的资产，包括但不限于IT设备、办公家具、车辆、生产机械等。

**业务规则**:
1. 资产类型采用分层分类结构
2. 每种资产类型有特定的属性模板
3. 资产类型可动态扩展
4. 类型变更需审计跟踪

**前置条件**:
- 用户具有"资产注册"权限
- 资产类型分类体系已配置

**后置条件**:
- 新资产成功注册到系统
- 资产类型信息记录完整

**输入数据**:
1. 资产类型代码（必填）
2. 资产类型名称（必填）
3. 资产类型描述（可选）
4. 父类型ID（用于层次结构）
5. 属性模板ID

**输出数据**:
1. 资产类型注册确认
2. 分配的唯一类型ID
3. 创建时间戳

**验证规则**:
1. 类型代码必须唯一
2. 类型名称不能为空
3. 如果指定父类型，必须存在且有效
4. 代码格式：字母数字，最大20字符

**错误处理**:
1. 重复类型代码：返回错误"类型代码已存在"
2. 无效父类型：返回错误"父类型不存在"
3. 必填字段缺失：返回具体字段错误

**优先级**: 高  
**相关用户故事**: US-001, US-002

---

### 2.1.2 自定义资产类别和子类别
**功能ID**: F-2.1.2  
**功能名称**: 自定义分类体系  
**功能描述**: 允许用户根据企业特定需求创建和管理自定义的资产类别和子类别。

**业务规则**:
1. 类别支持无限级层次结构
2. 子类别继承父类别的属性
3. 类别可启用/禁用
4. 类别变更需版本控制

**前置条件**:
- 用户具有"分类管理"权限
- 系统初始化完成

**后置条件**:
- 新类别添加到分类体系
- 分类关系正确建立

**输入数据**:
1. 类别名称（必填）
2. 类别代码（必填）
3. 父类别ID（可选）
4. 描述信息
5. 排序序号
6. 启用状态

**输出数据**:
1. 类别创建成功响应
2. 分配的类别ID
3. 完整类别路径

**验证规则**:
1. 类别代码在兄弟节点中唯一
2. 类别名称不能为空
3. 不能创建循环依赖
4. 代码格式：字母数字下划线，最大50字符

**错误处理**:
1. 循环依赖检测：返回错误"类别关系存在循环"
2. 代码重复：返回错误"类别代码在同级中已存在"
3. 无效父类别：返回错误"父类别不存在或已禁用"

**优先级**: 中  
**相关用户故事**: US-002

---

### 2.1.3 资产数据批量导入/导出
**功能ID**: F-2.1.3  
**功能名称**: 批量数据操作  
**功能描述**: 支持通过CSV、Excel、JSON格式批量导入和导出资产数据。

**业务规则**:
1. 导入支持增量更新和全量替换
2. 导出支持选择性字段
3. 导入前需数据验证
4. 支持导入模板下载

**前置条件**:
- 用户具有"批量操作"权限
- 文件格式符合要求

**后置条件**:
- 资产数据成功导入系统
- 导入结果报告生成

**输入数据**:
1. 数据文件（CSV/Excel/JSON）
2. 导入模式（新增/更新/替换）
3. 字段映射配置
4. 错误处理策略

**输出数据**:
1. 导入结果报告
2. 成功/失败记录统计
3. 错误详细信息
4. 处理日志文件

**验证规则**:
1. 文件大小限制：最大100MB
2. 文件格式验证
3. 必填字段检查
4. 数据格式校验（日期、数字等）
5. 业务规则验证

**错误处理**:
1. 文件格式错误：返回"不支持的文件格式"
2. 数据验证失败：记录具体行和错误
3. 系统错误：回滚事务，返回错误信息

**优先级**: 高  
**相关用户故事**: US-003

### 2.1.4 条码/二维码生成和打印
**功能ID**: F-2.1.4  
**功能名称**: 资产标识生成  
**功能描述**: 自动生成资产条码或二维码标签，并支持打印功能。

**业务规则**:
1. 条码编码规则可配置
2. 支持多种条码格式（Code128, QR等）
3. 标签模板可自定义
4. 打印历史记录

**前置条件**:
- 资产已注册
- 条码打印机已配置
- 用户具有"标签打印"权限

**后置条件**:
- 资产标签生成并关联
- 打印任务完成

**输入数据**:
1. 资产ID或资产列表
2. 条码格式选择
3. 标签模板选择
4. 打印份数

**输出数据**:
1. 条码图像文件
2. 打印任务ID
3. 打印状态

**验证规则**:
1. 资产必须存在且有效
2. 条码格式必须支持
3. 打印份数限制：1-100份

**错误处理**:
1. 打印机未连接：返回"打印机不可用"
2. 资产不存在：返回"资产不存在"
3. 条码生成失败：返回"条码生成错误"

**优先级**: 高  
**相关用户故事**: US-004

### 2.1.5 RFID标签集成支持
**功能ID**: F-2.1.5  
**功能名称**: RFID集成  
**功能描述**: 支持RFID标签的读写和集成，用于快速资产识别和跟踪。

**业务规则**:
1. 支持多种RFID频率（LF, HF, UHF）
2. RFID标签与资产一对一绑定
3. 标签状态管理（激活/停用/丢失）
4. 批量标签初始化

**前置条件**:
- RFID读写器已连接
- 标签硬件可用
- 用户具有"RFID管理"权限

**后置条件**:
- RFID标签与资产成功关联
- 标签数据写入完成

**输入数据**:
1. 资产ID
2. RFID标签UID
3. 标签类型
4. 写入数据（资产信息）

**输出数据**:
1. 标签写入结果
2. 关联确认
3. 标签状态

**验证规则**:
1. 标签UID必须有效且唯一
2. 资产必须存在
3. 标签类型必须支持

**错误处理**:
1. 读写器连接失败：返回"RFID设备未连接"
2. 标签读写失败：返回"标签操作失败"
3. 标签已关联：返回"标签已被其他资产使用"

**优先级**: 中  
**相关用户故事**: US-004

### 2.1.6 资产层次关系
**功能ID**: F-2.1.6  
**功能名称**: 资产层次结构  
**功能描述**: 支持资产之间的父子层次关系，用于管理组件和组装体。

**业务规则**:
1. 支持多级层次结构
2. 子资产可独立管理
3. 层次变更需审计
4. 支持虚拟资产节点

**前置条件**:
- 相关资产已注册
- 用户具有"关系管理"权限

**后置条件**:
- 资产层次关系建立
- 关系变更记录保存

**输入数据**:
1. 父资产ID
2. 子资产ID列表
3. 关系类型（组件、附属等）
4. 关系属性

**输出数据**:
1. 关系建立确认
2. 层次结构视图

**验证规则**:

### 2.1.13 关键性分类
**功能ID**: F-2.1.13  
**功能名称**: 资产关键性评级  
**功能描述**: 根据生产影响对资产进行A/B/C关键性分类。

**业务规则**:
1. A类：生产关键，故障导致停产
2. B类：生产重要，故障影响产能
3. C类：非关键，故障影响有限
4. 关键性动态评估

**前置条件**:
- 资产已注册
- 关键性评估标准定义
- 用户具有"关键性管理"权限

**后置条件**:
- 资产关键性评级完成
- 相关策略应用（维护、备件等）

**输入数据**:
1. 资产ID
2. 关键性等级（A/B/C）
3. 评估依据
4. 影响范围描述
5. 评估有效期

**输出数据**:
1. 关键性评级确认
2. 评级记录ID
3. 相关策略应用结果

**验证规则**:
1. 资产必须存在
2. 关键性等级必须有效
3. 评估依据不能为空

**错误处理**:
1. 无效等级：返回"关键性等级必须是A、B或C"
2. 资产不存在：返回"资产ID不存在"
3. 评估依据缺失：返回"必须提供评估依据"

**优先级**: 高（针对制造企业）  
**相关用户故事**: US-041, US-042
