# 详细需求规格 - 多公司集团管理 (2.8)

## 概述
本模块支持多公司集团架构下的资产管理，包括多层次组织层次结构、法人实体管理、数据隔离、集团合并报告、跨公司资产转移、公司间租赁、集团政策模板、集中采购、集团标准化等功能，满足跨国企业、控股集团、企业集团的复杂管理需求。

## 功能详细规格

### 2.8.1 多层次组织层次结构
**功能ID**: F-2.8.1  
**功能名称**: 集团组织架构管理  
**功能描述**: 管理多层次的集团组织架构，包括集团→控股公司→子公司→事业部→部门的多级结构，支持灵活的组织架构调整和映射。

**业务规则**:
1. 层次结构：支持无限级组织层次，典型结构：集团→控股公司→子公司（全资、控股、参股）→事业部→部门→团队
2. 组织类型：法人实体（公司）、非法人实体（部门、项目组）、虚拟组织
3. 组织属性：每个组织层级可配置独立属性（地理位置、行业、货币、语言）
4. 架构调整：组织架构变更（拆分、合并、重组）支持历史跟踪
5. 映射关系：组织架构与财务系统、HR系统、ERP系统组织架构映射
6. 可视化：组织架构图可视化展示，支持钻取和筛选
7. 权限继承：组织层级间的权限继承和覆盖机制
8. 合规要求：满足不同国家/地区对组织架构的法定要求

**前置条件**:
- 集团组织架构信息可用
- 组织类型定义明确
- 用户具有"组织架构管理"权限
- 集成系统接口可用（如需要）

**后置条件**:
- 组织架构数据录入系统
- 组织层级关系建立
- 权限继承规则生效
- 集成映射完成

**输入数据**:
1. 组织代码（唯一标识）
2. 组织名称（多语言）
3. 组织类型
4. 上级组织ID
5. 组织属性（地理位置、货币、语言、行业等）
6. 生效日期
7. 组织负责人
8. 集成映射信息

**输出数据**:
1. 组织架构记录
2. 组织层级关系
3. 组织架构图
4. 权限继承规则
5. 集成映射确认
6. 合规检查报告

**验证规则**:
1. 组织代码必须唯一
2. 组织名称不能为空
3. 上级组织必须存在（根组织除外）
4. 组织类型必须有效
5. 生效日期不能早于当前日期

**错误处理**:
1. 循环依赖检测：拒绝创建循环依赖的组织结构
2. 代码重复：提示修改组织代码
3. 无效上级：提示选择有效上级组织
4. 集成冲突：检测与外部系统组织架构冲突

**优先级**: 高  
**相关用户故事**: US-2.8.1, US-2.8.2

### 2.8.2 法人实体管理
**功能ID**: F-2.8.2  
**功能名称**: 法人实体管理  
**功能描述**: 管理集团内的法人实体（公司），包括公司注册信息、法律状态、税务信息、独立账簿设置等。

**业务规则**:
1. 公司类型：全资子公司、控股子公司、参股公司、合资公司、分公司
2. 注册信息：公司名称、注册地址、注册号、注册资本、法定代表人
3. 法律状态：营业状态（正常、停业、清算、注销）、营业期限
4. 税务信息：纳税人识别号、税务登记地、适用税种税率
5. 独立账簿：每个法人实体独立会计账簿，支持不同会计准则
6. 公司关系：股权关系（持股比例、投资日期）、控制关系
7. 合规要求：满足不同国家/地区的公司法律要求
8. 审计跟踪：公司信息变更审计跟踪

**前置条件**:
- 法人实体法律信息可用
- 会计准则定义明确
- 用户具有"法人实体管理"权限
- 税务规则库可用

**后置条件**:
- 法人实体记录创建
- 独立账簿设置
- 税务配置完成
- 股权关系记录

**输入数据**:
1. 公司代码（唯一标识）
2. 公司名称（法定名称、常用名称）
3. 公司类型
4. 注册信息（地址、注册号、资本、法人）
5. 法律状态
6. 税务信息
7. 会计准则（中国会计准则、国际会计准则等）
8. 股权关系数据

**输出数据**:
1. 法人实体记录
2. 独立账簿配置
3. 税务配置
4. 股权结构图
5. 合规检查报告
6. 审计跟踪记录

**验证规则**:
1. 公司代码必须唯一
2. 注册信息必须完整（根据公司类型）
3. 税务信息必须符合当地法规
4. 股权比例总和必须合理（如子公司股权总和≤100%）
5. 会计准则必须支持

**错误处理**:
1. 公司已存在：提示重复并建议合并或修改代码
2. 税务信息无效：提示税务验证失败
3. 会计准则不支持：提示选择支持的标准或申请扩展
4. 股权关系矛盾：检测股权循环或比例异常

**优先级**: 高  
**相关用户故事**: US-2.8.3, US-2.8.4

### 2.8.3 按公司数据隔离
**功能ID**: F-2.8.3  
**功能名称**: 公司数据隔离  
**功能描述**: 实现按公司的数据隔离，确保每个公司的资产数据、财务数据、运营数据独立且安全，同时支持集团级汇总视图。

**业务规则**:
1. 数据隔离级别：行级数据隔离，基于公司ID过滤
2. 权限控制：用户只能访问其所属公司（及下级公司）的数据
3. 集团视图：集团管理员可查看所有公司数据，支持数据汇总
4. 数据共享：支持公司间数据共享（需授权），如资产转移、租赁
5. 隔离策略：可配置的隔离策略（严格隔离、松散隔离）
6. 审计日志：所有跨公司数据访问审计日志
7. 数据导出：数据导出时自动按公司过滤，防止数据泄露
8. 集成隔离：与外部系统集成时传递公司上下文，确保数据隔离

**前置条件**:
- 公司组织结构已定义
- 用户公司归属已分配
- 数据隔离策略已配置
- 审计日志系统可用

**后置条件**:
- 数据查询自动按公司过滤
- 权限检查生效
- 审计日志记录
- 集团视图可用

**输入数据**:
1. 用户公司归属
2. 数据隔离策略配置
3. 公司间数据共享规则
4. 集团视图权限配置
5. 审计日志配置
6. 集成上下文参数
7. 数据导出限制规则
8. 异常访问处理规则

**输出数据**:
1. 数据查询结果（按公司过滤）
2. 权限检查结果
3. 集团汇总视图
4. 审计日志记录
5. 数据共享授权记录
6. 异常访问警报
7. 隔离策略执行报告

**验证规则**:
1. 用户必须有明确公司归属
2. 数据查询必须包含公司上下文
3. 权限检查必须在数据访问前执行
4. 集团视图必须基于明确授权
5. 审计日志必须完整不可篡改

**错误处理**:
1. 公司上下文缺失：拒绝数据访问
2. 权限不足：返回空数据或错误提示
3. 隔离策略冲突：使用默认策略并提示管理员
4. 审计日志失败：阻止数据访问并警报

**优先级**: 高  
**相关用户故事**: US-2.8.5, US-2.8.6

### 2.8.4 集团合并报告
**功能ID**: F-2.8.4  
**功能名称**: 集团合并报告  
**功能描述**: 生成集团合并报告，包括资产汇总、财务状况、运营绩效等，支持多币种转换、公司间交易抵消、少数股权处理。

**业务规则**:
1. 报告范围：按组织层级（集团、控股公司、业务板块）生成报告
2. 币种转换：多币种数据转换为报告币种（集团本位币），使用适当汇率
3. 公司间抵消：抵消公司间交易（资产转移、租赁、服务）以避免重复计算
4. 少数股权：处理非全资子公司的少数股权份额
5. 报告周期：月度、季度、年度合并报告
6. 报告版本：实际数、预算数、预测数、同比、环比
7. 审计跟踪：合并调整分录、汇率使用、抵消计算完整审计
8. 披露要求：满足不同国家/地区的财务报告披露要求

**前置条件**:
- 各公司财务和资产数据可用
- 汇率数据可用
- 公司间交易数据完整
- 股权比例数据准确
- 报告周期定义

**后置条件**:
- 集团合并报告生成
- 合并调整分录记录
- 审计跟踪保存
- 报告分发和归档

**输入数据**:
1. 各公司财务和资产数据
2. 汇率数据（历史汇率、平均汇率、期末汇率）
3. 公司间交易明细
4. 股权比例数据
5. 报告周期和范围
6. 报告币种
7. 合并规则（抵消规则、少数股权处理方法）
8. 披露要求模板

**输出数据**:
1. 集团合并报告（资产、负债、权益、收入、费用等）
2. 合并工作底稿
3. 合并调整分录
4. 币种转换计算表
5. 公司间交易抵消表
6. 少数股权计算表
7. 审计跟踪报告
8. 披露附注

**验证规则**:
1. 数据必须完整（所有纳入合并范围的公司）
2. 汇率必须适用于相应期间
3. 公司间交易必须匹配（金额、期间）
4. 股权比例必须准确
5. 合并规则必须符合会计准则

**错误处理**:
1. 数据缺失：标记报告为不完整，提示数据缺口
2. 汇率缺失：使用最近可用汇率并标记
3. 交易不匹配：调查差异并人工调整
4. 规则冲突：会计部门确定适用规则

**优先级**: 高  
**相关用户故事**: US-2.8.7, US-2.8.8

### 2.8.5 跨公司资产转移
**功能ID**: F-2.8.5  
**功能名称**: 跨公司资产转移  
**功能描述**: 管理集团内跨公司资产转移，包括转移审批、成本分配、税务处理、账务调整等完整流程。

**业务规则**:
1. 转移类型：有偿转移（内部销售）、无偿转移（捐赠）、租赁转移、临时借用
2. 审批流程：转出公司审批→转入公司审批→集团审批（如需要）
3. 定价机制：市场价、成本价、协议价、税务核定价格
4. 税务处理：增值税、关税、转让定价合规、税务申报
5. 账务处理：转出公司资产减少、转入公司资产增加、内部往来科目
6. 文档要求：转移协议、发票（如需要）、运输单据、验收证明
7. 跟踪要求：转移资产历史跟踪（原公司、现公司、转移日期）
8. 合规要求：满足不同国家/地区的资产转移法规和税务要求

**前置条件**:
- 转出和转入公司明确
- 转移资产可转移（非限制资产）
- 审批流程定义
- 税务规则可用

**后置条件**:
- 资产所有权变更
- 财务账务调整
- 税务申报触发
- 转移记录保存

**输入数据**:
1. 转移资产ID列表
2. 转出公司ID
3. 转入公司ID
4. 转移类型
5. 转移价格（如适用）
6. 转移日期
7. 审批人列表
8. 转移文档附件

**输出数据**:
1. 资产转移记录
2. 审批状态
3. 财务调整分录
4. 税务处理建议
5. 转移文档包
6. 资产历史更新
7. 合规检查报告
8. 内部往来对账记录

**验证规则**:
1. 转出公司必须有资产所有权
2. 转入公司必须有接收资产能力（技术、合规）
3. 转移价格必须合理（符合转让定价规则）
4. 审批流程必须完成
5. 文档必须完整（根据转移类型）

**错误处理**:
1. 资产不可转移：提示限制原因（抵押、租赁、限制出口等）
2. 定价不合理：税务部门审核调整
3. 审批拒绝：记录拒绝原因并通知相关方
4. 税务不合规：暂停转移直至合规解决

**优先级**: 高  
**相关用户故事**: US-2.8.9, US-2.8.10

### 2.8.6 公司间资产租赁
**功能ID**: F-2.8.6  
**功能名称**: 公司间资产租赁  
**功能描述**: 管理集团内公司间资产租赁，包括租赁合同管理、租金计算、费用结算、税务处理、会计处理等。

**业务规则**:
1. 租赁类型：经营租赁、融资租赁、短期租赁、长期租赁
2. 合同管理：租赁合同创建、审批、执行、续约、终止
3. 租金计算：基于资产价值、市场租金、协商租金，支持固定租金和变动租金
4. 费用结算：定期租金结算、费用分摊、内部往来对账
5. 税务处理：租金发票、增值税处理、税务申报
6. 会计处理：租赁会计准则（IFRS 16/ASC 842）合规处理
7. 资产跟踪：租赁期间资产位置、状态、使用情况跟踪
8. 合规要求：满足不同国家/地区的租赁法规和税务要求

**前置条件**:
- 出租公司和承租公司明确
- 租赁资产可出租
- 租赁合同模板可用
- 会计准则配置

**后置条件**:
- 租赁合同生效
- 租金计算和结算设置
- 会计处理配置
- 税务申报设置

**输入数据**:
1. 租赁资产ID
2. 出租公司ID
3. 承租公司ID
4. 租赁类型
5. 租赁条款（期限、租金、支付频率）
6. 合同文档
7. 会计准则选择
8. 税务处理规则

**输出数据**:
1. 租赁合同记录
2. 租金计算表
3. 费用结算计划
4. 会计处理分录
5. 税务处理建议
6. 租赁资产跟踪记录
7. 合同执行报告
8. 合规检查报告

**验证规则**:
1. 出租公司必须有资产所有权或转租权
2. 租赁条款必须合法合规
3. 租金必须合理（符合市场水平）
4. 会计准则必须正确应用
5. 税务处理必须合规

**错误处理**:
1. 资产不可出租：提示限制原因
2. 条款不合法：法律部门审核修改
3. 租金争议：重新协商或仲裁
4. 会计处理错误：会计部门调整

**优先级**: 中  
**相关用户故事**: US-2.8.11, US-2.8.12

### 2.8.7 集团级政策模板
**功能ID**: F-2.8.7  
**功能名称**: 集团政策管理  
**功能描述**: 管理集团级资产政策模板，支持政策制定、发布、本地化定制、执行跟踪、合规检查。

**业务规则**:
1. 政策类型：资产采购政策、资产分类政策、折旧政策、处置政策、使用政策
2. 模板管理：集团级政策模板创建、版本管理、发布流程
3. 本地化定制：子公司基于集团模板进行本地化定制（基于法规、业务差异）
4. 政策执行：政策与业务流程集成，自动检查政策合规性
5. 例外管理：政策例外申请、审批、跟踪
6. 合规报告：政策执行合规报告、例外报告
7. 政策培训：政策培训材料、培训记录
8. 审计就绪：政策文档、执行证据、例外记录审计就绪

**前置条件**:
- 集团政策需求明确
- 政策制定流程定义
- 本地化定制规则定义
- 合规检查机制可用

**后置条件**:
- 集团政策模板发布
- 子公司本地化完成
- 政策合规检查集成
- 审计材料准备

**输入数据**:
1. 政策类型
2. 政策内容（文本、规则、参数）
3. 适用范围（公司列表、资产类型列表）
4. 本地化定制规则
5. 例外审批流程
6. 合规检查规则
7. 培训材料
8. 审计要求

**输出数据**:
1. 集团政策模板
2. 子公司本地化政策
3. 政策合规检查结果
4. 例外审批记录
5. 合规报告
6. 培训记录
7. 审计材料
8. 政策执行仪表板

**验证规则**:
1. 政策内容必须合法合规

2. 优化维度：资产分配优化、共享优化、处置优化、采购优化
3. 优化模型：使用运筹学模型、成本效益分析、风险分析
4. 建议类型：资产转移建议、共享池建立建议、处置建议、采购建议
5. 实施考虑：实施成本、技术可行性、组织阻力、时间要求
6. 效益评估：预计成本节约、价值提升、风险降低
7. 实施跟踪：建议采纳情况、实施进度、实际效益
8. 持续优化：定期重新评估和优化，适应变化

**前置条件**:
- 集团资产全景数据可用
- 优化模型已建立
- 实施能力评估方法可用
- 效益评估指标定义

**后置条件**:
- 优化建议生成
- 效益评估完成
- 实施跟踪开始
- 持续优化机制建立

**输入数据**:
1. 集团资产全景数据
2. 优化目标和约束
3. 优化模型参数
4. 实施成本数据
5. 风险评估参数
6. 效益评估指标
7. 跟踪机制
8. 重新评估周期

**输出数据**:
1. 集团资产优化建议报告
2. 优化方案详情
3. 成本效益分析
4. 风险评估
5. 实施计划
6. 跟踪仪表板
7. 实际效益报告
8. 持续优化计划

**验证规则**:
1. 优化模型必须合理
2. 数据必须完整准确
3. 实施考虑必须全面
4. 效益评估必须可靠
5. 跟踪必须及时

**错误处理**:
1. 模型无解：放松约束或调整目标
2. 数据质量差：数据清洗或使用替代数据
3. 实施不可行：调整建议或分阶段实施
4. 效益不达预期：分析原因并调整模型

**优先级**: 高  
**相关用户故事**: US-2.8.29, US-2.8.30

### 2.8.16 国内集团管理合规
**功能ID**: F-2.8.16  
**功能名称**: 国内集团合规  
**功能描述**: 确保集团管理符合中国国内法规要求，包括国有企业监管、集团报告、关联交易披露、内部控制和风险管理等。

**业务规则**:
1. 法规库：中国集团管理相关法规（企业国有资产法、公司法、证券法、国资委规定）
2. 合规检查：自动检查集团管理操作是否符合法规要求
3. 报告要求：满足国内监管机构报告要求（国资委、证监会、交易所）
4. 关联交易：关联方识别、关联交易审批、披露、定价合规
5. 内部控制：符合国内内部控制规范要求
6. 风险管理：符合国内风险管理要求（中央企业全面风险管理指引）
7. 审计要求：满足国内审计要求（内部控制审计、财务审计）
8. 违规处理：违规检测、报告、整改、处罚跟踪

**前置条件**:
- 国内法规库可用
- 合规检查规则定义
- 报告模板可用
- 关联方数据完整

**后置条件**:
- 合规检查持续运行
- 监管报告可生成
- 关联交易合规管理
- 内控和风险管理就绪

**输入数据**:
1. 国内法规内容
2. 合规检查规则
3. 报告模板和要求
4. 关联方数据
5. 内控要求
6. 风险管理框架
7. 审计要求
8. 违规处理流程

**输出数据**:
1. 合规检查报告
2. 监管报告
3. 关联交易合规报告
4. 内控评估报告
5. 风险管理报告
6. 审计准备材料
7. 违规处理记录
8. 合规状态仪表板

**验证规则**:
1. 法规库必须最新
2. 合规检查必须准确
3. 报告必须满足监管格式
4. 关联交易必须合规
5. 内控必须有效

**错误处理**:
1. 法规更新延迟：提示法规库过期风险
2. 合规检查误报：优化检查规则
3. 报告格式不符：调整报告模板
4. 关联交易违规：暂停交易直至合规

**优先级**: 高  
**相关用户故事**: US-2.8.31, US-2.8.32

### 2.8.18 国内会计准则支持
**功能ID**: F-2.8.18  
**功能名称**: 国内会计准则支持  
**功能描述**: 支持中国会计准则（CAS）要求，包括资产确认、计量、折旧、减值、披露等全套会计处理。

**业务规则**:
1. 准则范围：企业会计准则（CAS）及相关解释、应用指南
2. 资产确认：符合CAS资产确认条件（未来经济利益很可能流入、成本可靠计量）
3. 计量模式：成本模式为主，特定情况公允价值模式
4. 折旧方法：直线法、工作量法、双倍余额递减法、年数总和法
5. 减值处理：CAS资产减值准则，减值测试、减值准备计提和转回
6. 披露要求：财务报表附注披露要求（资产分类、原值、累计折旧、减值准备等）
7. 税会差异：会计处理与税务处理差异记录和调整
8. 准则更新：跟踪准则更新，及时系统适配

**前置条件**:
- 会计准则要求明确
- 折旧方法配置
- 减值测试模型可用
- 披露模板定义

**后置条件**:
- 会计准则配置生效
- 折旧计算合规
- 减值处理就绪
- 披露报告可生成

**输入数据**:
1. 会计准则版本
2. 资产确认条件
3. 计量模式选择
4. 折旧方法参数
5. 减值测试模型
6. 披露模板
7. 税会差异规则
8. 准则更新跟踪

**输出数据**:
1. 会计准则合规报告
2. 折旧计算表
3. 减值测试报告
4. 减值准备计提记录
5. 财务报表披露
6. 税会差异调整表
7. 准则更新影响分析
8. 审计意见支持材料

**验证规则**:
1. 会计处理必须符合CAS
2. 折旧方法必须适当
3. 减值测试必须及时
4. 披露必须完整准确
5. 税会差异必须正确处理

**错误处理**:
1. 准则理解偏差：会计专家确认正确理解
2. 折旧方法不当：调整方法并追溯调整
3. 减值测试不及时：补测试并评估影响
4. 披露不完整：补充披露并改进流程

**优先级**: 高  
**相关用户故事**: US-2.8.35, US-2.8.36
