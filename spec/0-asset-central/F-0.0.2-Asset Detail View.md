# F-0.0.2 - Asset Detail View

**Requirement ID:** F-0.0.2  
**Module:** Asset Central (0)  
**Priority:** High

---

## Requirement Description

The system shall provide a comprehensive Asset Detail View that displays all information about a specific asset. This view shall serve as the central hub for all asset-centric operations, enabling users to manage maintenance, financial, documents, tracking, and lifecycle aspects of the asset from a single interface.

---

## Business Rules

1. Asset Detail View shall be accessible by clicking on any asset from the Asset Central Hub
2. All tabs shall load relevant data for the selected asset
3. Quick Actions shall provide one-click access to common operations
4. Breadcrumb navigation shall allow easy return to Asset Central Hub
5. Edit operations shall open the appropriate edit form in a modal or navigate to the relevant module
6. All operations initiated from Asset Detail shall pre-populate the asset context

---

## Tab Details

### Overview Tab

**Basic Information:**
- Asset Type, Category, Criticality
- Manufacturer, Model, Serial Number
- Acquisition Date, Installation Date

**Location & Organization:**
- Company, Plant, Production Line
- Department, Cost Center
- Physical Location (Building, Zone, Station)

**Technical Specifications:**
- Power Rating, Voltage, Weight
- Year Built, Warranty Expiry

**Quick Actions:**
- Create Work Order
- Record Inspection
- Borrow / Return
- Reserve Asset
- View Contracts
- Dispose Asset

---

### Maintenance Tab

**Work Orders Section:**
- Active work orders list
- Work order status badges
- Link to create new work order

**Maintenance History:**
- Chronological list of completed maintenance
- Type (Preventive, Corrective, Calibration)
- Cost, Duration, Technician

**Sub-tabs:**
- Work Orders (5)
- Preventive Schedule
- Predictive Data
- Spare Parts

---

### Financial Tab

**Summary Cards:**
- Original Cost
- Current Book Value
- Accumulated Depreciation
- Residual Value

**Depreciation Details:**
- Method (Straight-Line, Declining Balance)
- Useful Life, Salvage Value
- Monthly Depreciation, Remaining Life

**Insurance & Tax:**
- Insurance Policy Number, Value
- Tax Classification
- Property Tax Amount
- Last Appraisal Date

**Depreciation Schedule:**
- Visual chart showing depreciation over time
- Year-by-year book value projection

---

### Documents Tab

**Document Categories:**
- Certificate of Origin
- Warranty Certificate
- Technical Specifications
- Photos

**Document List:**
- Document name, type, upload date
- File size, download button

---

### History Tab

**Timeline View:**
- All changes to the asset
- Date, time, change type
- User who made the change
- Previous and new values

---

### Hierarchy Tab

**Parent Asset:**
- Parent asset name and ID
- Link to parent asset detail

**Child Components:**
- List of sub-components
- Component ID, name, status

**Visual Tree:**
- Hierarchical tree view
- Expandable/collapsible nodes

---

### Tracking Tab

**Current Location:**
- Building, Zone, Line/Station
- GPS Coordinates
- RFID Status

**Check-in/Check-out:**
- Current status (Checked In/Out)
- Last check-in/out time
- User who has the asset

**Transfer History:**
- All transfers for this asset
- Transfer type, date, approval

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full access to all assets; all actions |
| Company Admin | Full access to assets within company; all actions |
| Division Manager | View and perform actions on assets within division |
| Plant Manager | View and perform actions on assets within plant |
| Department Manager | View and perform actions on assets within department |
| Operator | View assets; limited actions (inspection, check-in/out) |

---

## Multi-Company Data Scope

- **Data Level**: Asset's organizational level (Company → Division → Plant → Department → Cost Center)
- **Access**: User must have access to the asset's organizational level
- **Cross-Company**: System Admin can view assets across all companies
- **Data Isolation**: Asset data is isolated by company's data boundaries
