# F-2.3.1 - Scheduled Maintenance Planning

**Requirement ID:** F-2.3.1  
**Module:** Maintenance Management  
**Priority:** High

---

## Requirement Description

Create and manage regular maintenance schedules for assets including frequency, content, and resource allocation. Maintenance plans are based on time or usage (runtime hours, production cycles) with support for multiple frequencies (daily, weekly, monthly, quarterly, yearly). Plans support advance notice days, templates, batch creation, and require reason documentation and approval for changes.

---

## Business Rules

1. Maintenance plans can be created for assets that are in Active or In-Use status
2. Plans must specify either time-based or usage-based frequency with valid intervals
3. Work orders may be auto-generated when a plan becomes due
4. Plans require approval workflow for changes with documented reason
5. Time-based plans calculate next execution from start date and frequency interval
6. Usage-based plans trigger when asset usage reaches threshold
7. Plans can be suspended with reason and later reactivated

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset | Yes | Asset to be maintained |
| Maintenance Type | Yes | Preventive, Corrective, Predictive, or Inspection |
| Frequency Type | Yes | Time-based, Usage-based, or Hybrid |
| Frequency Value | Yes | Number of units between maintenance |
| Frequency Unit | Yes | Day, Week, Month, Quarter, or Year |
| Usage Threshold | No | Usage hours/cycles that trigger maintenance |
| Start Date | Yes | When the maintenance plan begins |
| Advance Notice Days | Yes | Days before due date to send notification |
| Estimated Hours | Yes | Expected labor hours for maintenance |
| Estimated Cost | Yes | Expected total cost |
| Assigned Technician | Yes | Person responsible for the maintenance |
| Auto Generate Work Order | Yes | Whether to automatically create work order when due |
| Parent Template | No | Template to copy tasks from |

---

## Output Data

| Data | Description |
|------|-------------|
| Maintenance Plan | Created maintenance schedule record |
| Next Execution Date | Calculated date of next maintenance |
| Work Order | Auto-generated work order (if enabled) |
| Notification | Scheduled advance notice to technician |

---

## Validation Rules

1. Asset must exist and be in Active or In-Use status
2. Start date cannot be in the past unless creating a historical plan
3. Advance notice days must be between 0 and 30
4. Frequency value must be a positive integer
5. Assigned technician must exist and be active
6. No duplicate active plan for same asset and maintenance type without override reason

---

## Access Control

| Role | Permission |
|------|------------|
| Maintenance Manager | Create, edit, approve plans |
| Plant Supervisor | View and modify within plant |
| Technician | View assigned plans |
| Department User | View within department only |

---

## Multi-Company Data Scope

- **Data Level**: Company level with plant inheritance
- **Access**: Users can only access plans for assets within their organizational hierarchy
- **Isolation**: Company Code filters all data access; cross-company access restricted |

---

## Compliance Requirements

- ISO 55000 asset management planning alignment
- SOX audit trail for plan modifications and cost tracking
- OSHA safety-related maintenance schedule documentation
- GAMP 5 calibration and maintenance documentation for regulated industries
