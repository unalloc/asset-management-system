### 1.1 Requirement Description

Create and manage regular maintenance schedules for assets including frequency, content, and resource allocation. Maintenance plans are based on time or usage (runtime hours, production cycles) with support for multiple frequencies (daily, weekly, monthly, quarterly, yearly). Plans support advance notice days, templates, batch creation, and require reason documentation and approval for changes.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset must exist and be in Active/In-Use status | Asset.Status IN ('Active', 'In-Use') AND Asset.CompanyCode = Session.CompanyCode | MPR-001 | "Asset {AssetID} does not exist or is not in an active maintenance-eligible status" |
| PRE-002 | Maintenance type must be valid | MaintenanceType IN ('Preventive', 'Corrective', 'Predictive', 'Inspection') | MPR-002 | "Invalid maintenance type: {MaintenanceType}" |
| PRE-003 | Frequency value must be positive integer | FrequencyValue > 0 AND FrequencyValue IS INTEGER | MPR-003 | "Frequency value must be a positive integer" |
| PRE-004 | Usage threshold must be positive number (if provided) | UsageThreshold IS NULL OR UsageThreshold > 0 | MPR-004 | "Usage threshold must be a positive number" |
| PRE-005 | Start date cannot be in past (unless historical plan) | StartDate >= CURRENT_DATE OR IsHistoricalPlan = TRUE | MPR-005 | "Start date cannot be in the past unless this is a historical plan" |
| PRE-006 | Advance notice days within valid range | AdvanceNoticeDays >= 0 AND AdvanceNoticeDays <= 30 | MPR-006 | "Advance notice days must be between 0 and 30" |
| PRE-007 | Responsible team/person must exist and be active | AssignedTo IS NOT NULL AND AssignedTo.Active = TRUE | MPR-007 | "Assigned team/person {AssignedTo} does not exist or is inactive" |

**Pre-condition Validation Matrix:**

```
IF Asset.Status = 'Active' OR Asset.Status = 'In-Use'
   AND Asset.CompanyCode = Session.CompanyCode
   AND MaintenanceType IN ('Preventive', 'Corrective', 'Predictive', 'Inspection')
   AND FrequencyValue > 0
   AND (UsageThreshold IS NULL OR UsageThreshold > 0)
   AND (StartDate >= CURRENT_DATE OR IsHistoricalPlan = TRUE)
   AND AdvanceNoticeDays >= 0 AND AdvanceNoticeDays <= 30
   AND AssignedTo IS NOT NULL AND AssignedTo.Active = TRUE
THEN proceed to plan creation
ELSE throw appropriate PRE-xxx error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Maintenance plan created with computed next execution date | PlanID generated, NextExecutionDate calculated correctly | Query MaintenancePlan WHERE PlanID = {PlanID} |
| POST-002 | Work order auto-generated (if auto-trigger enabled) | WorkOrder created linked to MaintenancePlan | Query WorkOrder WHERE SourcePlanID = {PlanID} |
| POST-003 | Notification sent to responsible party | NotificationLog entry created | Query NotificationLog WHERE PlanID = {PlanID} |

**Post-condition Verification:**

Upon successful maintenance plan creation:
- MaintenancePlan record created with status = 'Active'
- NextExecutionDate calculated based on StartDate and Frequency
- If AutoGenerateWO = TRUE, WorkOrder created with status = 'Pending'
- Notification scheduled for AdvanceNoticeDays before NextExecutionDate

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
[Validate Asset Status]
  │
  ├─► Invalid ──► MPR-001: Asset not eligible
  │
  ▼
[Validate Frequency Type]
  │
  ├─► Time-Based ──► [Calculate Next Execution = StartDate + (FrequencyValue * FrequencyUnit)]
  │
  ├─► Usage-Based ──► [Set Next Execution = NULL, trigger on Usage Threshold crossing]
  │
  └─► Hybrid ──► [Calculate both, trigger on whichever comes first]
        │
        ▼
[Calculate Costs]
  │
  ▼
[Check Approval Required?]
  │
  ├─► YES ──► [Create plan in Pending Approval status]
  │            └─► [On Approval] ──► [Set Status = Active]
  │
  └─► NO ──► [Set Status = Active]
        │
        ▼
   [Generate Notifications]
        │
        ▼
       END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Next Execution Calculator | Plan.Status = 'Active' AND Plan.FrequencyType = 'Time' | NextExecutionDate = LastExecutionDate + (FrequencyValue * FrequencyUnit) | 1 |
| BR-002 | Usage Threshold Trigger | Plan.FrequencyType = 'Usage' AND Asset.CurrentUsage >= Plan.UsageThreshold | Create WorkOrder, Reset CurrentUsage | 1 |
| BR-003 | Auto-Generation Rule | Plan.AutoGenerateWO = TRUE AND NextExecutionDate <= CURRENT_DATE | Create WorkOrder automatically | 2 |
| BR-004 | Batch Template Application | ParentTemplateID IS NOT NULL | Copy template tasks to new plan | 3 |
| BR-005 | Cost Estimation Update | Plan.EstimatedCost updated | Recalculate TotalPlannedCost | 2 |

**Calculation Logic:**

```
NextExecutionDate (Time-Based):
  NextExecutionDate = StartDate + (FrequencyValue * FrequencyUnit)
  
  Where FrequencyUnit conversions:
    - Daily: multiply by 1
    - Weekly: multiply by 7
    - Monthly: add FrequencyValue months (last day handling)
    - Quarterly: add FrequencyValue * 3 months
    - Yearly: add FrequencyValue years

NextExecutionDate (Usage-Based):
  NextExecutionDate = NULL
  TriggerCondition: Asset.CurrentUsage >= (LastResetUsage + UsageThreshold)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Asset with 0 usage hours | Usage-based plan never triggers | Default to time-based fallback; warn user |
| EC-002 | Start date on month-end (31st) for monthly frequency | Schedule correctly handles Feb/etc | Use DATEFROMPARTS with proper month arithmetic |
| EC-003 | Historical plan creation | Backfill maintenance history | Set IsHistoricalPlan = TRUE, skip NextExecution calc |
| EC-004 | Plan with no assigned technician | Allow creation but flag for assignment | Warning WRN-001, block auto-generation |
| EC-005 | Duplicate plan for same asset/frequency | Prevent creation | Check existing plans, throw MPR-008 |
| EC-006 | Frequency change mid-cycle | Reset calculation from change date | Create audit record, recalculate NextExecution |

**Edge Case Decision Matrix:**

| FrequencyType | UsageThreshold | StartDate | NextExecution | Outcome |
|---------------|----------------|-----------|---------------|---------|
| Time | NULL | Future | Calculated | Normal time-based schedule |
| Time | Provided | Future | Calculated | Time-based, usage ignored until threshold met |
| Usage | Provided | Past/Today | NULL | Usage-triggered only |
| Usage | NULL | Any | ERROR | MPR-009: Usage threshold required |
| Hybrid | Provided | Future | Earlier of both | Dual-trigger schedule |
| NULL | ANY | Any | ERROR | MPR-010: Frequency type required |
