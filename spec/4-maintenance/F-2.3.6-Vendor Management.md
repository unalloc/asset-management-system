# F-2.3.6 - Vendor Management

## Requirement ID
F-2.3.6

## Function Name
Service Vendor Management

## Function Description
Manage maintenance service vendor information, qualifications, performance, and business relationships.

## Business Rules

1. Vendor basic information management
2. Qualification certificates and validity period tracking
3. Service classification and capability assessment
4. Performance scoring and evaluation system
5. Contract history and cooperation records
6. Vendor risk assessment

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Vendor Basic Info | Yes | Name, type, size |
| Contact Information and Address | Yes | How to reach |
| Qualification Certificates | Yes | Type, number, validity |
| Service Capability Classification | Yes | What they can do |
| Historical Cooperation Evaluation | No | Past performance |
| Risk Assessment Factors | No | Risk indicators |
| Bank Account Information | No | For payments |
| Qualification Document Attachments | No | Proof files |

## Output Data

1. Vendor ID
2. Vendor profile
3. Qualification status (Valid/Expiring/Expired)
4. Performance score report
5. Risk assessment report
6. Qualified vendor list
7. Qualification expiration alerts

## Validation Rules

1. Vendor name must be unique
2. Certificate validity must be reasonable
3. Contact information must be complete
4. Score must be within valid range (e.g., 1-5)
5. Risk assessment factors must be complete

## Priority
Medium
