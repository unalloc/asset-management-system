## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset criticality must be rated | Asset.CriticalityClass IS NOT NULL | WOP-001 | "Asset must have criticality rating" |
| PRE-002 | Production schedule must be available | ProductionSchedule EXISTS for date range | WOP-002 | "Production schedule required for prioritization" |
| PRE-003 | Maintenance time estimates must be reasonable | EstimatedHours > 0 | WOP-003 | "Estimated hours must be positive" |
| PRE-004 | Resource constraints must be defined | Resources defined for scheduling | WOP-004 | "Resource constraints must be defined" |

### 3.2 Business Logic

**Priority Score Calculation:**

```
PriorityScore = (CriticalityWeight * AssetCriticality) + 
                (ProductionWeight * ProductionImpact) + 
                (SafetyWeight * SafetyRisk) +
                (UrgencyWeight * UrgencyFactor)

Where:
  AssetCriticality = A=5, B=3, C=1
  ProductionImpact = EstimatedDowntimeHours * HourlyProductionLoss / 10000 (normalized)
  SafetyRisk = 1-5 (from SafetyEnvironmentalRiskLevel)
  UrgencyFactor = DaysOverdue / 30 (capped at 5)

Default Weights:
  CriticalityWeight = 0.35
  ProductionWeight = 0.30
  SafetyWeight = 0.25
  UrgencyWeight = 0.10
```

### 3.3 State Transitions

**Work Order Priority States:**

| State | Description | Valid Transitions |
|-------|-------------|-------------------|
| LOW | Score < 2.0 | MEDIUM |
| MEDIUM | Score >= 2.0 AND < 3.5 | LOW, HIGH |
| HIGH | Score >= 3.5 AND < 4.5 | MEDIUM, EMERGENCY |
| EMERGENCY | Score >= 4.5 | HIGH |
