# F-2.3.11 - Work Order Prioritization

**Requirement ID:** F-2.3.11  
**Module:** Maintenance Management  
**Priority:** High

---

## Requirement Description

Automatically determine work order priority based on asset criticality and production impact. Includes asset criticality rating (A/B/C class), production impact assessment (downtime, production loss), safety environmental risk consideration, resource availability constraints, dynamic priority adjustment, and scheduling algorithm optimization.

---

## Business Rules

1. Priority Score = (CriticalityWeight x AssetCriticality) + (ProductionWeight x ProductionImpact) + (SafetyWeight x SafetyRisk) + (UrgencyWeight x UrgencyFactor)
2. Default weights: Criticality 35%, Production 30%, Safety 25%, Urgency 10%
3. Asset Criticality: A=5, B=3, C=1
4. Emergency priority for score >= 4.5
5. High priority for score >= 3.5
6. Medium priority for score >= 2.0
7. Low priority for score < 2.0

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Work Order | Yes | Work order to prioritize |
| Asset Criticality | Yes | A, B, or C classification |
| Production Impact | Yes | Downtime hours and production loss |
| Safety Risk | Yes | Safety risk level (1-5) |
| Urgency Factor | Yes | Days overdue / 30 (capped at 5) |
| Resource Constraints | Yes | Available technician hours |

---

## Output Data

| Data | Description |
|------|-------------|
| Priority Score | Calculated priority number |
| Priority Level | Low, Medium, High, or Emergency |
| Scheduling Recommendation | Suggested execution order |

---

## Validation Rules

1. Asset must have criticality rating
2. Production schedule must be available for prioritization
3. Estimated hours must be positive
4. Resource constraints must be defined

---

## Access Control

| Role | Permission |
|------|------------|
| Maintenance Manager | Set weights and priorities |
| Plant Supervisor | View and reprioritize within plant |
| Scheduler | Adjust scheduling priorities |

---

## Multi-Company Data Scope

- **Data Level**: Company level with plant-level adjustments
- **Access**: Prioritization view at all levels, modification at plant level
- **Isolation**: Priority calculations respect company boundaries |

---

## Compliance Requirements

- OSHA safety-critical work orders prioritized
- GAMP 5 critical equipment prioritization for validated systems
