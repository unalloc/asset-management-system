### 1.1 Requirement Description

Link maintenance activities with spare parts inventory system, tracking spare parts usage and inventory changes. Maintenance work orders can link required spare parts, spare parts requisition and approval process, actual usage recording and inventory deduction, spare parts usage cost charged to maintenance cost, spare parts inventory alerts (below safety stock), and spare parts usage trend analysis.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Spare part must exist and be available | Part.PartID = PartID AND Part.AvailableQty >= RequestedQty | SPI-001 | "Spare part {PartID} is not available in required quantity" |
| PRE-002 | Requested quantity cannot exceed available | RequestedQty <= Part.AvailableQty | SPI-002 | "Requested quantity exceeds available inventory" |
| PRE-003 | Actual usage cannot exceed requisitioned | ActualUsage <= RequisitionQty | SPI-003 | "Actual usage cannot exceed requisitioned quantity" |
| PRE-004 | Return quantity cannot exceed unused | ReturnQty <= (RequisitionQty - ActualUsage) | SPI-004 | "Return quantity cannot exceed unused quantity" |
| PRE-005 | Serial number must be valid and unused (if tracking) | IF Part.TrackSerial = TRUE THEN SerialNumber NOT IN UsedSerials | SPI-005 | "Serial number is invalid or already used" |
| PRE-006 | Work order must allow parts linking | WO.Status IN ('Pending', 'Assigned', 'InProgress') | SPI-006 | "Cannot add parts to work order in current status" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Inventory reserved | Part.AvailableQty -= RequestedQty |
| POST-002 | Cost allocated to work order | WO.PartsCost += (UnitCost * ActualUsage) |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
[Technician requests parts]
  │
  ▼
[Validate Part Availability]
  │
  ├─► FAIL ──► Check alternatives ──► [Suggest substitute or await restock]
  │
  ▼
[Reserve Inventory]
  │
  ▼
[Technician retrieves parts]
  │
  ▼
[Record actual usage]
  │
  ▼
[Return unused parts]
  │
  ▼
[Adjust inventory + cost]
  │
  ▼
   END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Auto Reserve | Requisition approved | Inventory.ReservedQty += RequestedQty | 1 |
| BR-002 | Cost Calculation | Parts used | WO.PartsCost += (UnitCost * ActualUsage) | 1 |
| BR-003 | Return Processing | Parts returned | Inventory.AvailableQty += ReturnQty | 1 |
| BR-004 | Safety Stock Alert | AvailableQty < SafetyStock | Generate alert | 1 |
| BR-005 | FIFO Allocation | Multiple lots available | Allocate oldest lot first | 2 |
