### 1.1 Requirement Description

Manage the complete workflow for preventive maintenance from plan triggering to work order creation, execution, and closure. The workflow includes auto-generation of work orders from maintenance plans, detailed inspection items and standards, work instructions and checklists, execution recording (start time, end time, results), maintenance result verification and approval, and automatic maintenance history update.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Maintenance plan must exist and be active | Plan.PlanID = {PlanID} AND Plan.Status = 'Active' | PMW-001 | "Maintenance plan {PlanID} does not exist or is not active" |
| PRE-002 | Asset must be in maintenance-eligible status | Asset.Status IN ('Active', 'In-Use') | PMW-002 | "Asset {AssetID} is not eligible for maintenance work" |
| PRE-003 | Inspection checklist must have at least one item | Checklist.Items.Count >= 1 | PMW-003 | "Inspection checklist must have at least one item" |
| PRE-004 | Priority must be valid | Priority IN ('Low', 'Medium', 'High', 'Emergency') | PMW-004 | "Priority must be Low, Medium, High, or Emergency" |
| PRE-005 | Execution date must be within valid range | ExecutionDate >= CURRENT_DATE AND ExecutionDate <= CURRENT_DATE + 365 | PMW-005 | "Execution date must be within the next 12 months" |
| PRE-006 | Scheduled resources must be available | ResourceAvailabilityCheck(Resources) = TRUE | PMW-006 | "One or more required resources are not available" |

**Pre-condition Validation Matrix:**

```
IF Plan.Status = 'Active'
   AND Asset.Status IN ('Active', 'In-Use')
   AND Checklist.Items.Count >= 1
   AND Priority IN ('Low', 'Medium', 'High', 'Emergency')
   AND ExecutionDate >= CURRENT_DATE
   AND ExecutionDate <= CURRENT_DATE + 365
   AND ResourcesAvailable = TRUE
THEN proceed to work order creation
ELSE throw appropriate PRE-xxx error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
[Trigger Event: Plan Due OR Manual Request]
  │
  ▼
[Validate Pre-conditions]
  │
  ├─► FAIL ──► Return error, log failure
  │
  ▼
[Create Work Order]
  │
  ▼
[Calculate Dynamic Priority]
  │  Priority = Asset.Criticality * Impact + Urgency
  │
  ▼
[Assign Technician]
  │
  ├─► Auto-assign based on skills ──► [Send Notification]
  │
  └─► Manual assignment required ──► [Hold for Assignment]
        │
        ▼
   [Technician accepts assignment]
        │
        ▼
   [Execute Work]
        │
        ▼
   [Complete Checklist Items]
        │
        ▼
   [Record Results]
        │
        ▼
   [Supervisor Verification]
        │
        ├─► APPROVE ──► [Update History] ──► [Close WO]
        │
        └─► REJECT ──► [Return to Technician] ──► [Re-execute]
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Auto WO Generation | Plan.AutoGenerateWO = TRUE AND DueDate <= CURRENT_DATE | Create WorkOrder, Set Status = Pending | 1 |
| BR-002 | Dynamic Priority Calculation | Always | PriorityScore = (AssetCriticality * 0.4) + (ProductionImpact * 0.3) + (SafetyRisk * 0.3) | 1 |
| BR-003 | Technician Assignment | WO created | Match skills: WO.RequiredSkills ∩ Technician.Certifications | 2 |
| BR-004 | Checklist Completion Validation | All items required = TRUE | All RequiredItems = Checked | 1 |
| BR-005 | Auto History Update | WO.Status = 'Completed' AND Approved = TRUE | Create MaintenanceHistory record | 1 |

**Calculation Logic:**

```
Dynamic Priority Score:
  Score = (AssetCriticality * 0.4) + (ProductionImpact * 0.3) + (SafetyRisk * 0.3)
  
  Where:
    AssetCriticality = 1-5 (A-E class mapped to 5-1)
    ProductionImpact = EstimatedDowntimeHours * HourlyProductionLoss
    SafetyRisk = 1-5 based on SafetyEnvironmentalRiskLevel
  
  Priority Mapping:
    Score >= 4.0 ──► Emergency
    Score >= 3.0 ──► High
    Score >= 2.0 ──► Medium
    Score < 2.0 ──► Low

Estimated Downtime Calculation:
  EstDowntime = WO.EstimatedHours + TravelTime + PartsRetrievalTime
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Plan generates WO but asset is under repair | Prevent duplicate concurrent WOs | Check existing open WOs for same asset |
| EC-002 | Technician becomes unavailable after assignment | Re-assign or escalate to supervisor | Alert supervisor, auto-suggest alternatives |
| EC-003 | Work order exceeds estimated time | Allow continuation with reason | Log extension, update estimated cost |
| EC-004 | Parts not available at execution time | Allow work with substitute parts | Require substitute approval, record difference |
| EC-005 | Emergency WO while resources busy | Escalate priority, potentially preempt lower priority | Resource reallocation algorithm |
| EC-006 | Multiple WOs for same asset same day | Merge or sequence | User decision prompt |

**Edge Case Decision Matrix:**

| AssetStatus | OpenWOCount | WOType | Action |
|-------------|-------------|---------|--------|
| Active | 0 | Any | Allow WO creation |
| Active | 1 | Preventive | Warn, allow override |
| Active | >= 2 | Any | Block - investigate |
| Under Repair | >= 1 | Any | Block - asset unavailable |
| Inactive | Any | Any | Block - invalid status |
