# F-2.3.2 - Preventive Maintenance Workflow

**Requirement ID:** F-2.3.2  
**Module:** Maintenance Management  
**Priority:** High

---

## Requirement Description

Manage the complete workflow for preventive maintenance from plan triggering to work order creation, execution, and closure. The workflow includes auto-generation of work orders from maintenance plans, detailed inspection items and standards, work instructions and checklists, execution recording (start time, end time, results), maintenance result verification and approval, and automatic maintenance history update.

---

## Business Rules

1. Work orders are automatically created from active maintenance plans when due
2. Dynamic priority calculation based on asset criticality, production impact, safety risk, and urgency
3. Technician assignment based on required skills and certifications
4. All required checklist items must be completed before work order can be closed
5. Supervisor verification and approval required before work order completion
6. Completed and approved work orders automatically create maintenance history records
7. Segregation of duties enforced: requestor cannot be approver

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Source Maintenance Plan | No | Plan that triggered the work order |
| Asset | Yes | Asset to be maintained |
| Priority | Yes | Low, Medium, High, or Emergency |
| Scheduled Date | Yes | When work is scheduled |
| Assigned Technician | Yes | Person assigned to perform work |
| Estimated Hours | Yes | Expected labor hours |
| Estimated Cost | Yes | Expected total cost |
| Safety Notes | No | Safety considerations for the work |
| Work Permit Required | Yes | Whether special permit is needed |
| Checklist Items | Yes | Inspection items and tasks to complete |

---

## Output Data

| Data | Description |
|------|-------------|
| Work Order | Created work order with unique code |
| Task List | Detailed tasks from work order |
| Notification | Alert sent to assigned technician |
| Maintenance History | Record created upon completion |

---

## Validation Rules

1. Maintenance plan must exist and be active (if auto-generated)
2. Asset must be in Active or In-Use status
3. Inspection checklist must have at least one item
4. Priority must be Low, Medium, High, or Emergency
5. Scheduled date must be within the next 12 months
6. Required resources must be available
7. Technician must have required certifications for the work

---

## Access Control

| Role | Permission |
|------|------------|
| Maintenance Manager | Create, assign, approve work orders |
| Plant Supervisor | Assign, start, complete work orders within plant |
| Technician | View and execute assigned work orders |
| Department User | View work orders within department |

---

## Multi-Company Data Scope

- **Data Level**: Company level with division/plant/department inheritance
- **Access**: Work order creation restricted to Plant level and below
- **Isolation**: Company Code filters all data; cross-plant work orders require Company Admin authorization |

---

## Compliance Requirements

- SOX audit trail for work order status changes and cost tracking
- OSHA safety checklists and LOTO procedures for applicable equipment
- GAMP 5 electronic signatures and audit trail for regulated industries
- GDPR data minimization for technician assignment data
