# F-2.3.17 - Root Cause Analysis

**Requirement ID:** F-2.3.17  
**Module:** Maintenance Management  
**Priority:** Medium

---

## Requirement Description

Provide structured root cause analysis tools that guide technicians through systematic failure investigation using methods like 5-Why analysis and fishbone diagrams, capturing learnings in a knowledge base to prevent recurrence. Guided 5-Why analysis workflow, fishbone (Ishikawa) diagram template, failure pattern recognition and suggestions, knowledge base of known issues and solutions, corrective action tracking and verification, link to similar historical failures, analysis approval workflow, and recurrence tracking and metrics.

---

## Business Rules

1. 5-Why analysis requires asking "why" at least 5 times until fundamental root cause found
2. Fishbone diagram uses 6M categories: Machine, Method, Material, Man, Measurement, Maintenance
3. Corrective actions require owner and due date
4. Similar failure patterns suggest existing knowledge base articles
5. Recurrence tracking flags repeated failures for special attention
6. Corrective action status: Open, In Progress, Completed, Verified, Closed

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Work Order | Yes | Completed work order to analyze |
| Asset | Yes | Asset that failed |
| Failure Symptom | Yes | Observable failure description |
| 5-Why Answers | Yes | Sequential why questions and answers |
| Fishbone Causes | No | Causes organized by category |
| Corrective Actions | Yes | Actions to prevent recurrence |
| Knowledge Article | No | Article summarizing the failure and solution |

---

## Output Data

| Data | Description |
|------|-------------|
| RCA Report | Complete analysis documentation |
| Corrective Actions | Tracked actions with status |
| Knowledge Base Article | Published article for future reference |

---

## Validation Rules

1. Work order must be completed before RCA
2. Root cause requires supervisor approval
3. Corrective action must have owner and due date
4. Knowledge entry must follow approved template

---

## Access Control

| Role | Permission |
|------|------------|
| Technician | Create RCA analyses |
| Supervisor | Approve root causes |
| Reliability Engineer | Manage knowledge base |

- **Isolation**: Company-level data isolation maintained |
