### 1.1 Requirement Description

Provide structured root cause analysis tools that guide technicians through systematic failure investigation using methods like 5-Why analysis and fishbone diagrams, capturing learnings in a knowledge base to prevent recurrence. Guided 5-Why analysis workflow, fishbone (Ishikawa) diagram template, failure pattern recognition and suggestions, knowledge base of known issues and solutions, corrective action tracking and verification, link to similar historical failures, analysis approval workflow, and recurrence tracking and metrics.

### 3.3 Business Logic

**5-Why Analysis Flow:**

```
START
  │
  ▼
[Document Failure Symptom]
  │
  ▼
[Why #1: Ask why failure occurred]
  │
  ├─► If Root Cause Found ──► [Is it fundamental?]
  │                              ├─► YES ──► END (Root Cause)
  │                              └─► NO ──► Continue
  │
  ▼
[Why #2: Ask why that cause exists]
  │
  ▼
[Why #3: Ask why that cause exists]
  │
  ▼
[Why #4: Ask why that cause exists]
  │
  ▼
[Why #5: Ask why that cause exists]
  │
  ▼
[Identify Root Cause]
  │
  ▼
[Propose Corrective Actions]
  │
  ▼
   END
```

**Fishbone Categories (6M):**

```
Machine / Equipment
Method / Process
Material
Man / People
Measurement
Maintenance / Environment
```

**Corrective Action Tracking:**

| Status | Description |
|--------|-------------|
| OPEN | Action assigned, not started |
| IN_PROGRESS | Work started |
| COMPLETED | Action verified complete |
| VERIFIED | Effectiveness confirmed |
| CLOSED | RCA closed with CA |
