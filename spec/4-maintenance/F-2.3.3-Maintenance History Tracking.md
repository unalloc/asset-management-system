### 1.1 Requirement Description

Record and query complete maintenance history for assets to analyze maintenance patterns and costs. All maintenance activities are recorded automatically, history records cannot be deleted (can be marked invalid), support maintenance type classification statistics, cumulative maintenance cost calculation, failure mode recording and analysis, and maintenance effectiveness evaluation.

| POST-002 | Asset statistics updated | MTBF, cumulative cost recalculated | Query AssetStatistics |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
[Work Order Completed + Approved]
  │
  ▼
[Validate Work Order Status]
  │
  ▼
[Calculate Actual Cost]
  │  ActualCost = LaborCost + PartsCost + ExternalServiceCost
  │
  ▼
[Classify Maintenance Type]
  │
  ▼
[Record Failure Data (if applicable)]
  │
  ▼
[Calculate MTBF/MTTR]
  │
  ▼
[Update Asset Statistics]
  │
  ▼
[Link to RCA (if Failure)]
  │
  ▼
   END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Auto History Creation | WO.Status = 'Completed' AND Approved = TRUE | Create MaintenanceHistory | 1 |
| BR-002 | Cumulative Cost Calculation | History created | SUM(ActualCost) by AssetID | 1 |
| BR-003 | MTBF Calculation | History created | MeanTimeBetweenFailures calculation | 2 |
| BR-004 | MTTR Calculation | History created | MeanTimeToRepair calculation | 2 |
| BR-005 | Failure Mode Classification | Result = 'Failure' | Require failure mode code | 1 |

**Calculation Logic:**

```
MTBF (Mean Time Between Failures):
  MTBF = SUM(Uptime) / FailureCount
  Where:
    Uptime = TimeSinceLastFailure (for operational assets)
    FailureCount = COUNT(History WHERE Result = 'Failure' AND MaintenanceType = 'Corrective')

MTTR (Mean Time To Repair):
  MTTR = SUM(ActualHours) / COUNT(History WHERE MaintenanceType IN ('Corrective', 'Preventive'))

Cumulative Maintenance Cost:
  CumCost = SUM(ActualCost) WHERE AssetID = {AssetID} AND Status = 'Active'
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | First maintenance for asset (no baseline) | Calculate initial stats, flag for baseline | Set BaselineFlag = TRUE |
| EC-002 | Partial failure with multiple causes | Allow multiple failure mode codes | Create one record per failure mode |
| EC-003 | Maintenance by external vendor | Record vendor ID, invoice reference | Add VendorID, InvoiceRef fields |
| EC-004 | History record marked invalid | Keep record but exclude from statistics | Set InvalidatedFlag = TRUE, ExcludeFromStats = TRUE |
