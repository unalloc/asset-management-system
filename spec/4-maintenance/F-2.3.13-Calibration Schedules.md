### 1.1 Requirement Description

Manage calibration schedules, execution, and certificate tracking for measurement and test equipment to ensure measurement accuracy. Includes calibration cycle management (based on usage frequency or time), calibration agency qualification management, electronic calibration certificate management, calibration result analysis and trend tracking, overdue alerts and lockout functionality, and measurement uncertainty management.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Calibration cycle must comply with regulations | Cycle meets minimum regulatory requirement | CAL-001 | "Calibration cycle does not meet regulatory requirements" |
| PRE-002 | Calibration agency must be qualified | Agency.QualificationStatus = 'Qualified' | CAL-002 | "Calibration agency is not qualified" |
| PRE-003 | Calibration date must be reasonable | CalibrationDate <= CURRENT_DATE | CAL-003 | "Calibration date cannot be in the future" |
| PRE-004 | Measurement uncertainty must be defined | UncertaintyValue IS NOT NULL | CAL-004 | "Measurement uncertainty must be recorded" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Calibration due alert created | Alert scheduled for AdvanceNoticeDays before due |
| POST-002 | Lockout triggered (if overdue) | Equipment.LockoutStatus = TRUE if overdue |

### 3.3 Business Logic

**Calibration Status:**

| Status | Condition | Action |
|--------|-----------|--------|
| VALID | CurrentDate < DueDate | Normal operation |
| EXPIRING | CurrentDate >= DueDate - AdvanceNoticeDays | Alert, schedule calibration |
| EXPIRED | CurrentDate >= DueDate | Lockout (if enabled), immediate action required |

**As-Found/As-Left Logic:**

```
IsInTolerance = ABS(AsFound - Standard) <= Tolerance
AsLeft = AsFound (if in tolerance) OR AdjustedValue (if adjusted)
```
