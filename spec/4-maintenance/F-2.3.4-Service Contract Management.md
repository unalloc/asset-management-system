# F-2.3.4 - Service Contract Management

**Requirement ID:** F-2.3.4  
**Module:** Maintenance Management  
**Priority:** Medium

---

## Requirement Description

Manage maintenance contracts with external service providers, tracking contract execution and expenses. Includes contract basic information management (number, period, amount), service scope and SLA terms, payment plan and actual payment records, contract changes and renewal management, vendor performance evaluation, and contract expiration warnings.

---

## Business Rules

1. Contract numbers must be unique within a company
2. Contract end date must be after start date
3. Payment plan installments must sum to the total contract amount
4. Expiration notifications scheduled based on advance notice days
5. Vendor performance score calculated from SLA compliance
6. Renewal recommendations generated 90 days before expiration
7. Active vendor status required for contract creation

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Vendor | Yes | External service provider |
| Contract Number | Yes | Unique identifier for the contract |
| Contract Type | Yes | Type of service contract |
| Start Date | Yes | Contract effective date |
| End Date | Yes | Contract expiration date |
| Contract Amount | Yes | Total value of contract |
| Currency | Yes | Payment currency code |
| Payment Plan | No | Installment schedule |
| SLA Terms | No | Service level agreement details |

---

## Output Data

| Data | Description |
|------|-------------|
| Service Contract | Created contract record |
| Expiration Notification | Scheduled alert before contract expires |
| Vendor Performance Record | Calculated performance metrics |

---

## Validation Rules

1. Contract number must be unique per company
2. End date must be after start date
3. Contract amount must be positive
4. Vendor must exist and be active
5. Payment plan total must equal contract amount

---

## Access Control

| Role | Permission |
|------|------------|
| Company Admin | Create and modify contracts |
| Procurement | View and manage contracts |
| Division Manager | View within division |
| Accounting | Record payments |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Contract creation restricted to Company level
- **Isolation**: Each company sees only its own contracts |

---

## Compliance Requirements

- SOX contract approval workflow and payment tracking
- IFRS 16 service contracts distinguished from leases
- GDPR vendor contact data handled under legitimate interest
- GAMP 5 qualified supplier documentation
