### 1.1 Requirement Description

This cross-cutting requirement establishes the comprehensive data security and privacy framework for the entire asset management system. It applies to all modules and defines security controls, encryption requirements, access management, privacy by design principles, data retention policies, security monitoring, and third-party risk management aligned with Western market standards (US/EU).

## 4. Encryption Requirements

### 4.1 Data at Rest

| Requirement | Specification | Implementation |
|-------------|---------------|----------------|
| **Algorithm** | AES-256-GCM | Database-level encryption |
| **Key Length** | 256 bits minimum | Hardware security module (HSM) |
| **Mode** | GCM for authenticated encryption | Transparent data encryption (TDE) |
| **Key Rotation** | 365 days maximum | Automated rotation with re-encryption |
| **Scope** | All persistent storage | Database, file systems, backups |

**Encryption at Rest Controls:**

| Data Type | Classification | Encryption Required | Algorithm |
|-----------|----------------|--------------------| ----------|
| Database Data | RESTRICTED | ✓ Yes | AES-256-GCM |
| Database Data | CONFIDENTIAL | ✓ Yes | AES-256-GCM |
| Database Data | INTERNAL | Recommended | AES-128-GCM |
| File Attachments | RESTRICTED | ✓ Yes | AES-256-GCM |
| File Attachments | CONFIDENTIAL | ✓ Yes | AES-256-GCM |
| Backup Media | ALL | ✓ Yes | AES-256-GCM |
| Log Files | CONFIDENTIAL+ | ✓ Yes | AES-256-GCM |
| Configuration Files | RESTRICTED | ✓ Yes | AES-256-GCM |

### 4.2 Data in Transit

| Requirement | Specification | Implementation |
|-------------|---------------|----------------|
| **Protocol** | TLS 1.3 | All external communications |
| **TLS 1.2** | Allowed for backward compatibility | Internal services only |
| **SSL v2/v3** | Prohibited | Immediately disabled |
| **TLS 1.0/1.1** | Prohibited | Migration required |
| **Cipher Suites** | TLS_AES_256_GCM_SHA384, TLS_AES_128_GCM_SHA256 | Strong ciphers only |
| **Certificate Type** | X.509 v3 | Signed by approved CA |
| **Key Length** | RSA 2048-bit minimum, ECDSA P-256 | Certificate requirements |
| **HSTS** | Required for web applications | Strict-Transport-Security header |

**TLS Configuration:**

```
# Required TLS Settings
- Protocol Version: TLS 1.3 (TLS 1.2 minimum for internal)
- Cipher Suites: TLS_AES_256_GCM_SHA384, TLS_CHACHA20_POLY1305_SHA256, TLS_AES_128_GCM_SHA256
- Certificate Validation: Full chain validation required
- Hostname Verification: Required for all HTTPS connections
- Certificate Pinning: Required for mobile applications
```
