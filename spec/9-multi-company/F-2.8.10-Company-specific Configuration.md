# F-2.8.10 - Company-specific Configuration

## Requirement ID
F-2.8.10

## Module
Multi-Company Module

## Priority
Medium

## Requirement Description
Support each subsidiary to customize based on group standards, including business processes, approval permissions, report formats, language and currency, balancing standardization and flexibility.

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Inheritance Priority | Any configuration | Group > Region > Country > Company | 1 |
| BR-002 | Allowed Override Range | Company modification | Must be within min/max from parent | 1 |
| BR-003 | Configuration Version | Any change | Create new version | 1 |
| BR-004 | No Core Override | Group core principles | Cannot be overridden at company level | 1 |

**Configuration Types:**

| Type | Examples | Override Allowed |
|------|---------|------------------|
| Business Processes | Approval workflows, notification rules | ✓ Yes |
| Report Formats | Local currency, date format, language | ✓ Yes |
| Asset Categories | Local asset type taxonomy | ✓ Yes |
| Depreciation Rules | Local useful life, methods | ✓ Yes |
| Core Principles | SOX controls, GDPR requirements | ☐ No |

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Group configuration defined | GroupConfig NOT NULL | CC-001 | "Group configuration not defined" |
| PRE-002 | Company exists | CompanyCode VALID | CC-002 | "Company not found" |
| PRE-003 | User has configuration rights | User.Role IN ('GROUP_CONFIG_ADMIN', 'COMPANY_ADMIN', 'SYSTEM_ADMIN') | CC-003 | "Insufficient configuration rights" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Company config effective | Config values override group defaults |
| POST-002 | Inheritance chain validated | No circular inheritance |
| POST-003 | Change audit logged | Configuration change tracked |

## Access Control

| Role | Group Config | Company Config | Modify Core |
|------|---------------|----------------|-------------|
| Group Config Admin | ✓ Full | ✓ Full | ✓ |
| Company Admin | ☐ | ✓ Own | ☐ |
| Department User | ☐ | View | ☐ |

## Multi-Company Data Scope
Group Config Admin manages group defaults. Company Admin manages company-specific overrides. Core principles cannot be overridden at company level.
