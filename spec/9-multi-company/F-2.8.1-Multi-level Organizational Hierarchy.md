### 1.1 Requirement Description

Manage multi-level group organizational structure, including group → holding company → subsidiary → business unit → department multi-level structure, supporting flexible organizational structure adjustment and mapping.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Group organizational structure information available | Organization.OrgLevel = 'GROUP' AND Organization.Status = 'Active' | ORG-001 | "No active group organization exists. Please create group organization first." |
| PRE-002 | Organization type definition clear | OrganizationType IS NOT NULL AND OrganizationType IN ('LEGAL_ENTITY', 'BUSINESS_UNIT', 'DEPARTMENT', 'TEAM', 'VIRTUAL') | ORG-002 | "Invalid organization type. Must be one of: LEGAL_ENTITY, BUSINESS_UNIT, DEPARTMENT, TEAM, VIRTUAL" |
| PRE-003 | User has "Organizational Structure Management" permission | User.RolePermissions CONTAINS 'ORG_ADMIN' OR User.RolePermissions CONTAINS 'SYSTEM_ADMIN' | ORG-003 | "User does not have organizational structure management permission" |
| PRE-004 | Parent organization exists (except root) | IF ParentOrgCode IS NOT NULL THEN EXISTS(Organization WHERE OrganizationCode = ParentOrgCode) | ORG-004 | "Parent organization does not exist. Please select a valid parent organization." |
| PRE-005 | Integration system interfaces available | IF RequiresIntegration = TRUE THEN ExternalSystemAPI.Status = 'AVAILABLE' | ORG-005 | "Integration system unavailable. Organizational structure sync cannot proceed." |

**Pre-condition Validation Matrix:**

```
IF Action = 'CREATE_ROOT'
   THEN validate PRE-001 AND PRE-002 AND PRE-003
   ELSE validate PRE-001 AND PRE-002 AND PRE-003 AND PRE-004 AND PRE-005

IF OrganizationType = 'LEGAL_ENTITY'
   THEN require TaxRegistrationNumber IS NOT NULL
        AND require JurisdictionCode IS NOT NULL
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Is this creating a ROOT organization?
  ├─► YES ─► Validate Group-level requirements
  │         └─► Create Organization with OrgLevel='GROUP'
  └─► NO
       │
       ▼
  Is Parent Organization valid?
       ├─► NO ─► Return ORG-004 Error
       └─► YES
            │
            ▼
       Is OrganizationType = 'LEGAL_ENTITY'?
            ├─► YES ─► Validate Tax Registration AND Jurisdiction
            │         └─► Create Legal Entity Organization
            └─► NO
                 │
                 ▼
            Is OrganizationType = 'BUSINESS_UNIT'?
                 ├─► YES ─► Validate Parent is LEGAL_ENTITY
                 │         └─► Create Business Unit Organization
                 └─► NO
                      │
                      ▼
                 Create Department/Team/Virtual Organization
                      │
                      ▼
            Validate No Circular Dependency
                      │
                      ▼
                 Does validation pass?
                      ├─► NO ─► Return ORG-006 (Circular Dependency Error)
                      └─► YES
                           │
                           ▼
                      Create Audit Log Entry
                           │
                              ▼
                          END (Success)
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Root Organization Single Instance | OrganizationLevel = 'GROUP' | THROW ERROR if more than one GROUP-level organization exists | 1 |
| BR-002 | Legal Entity Under Group Only | OrganizationType = 'LEGAL_ENTITY' | REQUIRE ParentOrgLevel = 'GROUP' | 1 |
| BR-003 | Business Unit Parent Validation | OrganizationType = 'BUSINESS_UNIT' | REQUIRE ParentOrgLevel IN ('GROUP', 'LEGAL_ENTITY', 'BUSINESS_UNIT') | 2 |
| BR-004 | Department Depth Limit | OrganizationType = 'DEPARTMENT' | REQUIRE depth ≤ 8 levels from GROUP | 3 |
| BR-005 | Circular Dependency Prevention | Any organization creation/update | REQUIRE cycle detection using CONNECT BY PRIOR | 1 |
| BR-006 | Code Uniqueness | OrganizationCode | REQUIRE unique across entire organization (no company code prefix conflicts) | 1 |
| BR-007 | Effective Date Validation | Organization.EffectiveDate | REQUIRE EffectiveDate >= CURRENT_DATE for new orgs; allow past dates for corrections | 2 |
| BR-008 | Historical Tracking | Organization change event | REQUIRE creating new record with previous snapshot before modification | 1 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Creating organization with future effective date | System allows creation but org is inactive until effective date | Set status='PENDING' until EffectiveDate reached |
| EC-002 | Parent organization in different country | System allows but requires cross-border validation | Validate regulatory requirements for both jurisdictions |
| EC-003 | Organization merge/split scenario | System creates successor orgs with full history linkage | Create new orgs, mark old as 'MERGED' or 'SPLIT', preserve history |
| EC-004 | Maximum hierarchy depth exceeded | System rejects creation at depth > 8 | Return ORG-007 with depth limit message |
| EC-005 | Deleting organization with child orgs | System prevents deletion | Return ORG-008 "Cannot delete organization with child organizations" |
| EC-006 | Multiple root organizations attempted | System prevents second root | Return ORG-009 "Only one root organization allowed" |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| ParentExists | IsRootOrg | IsLegalEntity | Action |
| FALSE | * | * | ORG-004 Error |
| TRUE | TRUE | * | ORG-009 Error (cannot have parent for root) |
| TRUE | FALSE | TRUE | Validate Parent=GROUP, proceed |
| TRUE | FALSE | FALSE | Validate parent hierarchy, proceed |

| Current State | Event | Next State | Guard Conditions | Side Effects |

| RESTRUCTURED | Restructure complete | ACTIVE | New hierarchy validated | Old hierarchy snapshot archived |

**Access Control Implementation Rules:**
- Users can only view organizations within their organizational hierarchy (descendants of their assigned organization)
- Modification rights require explicit permission at the data's scope level
- Cross-company organization management requires Group Org Admin role
- Department and Team level users cannot modify organization structure
- All organization access is logged for audit purposes

### 5.3 Data Isolation

| Isolation Requirement | Implementation | Verification Method |
|----------------------|----------------|-------------------|
| Company A cannot view Company B organizations | Organization.CompanyCode filtering + row-level security | Automated test: attempt cross-company access returns empty |
| Subsidiary cannot view parent company structure | Hierarchical permission model | Role-based query filtering |
| Legal entity data separated from operational data | Organization.Type filtering | Database query verification |
| Cross-company org view requires explicit permission | Group-level role check | Permission assignment audit |

**Data Segregation Strategy:**

```
Database Level:
  - Organization table includes CompanyCode column
  - All queries automatically filtered by CompanyCode context
  - Row-level security (RLS) policies enforce CompanyCode filtering

Application Level:
  - Session context contains UserCompanyCode
  - Organization queries inject CompanyCode filter
  - User cannot access API endpoints that expose other company orgs

API Level:
  - Organization endpoints require company-scoped JWT claims
  - Cross-company access requires special Group-level token
  - Integration endpoints validate company context
```
