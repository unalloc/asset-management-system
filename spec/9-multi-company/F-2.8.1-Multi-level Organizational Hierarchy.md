# F-2.8.1 - Multi-level Organizational Hierarchy

## Requirement ID
F-2.8.1

## Module
Multi-Company Module

## Priority
High

## Requirement Description
Manage multi-level group organizational structure, including group → holding company → subsidiary → business unit → department multi-level structure, supporting flexible organizational structure adjustment and mapping.

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Root Organization Single Instance | OrganizationLevel = 'GROUP' | THROW ERROR if more than one GROUP-level organization exists | 1 |
| BR-002 | Legal Entity Under Group Only | OrganizationType = 'LEGAL_ENTITY' | REQUIRE ParentOrgLevel = 'GROUP' | 1 |
| BR-003 | Business Unit Parent Validation | OrganizationType = 'BUSINESS_UNIT' | REQUIRE ParentOrgLevel IN ('GROUP', 'LEGAL_ENTITY', 'BUSINESS_UNIT') | 2 |
| BR-004 | Department Depth Limit | OrganizationType = 'DEPARTMENT' | REQUIRE depth ≤ 8 levels from GROUP | 3 |
| BR-005 | Circular Dependency Prevention | Any organization creation/update | REQUIRE cycle detection using CONNECT BY PRIOR | 1 |
| BR-006 | Code Uniqueness | OrganizationCode | REQUIRE unique across entire organization | 1 |
| BR-007 | Effective Date Validation | Organization.EffectiveDate | REQUIRE EffectiveDate >= CURRENT_DATE for new orgs | 2 |
| BR-008 | Historical Tracking | Organization change event | REQUIRE creating new record with previous snapshot before modification | 1 |

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Group organizational structure exists | Organization.OrgLevel = 'GROUP' AND Organization.Status = 'Active' | ORG-001 | "No active group organization exists. Please create group organization first." |
| PRE-002 | Organization type definition clear | OrganizationType IN ('LEGAL_ENTITY', 'BUSINESS_UNIT', 'DEPARTMENT', 'TEAM', 'VIRTUAL') | ORG-002 | "Invalid organization type" |
| PRE-003 | User has organizational management permission | User.RolePermissions CONTAINS 'ORG_ADMIN' OR 'SYSTEM_ADMIN' | ORG-003 | "User does not have organizational structure management permission" |
| PRE-004 | Parent organization exists (except root) | IF ParentOrgCode IS NOT NULL THEN EXISTS(Organization) | ORG-004 | "Parent organization does not exist" |
| PRE-005 | Integration system available | IF RequiresIntegration = TRUE THEN ExternalSystemAPI.Status = 'AVAILABLE' | ORG-005 | "Integration system unavailable" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Organizational structure data entered | Organization record created with all required fields |
| POST-002 | Hierarchy relationships established | Parent-Child relationships created, no cycles |
| POST-003 | Permission inheritance effective | Permission inheritance configuration active |
| POST-004 | Integration mapping completed | ExternalSystemID populated AND sync_status = 'SYNCED' |

**Edge Cases:**

| Edge Case ID | Scenario | Expected Behavior |
|--------------|----------|-------------------|
| EC-001 | Creating org with future effective date | System allows but org is inactive until effective date |
| EC-002 | Parent org in different country | System allows but requires cross-border validation |
| EC-003 | Organization merge/split | System creates successor orgs with full history linkage |
| EC-004 | Maximum hierarchy depth exceeded | System rejects creation at depth > 8 |
| EC-005 | Deleting org with child orgs | System prevents deletion |
| EC-006 | Multiple root organizations attempted | System prevents second root |

## Multi-Company Data Scope
Group → Company (Legal Entity) → Division (Business Unit) → Plant (Operations) → Department → Team hierarchy. Company A cannot view Company B organizations. Cross-company org view requires Group-level role.
