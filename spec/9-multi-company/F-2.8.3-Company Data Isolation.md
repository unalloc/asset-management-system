### 1.1 Requirement Description

Implement company-based data isolation to ensure each company's asset data, financial data, and operational data are independent and secure, while supporting group-level consolidated views.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Company organizational structure defined | CompanyCode IS NOT NULL | DI-001 | "Company code required for data isolation" |
| PRE-002 | User company affiliation assigned | User.AssignedCompanyCode IS NOT NULL | DI-002 | "User must be assigned to a company" |
| PRE-003 | Data isolation strategy configured | IsolationStrategy IN ('STRICT', 'LOOSE') | DI-003 | "Isolation strategy must be defined" |
| PRE-004 | Audit log system available | AuditLogSystem.Status = 'AVAILABLE' | DI-004 | "Audit log system unavailable" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Data queries filtered by company | Query results match user's company scope | Test cross-company query |
| POST-002 | Permission checks effective | Unauthorized access blocked | Penetration test |
| POST-003 | Audit log records created | Cross-company access logged | Audit log query |
| POST-004 | Group view available for authorized | Group users see all company data | Group user test |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
User initiates data access request
  │
  ▼
Extract User.AssignedCompanyCode from session
  │
  ▼
Is User.Role IN ('SYSTEM_ADMIN', 'GROUP_ORG_ADMIN')?
  ├─► YES ─► Allow full access (bypass company filter)
  └─► NO
       │
       ▼
  Inject CompanyCode filter into query
       │
       ▼
  Execute query with filtered results
       │
       ▼
  Log access attempt to audit log
       │
       ▼
  Return filtered results
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Company Context Required | Any data access | REQUIRE User.AssignedCompanyCode NOT NULL | 1 |
| BR-002 | Strict Isolation Mode | IsolationStrategy = 'STRICT' | REQUIRE CompanyCode match for all data access | 1 |
| BR-003 | Loose Isolation with Override | IsolationStrategy = 'LOOSE' | Allow cross-company with EXPLICIT_GRANT only | 2 |
| BR-004 | Cross-Company Access Logging | Any cross-company access attempt | ALWAYS log to audit with reason | 1 |
| BR-005 | Data Export Company Filter | Data export operation | REQUIRE export filtered by company scope | 1 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | User not assigned to company | Block all data access | Return DI-002 error |
| EC-002 | User transfers between companies | Immediate access change to new company | Invalidate session cache |
| EC-003 | Company being liquidated | Gradually restrict access | Status-based filtering |
| EC-004 | Integration user without company | Use integration company context | Validate integration account |

### 3.5 State Transitions

**Isolation Modes:**

```
STRICT MODE: Company A ──✗──> Company B Data (always blocked)
LOOSE MODE:  Company A ──✓──> Company B Data (if explicit grant)
GROUP MODE:  Group User ──✓──> All Company Data (full access)
```

## 5. Multi-Company Data Hierarchy

### 5.1 Data Scope Level

| Dimension | Level | Creates Data | Views Data | Modifies Data |
|-----------|-------|--------------|------------|---------------|
| **Legal Entity** | Company | ✓ | ✓ | ✓ |
| **Business Unit** | Division | Read Only | Read Only | ☐ |
| **Operations** | Plant | ☐ | ☐ | ☐ |
| **Cost Center** | Department | ☐ | ☐ | ☐ |

**Data Creation Authority Matrix:**

```
                         │ Same Co │ Other Co (Grant) │ Group │
─────────────────────────┼─────────┼──────────────────┼───────┤
Read Data                │    X    │        X         │   X   │
Write Own Data           │    X    │        ☐         │   X   │
Write Other Data          │    ☐    │        ☐         │   X   │
Export Data              │    X    │        ☐         │   X   │
Create Access Grant      │    ☐    │        ☐         │   X   │
```

### 5.2 Access Control Rules

| Role | Same Company | Same Division | Same Plant | All Companies |
|------|--------------|---------------|------------|---------------|
| System Admin | ✓ Full | ✓ Full | ✓ Full | ✓ Full |
| Company Admin | ✓ Full | ☐ | ☐ | Read Only |
| Division Manager | ✓ Full | Read Only | ☐ | ☐ |
| Plant Manager | ✓ Full | ☐ | Read Only | ☐ |
| Department User | ✓ Full | ☐ | ☐ | ☐ |

**Access Control Implementation Rules:**
- All queries automatically filtered by User.AssignedCompanyCode
- Cross-company access requires explicit DataAccessGrant
- GROUP-level roles bypass company filtering
- Audit log records all access attempts (success and blocked)

### 5.3 Data Isolation

| Isolation Requirement | Implementation | Verification Method |
|----------------------|----------------|-------------------|
| Company A cannot view Company B assets | CompanyCode column + RLS policy | Automated test: query returns empty |
| Financial data per company | Separate accounting books per entity | Query verification |
| User can only see assigned company data | Session context + WHERE clause injection | Security audit |
| Cross-company access logged | AuditLog with cross-company flag | Log analysis |

**Data Segregation Strategy:**

```
Database Level:
  - Row-Level Security (RLS) policies
  - CompanyCode column on all data tables
  - Database user = application user (for auditing)

Application Level:
  - CompanyContextFilter on all queries
  - Session-based company assignment
  - Automatic WHERE clause injection

API Level:
  - Company scope validation on all endpoints
  - JWT claims contain CompanyCode
  - Response filtering based on scope
```

### 5.4 Consolidation

|-------------------|-----------|--------|
