# F-2.8.13 Support Different Company Types

## Function ID
F-2.8.13

## Function Name
Multi-company Type Support

## Function Description
Support different types of company entities, including wholly-owned subsidiaries, controlling subsidiaries, minority-owned companies, joint ventures, and branch offices, with different management requirements and integration levels for each type.

## Business Rules
1. Company type definition: Wholly-owned subsidiary (100% ownership), controlling subsidiary (>50% ownership), minority-owned company (<50% ownership), joint venture (joint control), branch office (non-legal entity)
2. Management differences: Control level (operational control, financial control), data sharing level, reporting requirements, integration depth
3. Ownership management: Ownership ratio tracking, investment cost, investment income, impairment testing
4. Consolidation methods: Full consolidation, proportionate consolidation, equity method, cost method
5. Compliance requirements: Statutory reporting requirements and disclosure requirements for different company types
6. Business processes: Business process differences for different company types (approval authority, policy application)
7. System integration: System integration depth with different company types (full integration, partial integration, interface integration)
8. Type conversion: Process and data migration for company type changes (e.g., minority to controlling)

## Preconditions
- Company type classification clear
- Management requirements for each type defined
- Ownership data accurate
- Compliance requirements clear

## Postconditions
- Company type configuration effective
- Differentiated management implemented
- Ownership management set
- Consolidation method determined

## Input Data
1. Company ID and type
2. Management difference rules
3. Ownership ratio data
4. Consolidation method selection
5. Compliance requirement mapping
6. Business process differences
7. System integration level
8. Type conversion rules

## Output Data
1. Company type configuration
2. Differentiated management implementation report
3. Ownership management records
4. Consolidation method application
5. Compliance check results
6. Business process adaptation
7. System integration status
8. Type conversion history

## Validation Rules
1. Company type must be correctly defined
2. Management differences must be reasonable
3. Ownership ratios must be accurate
4. Consolidation method must comply with accounting standards
5. Compliance requirements must be met

## Error Handling
1. Type definition error: Correct company type and related settings
2. Management difference conflict: Coordinate and resolve conflicts
3. Ownership data inconsistency: Verify and correct data
4. Consolidation method inappropriate: Accounting department adjusts

## Priority
Medium
