# F-2.8.21 - Audit and Compliance Framework

## Requirement ID
F-2.8.21

## Module
Multi-Company Module

## Priority
Critical

## Requirement Description
Define the audit trail and compliance framework that ensures all asset management activities are properly logged, traceable, and compliant with US/EU regulatory requirements.

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Complete Logging | Any scope event | ALWAYS write audit record | 1 |
| BR-002 | Immutability | Any audit record | Prevent UPDATE/DELETE on audit logs | 1 |
| BR-003 | Segregation of Duties | Financial transactions | Require dual approval | 1 |
| BR-004 | Personal Data Access Logging | Any personal data access | Log with user identification | 1 |
| BR-005 | Retention Enforcement | Audit record age > retention | Archive then delete per policy | 1 |

**Audit Event Types:**

| Event Type | Description | Required Data |
|------------|-------------|--------------|
| CREATE | New record created | User, timestamp, new values, reason |
| UPDATE | Record modified | User, timestamp, old values, new values, changed fields |
| DELETE | Record removed | User, timestamp, all values before deletion, reason |
| READ | Sensitive data accessed | User, timestamp, data accessed |

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Audit system operational | AuditSystem.Status = 'AVAILABLE' | ACF-001 | "Audit system unavailable" |
| PRE-002 | Audit scope defined | AuditEvents NOT EMPTY | ACF-002 | "Audit events not defined" |
| PRE-003 | Retention policy defined | RetentionPeriod > 0 | ACF-003 | "Retention policy not set" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Audit logs created | All tracked events logged |
| POST-002 | Audit log integrity verified | Daily integrity check passes |
| POST-003 | Retention enforced | Old logs archived/deleted per policy |

## Input Data

| Field Name | Data Type | Format | Required | Description |
|------------|-----------|--------|----------|-------------|
| AuditLogID | UUID | - | Yes | Audit log primary key |
| UserID | UUID | - | Yes | User who performed action |
| Username | VARCHAR(100) | - | Yes | Human-readable user name |
| Timestamp | DATETIME | ISO 8601 UTC | Yes | When action occurred |
| Action | ENUM | - | Yes | CREATE, UPDATE, DELETE, READ |
| EntityType | VARCHAR(50) | - | Yes | Type of asset affected |
| EntityID | UUID | - | Yes | Specific record affected |

| NewValues | JSON | - | No | New values (CREATE/UPDATE) |
| ChangedFields | ARRAY | - | No | List of modified fields |
| Reason | TEXT | - | No | Business justification |
| SessionID | VARCHAR(100) | - | Yes | Session identifier |
| IPAddress | VARCHAR(45) | - | Yes | Where action occurred |

## Access Control

| Role | View Audit Logs | Create Audit | Modify Audit | Export |
|------|----------------|--------------|--------------|--------|
| System Admin | ✓ | ☐ | ☐ | ✓ |
| Group Auditor | ✓ All | ☐ | ☐ | ✓ |
| Company Auditor | ✓ Own Company | ☐ | ☐ | ✓ |
| External Auditor | ✓ (read-only) | ☐ | ☐ | ✓ (limited) |

## Multi-Company Data Scope
Audit logs created at appropriate organizational level. Group Auditors can view all audit logs. Company Auditors can only view their own company's audit logs. Audit logs are immutable and cannot be modified by any user.

## Compliance Requirements

| Regulation | Applies | Compliance Status | Evidence/Mapping |
|------------|---------|-------------------|------------------|

| GDPR (EU) | Yes | Compliant | Article 5(2) accountability, Article 30 RoPA, Article 32 security |
| ISO 55000 | Yes | Compliant | Clause 7.5 documented information |
| ISO 55001 | Yes | Compliant | Audit trail requirements |
