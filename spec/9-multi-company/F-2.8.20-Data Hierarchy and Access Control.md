# F-2.8.20 - Data Hierarchy and Access Control

## Requirement ID
F-2.8.20

## Module
Multi-Company Module

## Priority
Critical

## Requirement Description
Define the organizational hierarchy structure and access control framework that governs how data is scoped, accessed, and managed across the multi-company asset management system.

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Hierarchical Data Filter | Any data query | Inject organizational scope filter | 1 |
| BR-002 | Company Code Required | All asset queries | REQUIRE CompanyCode in all queries | 1 |
| BR-003 | Cross-Company Agreement | Inter-company access | REQUIRE explicit inter-company agreement | 1 |
| BR-004 | Segregation of Duties | Approval actions | Asset owner cannot approve own transactions | 1 |

**Hierarchy Levels:**

| Level | Name | Description | Mandatory |
|-------|------|-------------|-----------|
| 1 | Group | Ultimate parent organization | No |
| 2 | Company | Legal entity with corporate registration | Yes |
| 3 | Division | Business unit or operating segment | No |
| 4 | Plant | Physical facility or manufacturing site | Yes |
| 5 | Department | Operational unit within plant | No |
| 6 | Cost Center | Financial accounting unit | Yes |

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | User assigned to organization | User.OrganizationID IS NOT NULL | DH-001 | "User must be assigned to organization" |
| PRE-002 | User has valid role | User.Role IS NOT NULL | DH-002 | "User must have valid role" |
| PRE-003 | Organization hierarchy exists | OrganizationHierarchy EXISTS | DH-003 | "Organization hierarchy must be configured" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-001 | Data scope enforced | All queries filtered by organizational scope |
| POST-002 | Access audit logged | All access attempts logged |
| POST-003 | Hierarchical inheritance | Parent data accessible to child users |

## Access Control

| Role | Group | Company | Division | Plant | Department |
|------|-------|---------|---------|-------|------------|
| Group Administrator | Full | Full | Full | Full | Full |
| Company Administrator | Read | Full | Full | Full | Full |
| Division Administrator | Read | Read | Full | Full | Full |
| Plant Administrator | Read | Read | Read | Full | Full |
| Department Administrator | Read | Read | Read | Read | Full |
| Technician | Read | Read | Read | Read (assigned) | Read (assigned) |
| Viewer | Read | Read | Read | Read | Read |

## Multi-Company Data Scope
All data queries filtered by user's organizational scope. Company Code mandatory in all asset queries. Cross-company access requires explicit inter-company agreement. Data inherits from parent organizational levels.
