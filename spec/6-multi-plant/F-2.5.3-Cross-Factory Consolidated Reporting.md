# F-2.5.3 - Cross-Factory Consolidated Reporting

**Requirement ID:** F-2.5.3  
**Module:** Multi-Plant Module  
**Priority:** High

---

## Requirement Description

Generate consolidated reports across multiple factories, supporting drill-down from group level to specific factory details. Reports must support currency conversion, data validation, and audit trail requirements for SOX and IFRS compliance. Implements consolidation rules for the multi-company data hierarchy established in F-2.5.1.

---

## Business Rules

1. Consolidated reports aggregate data from multiple organizations in the hierarchy
2. Currency conversion uses average exchange rate for the reporting period
3. Data completeness is validated before report generation
4. Abnormal data (outliers) are flagged for manual review
5. Intercompany transactions are eliminated when both source and destination are in same Company
6. Report snapshot is stored for audit trail
7. Historical reports can be regenerated but original is preserved

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| ReportType | Yes | Type of consolidated report |
| ReportPeriodStart | Yes | Start date of reporting period |
| ReportPeriodEnd | Yes | End date of reporting period |
| ReportCurrency | Yes | ISO 4217 currency code for consolidated values |
| ConsolidationRuleID | Yes | UUID reference to consolidation rule |
| OrganizationScope | Yes | Top-level organization for consolidation |

---

## Output Data

| Data | Description |
|------|-------------|
| ReportID | System-generated UUID |
| ConsolidatedValue | Aggregated values by dimension |
| FactoryBreakdown | Per-factory contribution to consolidated value |
| AbnormalDataFlag | Outlier detection results |
| ReportSnapshot | Point-in-time copy for audit |

---

## Validation Rules

1. Report type must be valid code from report type catalog
2. Report period start must be before or equal to end date
3. Report currency must be valid ISO 4217 code
4. Exchange rates must exist for all currency pairs in the period
5. All required factory data must be submitted before consolidation
6. Reporting period must be closed (or SystemAdmin override)

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all consolidated reports |
| CompanyAdmin | Full access to company-level and child reports |
| DivisionManager | Full access to division-level and child reports |
| PlantManager | Read-only access to own plant data in consolidated reports |
| Auditor | Read-only access to all consolidated reports |

---

## Multi-Company Data Scope

- **Data Level**: Company/Division/Plant
- **Access**: Users can only view consolidated data within authorized scope
- **Isolation**: Financial data filtered by CompanyCode; plant data visible only to parent organization

---

## Compliance Requirements

- SOX: Consolidated reports support Section 404 internal control reporting
- IFRS/GAAP: Organization hierarchy maps to consolidation entities for statutory reporting
- GDPR: Personal data anonymized in aggregates; report filters respect data residency
- ISO 55000: Supports Clause 9.1 monitoring, measurement, analysis requirements
