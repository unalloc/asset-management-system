# F-2.5.4 - Standardized Asset Classification

**Requirement ID:** F-2.5.4  
**Module:** Multi-Plant Module  
**Priority:** Medium

---

## Requirement Description

Implement standardized asset classification system across the group to ensure consistent and comparable asset data across all factories. Classification enables cross-factory comparison, consolidated reporting, and uniform depreciation treatment under IFRS/US GAAP.

---

## Business Rules

1. Asset classification follows hierarchical structure: Category > Class > Subclass
2. Group-level classifications are mandatory standards that cannot be overridden
3. Classification codes must be unique within the same parent
4. Each classification must be mapped to a depreciation key for IFRS/GAAP compliance
5. Asset classification determines depreciation treatment per IAS 16 (IFRS) and ASC 360 (US GAAP)
6. Legacy factory-specific classifications must map to standard classifications

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| ClassCode | Yes | Unique code (format: 2-4 letters + 4 digits) |
| ClassName | Yes | Display name (1-100 characters) |
| Level | Yes | GROUP, COMPANY, or PLANT level |
| ParentClassID | No | UUID of parent classification (null for root) |
| DepreciationKeyID | Yes | UUID reference to depreciation key |
| UsefulLifeYears | Yes | Expected useful life in years |
| Status | Yes | ACTIVE, INACTIVE, DEPRECATED |

---

## Output Data

| Data | Description |
|------|-------------|
| ClassID | System-generated UUID |
| ClassificationHierarchy | Full path from root to this classification |
| DepreciationSchedule | Calculated depreciation by classification |
| LegacyMappings | Mapping from legacy classifications to standard |

---

## Validation Rules

1. ClassCode must match pattern: 2-4 letters followed by 4 digits
2. ClassName cannot be empty
3. DepreciationKeyID must reference an existing depreciation key
4. ParentClassID must reference an existing classification (if provided)
5. Classification with assigned assets cannot be deleted
6. UsefulLifeYears must be positive integer

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all classifications |
| AssetAdmin | Full access at GROUP and COMPANY level |
| CompanyAdmin | Full access at COMPANY level |
| PlantManager | Read-only access |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant
- **Access**: GROUP classifications visible to all; COMPANY/PLANT visible within scope
- **Isolation**: Classification hierarchies isolated by organizational scope

---

## Compliance Requirements

- IFRS: IAS 16 requires depreciation by significant parts/asset class
- US GAAP: ASC 360 requires consistent depreciation by asset class
- SOX: Classification supports fixed asset internal controls
- OSHA: Safety equipment classification enables inspection tracking
- ISO 55000: Classification enables asset identification and KPI benchmarking
