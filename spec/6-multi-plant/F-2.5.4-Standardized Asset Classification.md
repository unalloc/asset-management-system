### 1.1 Requirement Description

Implement standardized asset classification system across the group to ensure consistent and comparable asset data across all factories. This requirement supports the multi-company data hierarchy by establishing consistent categorization that enables cross-factory comparison, consolidated reporting, and uniform depreciation treatment under IFRS/US GAAP.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-CLS-001 | Group standard classification exists | EXISTS(SELECT 1 FROM AssetClass WHERE Level='GROUP' AND Status='ACTIVE') | CLS-401 | "No active group classification system found" |
| PRE-CLS-002 | User has classification management permission | User.Role IN ('SystemAdmin', 'AssetAdmin') OR User.Permissions CONTAINS 'CLASSIFICATION_MANAGE' | CLS-402 | "User does not have classification management permission" |
| PRE-CLS-003 | Classification code is unique | NOT EXISTS(SELECT 1 FROM AssetClass WHERE ClassCode = Input.ClassCode AND ParentClassID = Input.ParentClassID) | CLS-403 | "Classification code already exists in this parent" |
| PRE-CLS-004 | Parent classification exists | EXISTS(SELECT 1 FROM AssetClass WHERE ClassID = Input.ParentClassID) | CLS-404 | "Parent classification does not exist" |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Is user authorized?
  │
  ├─► NO ─► RETURN CLS-402
  │
  ▼
YES ─► Is this creating a new classification?
  │
  ├─► YES ─► Validate code uniqueness
  │         │
  │         ▼
  │    Validate parent exists (if not root)
  │         │
  │         ▼
  │    Validate depreciation key assigned
  │         │
  │         ▼
  │    Create classification
  │
  └─► NO ─► Is this updating an existing classification?
              │
              ├─► YES ─► Is change allowed? (no assets assigned)
              │         │
              │         ▼
              │    Validate impact (if assets exist)
              │         │
              │         ▼
              │    Update classification
              │
              ▼
           END
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-CLS-001 | Classification has assets assigned | Prevent deletion, require reassignment | Return error with asset count |
| EC-CLS-002 | Factory uses legacy classification | Map to standard via mapping table | Translation on data exchange |
| EC-CLS-003 | Classification restructure | Maintain historical data with old classification | Version historical classifications |
