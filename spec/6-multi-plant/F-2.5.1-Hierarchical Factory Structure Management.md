# F-2.5.1 - Hierarchical Factory Structure Management

**Requirement ID:** F-2.5.1  
**Module:** Multi-Plant Module  
**Priority:** High

---

## Requirement Description

Manage enterprise hierarchical structure: group → business unit → factory → department multi-level organizational structure. This establishes the foundational multi-company data hierarchy that underpins all subsequent module 6 requirements. The organizational hierarchy defines the data scope levels, access control boundaries, and data isolation requirements for the entire asset management system.

---

## Business Rules

1. Organization hierarchy follows standard levels: Group → Company → Division → Plant → Department → WorkCenter → VirtualOrg
2. Each organization level inherits permissions from its parent level
3. Organization code must be unique within sibling organizations
4. Circular dependencies in organization hierarchy are not allowed
5. Active parent organization is required when creating child organizations
6. Level and materialized path are automatically calculated based on parent relationship
7. Matrix organizations (VirtualOrg type) support multiple secondary parent relationships

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| ParentOrganizationID | No | Parent organization UUID (null for root) |
| OrganizationType | Yes | Type: Group, Company, Division, Plant, Department, WorkCenter, VirtualOrg |
| OrganizationCode | Yes | Unique code within parent scope |
| OrganizationName | Yes | Display name (1-200 characters) |
| EffectiveDate | Yes | When organization becomes active |
| ExpirationDate | No | When organization expires |
| JurisdictionCode | No | ISO country code for data residency |
| Timezone | No | Timezone for the organization |
| CurrencyCode | No | ISO 4217 currency code |
| ResponsiblePersonID | No | UUID of responsible person |

---

## Output Data

| Data | Description |
|------|-------------|
| OrganizationID | System-generated UUID |
| Level | Calculated depth in hierarchy |
| MaterializedPath | Path from root (e.g., /GROUP01/COMPANY01/) |
| ChildCount | Number of direct child organizations |
| OrganizationHierarchy | Full path, depth, and ancestors |

---

## Validation Rules

1. Organization code must match pattern: 2-4 letters followed by 2-6 digits
2. Organization name cannot be empty
3. Effective date must be before or equal to expiration date
4. Jurisdiction code must be valid ISO 3166-1 alpha-2 code
5. Currency code must be valid ISO 4217 code
6. MANDATORY attributes must be populated based on organization type (Company requires JurisdictionCode, CurrencyCode, ResponsiblePersonID; Plant requires Timezone, LanguageCode)

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all organizations |
| OrganizationAdmin | Full access within assigned scope |
| CompanyAdmin | Full access at Company level and below |
| DivisionManager | Full access at Division level and below |
| PlantManager | Full access at Plant level |
| DepartmentLead | Read-only at Department level |
| Auditor | Read-only access to all organizations |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Users can only access organizations within their authorized scope
- **Isolation**: MaterializedPath filtering ensures company A cannot view company B organizations

---

## Compliance Requirements

- SOX: Organization hierarchy supports entity-level controls and internal control framework
- GDPR: Data residency enforced via JurisdictionCode attribute
- ISO 55000: Organizational hierarchy enables cascading of asset management objectives
- OSHA: Plant/Department hierarchy supports workplace-specific safety procedure assignment
