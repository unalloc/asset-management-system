# F-2.5.2 - Factory-Specific Asset Configuration

**Requirement ID:** F-2.5.2  
**Module:** Multi-Plant Module  
**Priority:** High

---

## Requirement Description

Allow each factory to configure asset attributes and maintenance strategies based on local requirements while maintaining group standards. Configuration inheritance follows the organizational hierarchy: Group → Company → Division → Plant → Department. This implements the multi-company data hierarchy by enabling factory-specific configuration overrides within the group-level framework.

---

## Business Rules

1. Configuration templates are defined at Group level
2. MANDATORY configurations cannot be overridden below Group level
3. PLANT_OVERRIDABLE configurations can be overridden at Plant level or below
4. OPTIONAL configurations can be set at any level or inherited from parent
5. Configuration value resolution walks up hierarchy to find nearest value
6. All configuration modifications require audit trail entry
7. Version number increments on every configuration modification
8. Rollback capability exists to restore previous version

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| ConfigTemplateID | Yes | UUID reference to configuration template |
| OrganizationID | Yes | UUID of organization setting the configuration |
| ConfigValue | Yes | JSON value matching template schema |
| EffectiveDate | Yes | When configuration becomes active |
| ExpirationDate | No | When configuration expires |
| ChangeReason | No | Reason for configuration change |

---

## Output Data

| Data | Description |
|------|-------------|
| ConfigValueID | System-generated UUID |
| VersionNumber | Incremented version for this configuration |
| ResolvedConfigValue | Effective value after inheritance resolution |
| ConfigComplianceReport | Variance analysis vs group template |

---

## Validation Rules

1. Configuration name must start with letter, contain only letters, numbers, underscore
2. ConfigValue must be valid JSON and match template schema
3. ConfigValue must be within allowed range defined by template
4. Configuration type must be valid enum (MANDATORY, PLANT_OVERRIDABLE, OPTIONAL)
5. Effective date must be today or in the future
6. Circular configuration dependencies are not allowed

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all configurations including MANDATORY |
| CompanyAdmin | Full access to Company and child configurations |
| DivisionManager | Full access to Division and child configurations |
| PlantManager | Full access to Plant-level configurations |
| DepartmentLead | Read-only access |
| Auditor | Read-only access with full scope |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Users can only modify configurations at their authorized organization level
- **Isolation**: Configuration data filtered by OrganizationID; cross-company access requires SystemAdmin

---

## Compliance Requirements

- SOX: Configuration changes tracked in audit trail for internal controls
- GAMP 5: Configuration management supports validation lifecycle
- GDPR: Configuration data isolation per plant for employee-related parameters
- IFRS/GAAP: Depreciation-related configurations require documented approval
