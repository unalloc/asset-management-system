### 1.1 Requirement Description

Manage asset transfer process between different factories, including cost allocation, approval workflow, and intercompany transaction handling. This requirement implements the cross-factory data movement within the multi-company hierarchy, ensuring proper accounting treatment for asset movements between legal entities (SOX, IFRS 16) and proper tracking for WEEE/environmental compliance.

**Regulatory Compliance Details:**

**SOX**: Asset transfers between factories represent material transactions:
- Intercompany transfers require explicit approval
- Transfer pricing documentation for cross-company moves
- Audit trail must capture before/after values and approval chain

**IFRS/GAAP Transfer Accounting**:
- Intercompany profit/loss elimination in consolidated statements
- Asset revaluation may be required at transfer
- Tax basis tracking for cost segregation

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-TRF-001 | Asset exists and is transferable | Asset.Status IN ('ACTIVE','IDLE') AND Asset.Transferable = TRUE | TRF-401 | "Asset current status not transferable" |
| PRE-TRF-002 | Source and destination factories are valid | EXISTS(SELECT 1 FROM Organization WHERE OrganizationID IN (SourceID, DestID) AND Status='ACTIVE') | TRF-402 | "Source or destination factory invalid" |
| PRE-TRF-003 | User has transfer permission | User.Role IN ('SystemAdmin', 'PlantManager', 'AssetManager') | TRF-403 | "User does not have transfer permission" |
| PRE-TRF-004 | Approval workflow complete | TransferApprovals count >= RequiredApprovals | TRF-404 | "Approval workflow incomplete" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-TRF-001 | Transfer completed and recorded | Asset.OrganizationID = DestinationID, TransferRecord created |
| POST-TRF-002 | Asset status updated | Asset.Status changed to 'IN_TRANSIT' then 'ACTIVE' |
| POST-TRF-003 | Cost allocation settled | Intercompany invoice created if cross-company |
| POST-TRF-004 | Audit trail recorded | AuditLog entry with full transfer details |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Is asset transferable?
  │
  ├─► NO ─► RETURN TRF-401
  │
  ▼
YES ─► Are source and destination valid?
  │
  ├─► NO ─► RETURN TRF-402
  │
  ▼
YES ─► Is this cross-company transfer?
  │
  ├─► YES ─► Calculate transfer price
  │         │
  │         ▼
  │    Generate intercompany invoice
  │
  ▼
Execute approval workflow
  │
  ▼
Is approval granted?
  │
  ├─► NO ─► RETURN TRF-404
  │
  ▼
YES ─► Update asset organization ID
  │
  ▼
Create transfer record
  │
  ▼
Update depreciation books
  │
  ▼
Write audit trail
  │
  ▼
END
```
