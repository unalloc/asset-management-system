### 1.1 Requirement Description

Centrally manage service contracts at group level, tracking execution status across all factories. This requirement enables the organization hierarchy to manage vendor contracts centrally while tracking factory-specific execution, supporting both group-level procurement optimization and plant-level service delivery.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-CON-001 | User has contract management permission | User.Role IN ('SystemAdmin', 'ContractAdmin', 'ProcurementManager') | CON-401 | "User does not have contract management permission" |
| PRE-CON-002 | Contract terms are complete | All required fields populated per contract type | CON-402 | "Contract information incomplete" |
| PRE-CON-003 | Vendor is approved | Vendor.Status = 'APPROVED' | CON-403 | "Vendor not approved" |
| PRE-CON-004 | SLA definitions are measurable | EXISTS(SELECT 1 FROM SLA WHERE IsMeasurable = TRUE) | CON-404 | "SLA definition not measurable" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-CON-001 | Contract execution tracked | ExecutionStatus updated per factory |
| POST-CON-002 | Contract performance evaluated | PerformanceScore calculated |
| POST-CON-003 | Cost allocation settled | Invoices allocated to factories |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Is user authorized?
  │
  ├─► NO ─► RETURN CON-401
  │
  ▼
YES ─► Is this group-level or factory-level contract?
  │
  ├─► GROUP ─► Create at group level
  │            │
  │            ▼
  │       Assign to factories
  │
  └─► FACTORY ─► Create at factory level
                 │
                 ▼
            Track factory execution
              │
              ▼
         Aggregate to group reporting
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior |

| EC-CON-002 | Contract renewal auto-trigger | Notify 90 days before expiration |
| EC-CON-003 | SLA breach | Trigger corrective action workflow |
