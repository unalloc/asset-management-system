# F-2.9.2 - China Accounting Depreciation Methods

## Function ID
F-2.9.2

## Function Name
China Accounting Depreciation Method Management

## Function Description
Support depreciation methods specified in China accounting standards, including straight-line method, double-declining balance method, and sum-of-years-digits method, supporting depreciation calculation, adjustment, and retrospective calculation.

## Business Rules
1. Depreciation methods: Support straight-line method, double-declining balance method, sum-of-years-digits method
2. Calculation method: Monthly automatic depreciation calculation, support manual adjustment
3. Depreciation period: Start from the month after启用日期, until asset disposal or full depreciation
4. Residual value: Support expected residual value rate or fixed residual value amount
5. Depreciation adjustment: Support retrospective adjustment for depreciation policy changes
6. Depreciation suspension: Support depreciation suspension during asset idle period
7. Accelerated depreciation: Support accelerated depreciation calculation for tax incentives
8. Depreciation report: Generate monthly depreciation accrual table, depreciation detail table

## Preconditions
- Asset original value, useful life, residual value information complete
- Depreciation method policy defined
- Accounting period configured correctly
- User has "Depreciation Calculation" permission

## Postconditions
- Monthly depreciation calculation completed
- Depreciation voucher generated
- Depreciation report queryable
- Asset net value updated

## Input Data
1. Asset ID
2. Depreciation method (straight-line, double-declining, sum-of-years)
3. Useful life (months)
4. Expected residual value rate (%)
5. Start date
6. Depreciation policy change date (if applicable)
7. Accelerated depreciation flag (if applicable)
8. Idle period (if applicable)

## Output Data
1. Monthly depreciation calculation table
2. Depreciation accrual voucher
3. Asset depreciation detail table
4. Depreciation policy change record
5. Depreciation adjustment entries

## Validation Rules
1. Useful life must be greater than 0
2. Expected residual value rate must be between 0-100%
3. Start date must be valid
4. Depreciation method must be one of the three types
5. Asset status must be "In Use" or "Idle"

## Error Handling
1. Calculation error: Show depreciation calculation parameter error
2. Period error: Show accounting period not opened or already closed
3. Voucher error: Show voucher generation failure reason
4. Retrospective error: Show retrospective adjustment exceeds allowed range

## Priority
High
