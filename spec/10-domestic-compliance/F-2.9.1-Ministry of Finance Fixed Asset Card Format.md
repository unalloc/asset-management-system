# F-2.9.1 - Ministry of Finance Fixed Asset Card Format

## Function ID
F-2.9.1

## Function Name
Ministry of Finance Fixed Asset Card Management

## Function Description
Support fixed asset card format compliant with China Ministry of Finance "Fixed Asset Management Measures" requirements, including card design, field mapping, print output, and electronic archiving.

## Business Rules
1. Card format: Compliant with Ministry of Finance "Fixed Asset Card (Standard Format)"
2. Required fields: Asset number, asset name, specification model, unit of measure, quantity, original value, using department, user, storage location,启用日期
3. Optional fields: Manufacturer, factory number, useful life, expected residual value rate, depreciation method, accumulated depreciation, net value
4. Card printing: Support standard A4 paper printing, support pre-printed form printing
5. Electronic card: Support PDF format export with digital signature
6. History tracking: Card information change history tracking
7. Multi-language support: Card headers support Chinese-English对照
8. Custom fields: Allow adding enterprise custom fields without affecting standard format

## Preconditions
- Asset basic information entered into system
- Ministry of Finance card format template available
- User has "Fixed Asset Card Management" permission
- Printing equipment configured

## Postconditions
- Fixed asset card record created
- Card format validation passed
- Print/export function available
- Electronic archiving completed

## Input Data
1. Asset number
2. Asset name
3. Specification model
4. Unit of measure
5. Quantity
6. Original value (RMB)
7. Using department
8. User
9. Storage location
10. Start date
11. Manufacturer (optional)
12. Factory number (optional)
13. Useful life (optional)
14. Expected residual value rate (optional)
15. Depreciation method (optional)
16. Custom fields (optional)

## Output Data
1. Fixed asset card record
2. Card print file (PDF/paper)
3. Card format validation report
4. Electronic archiving record
5. Change history record

## Validation Rules
1. Asset number must be unique
2. Asset name cannot be empty
3. Original value must be greater than 0
4. Start date cannot be later than current date
5. Using department must exist in organizational structure
6. Unit of measure must be standard unit

## Error Handling
1. Format error: Show specific fields that do not comply with Ministry of Finance standard format
2. Data missing: Show required fields not filled
3. Print failure: Show printer status or configuration problem
4. Save failure: Show database connection or permission problem

High
