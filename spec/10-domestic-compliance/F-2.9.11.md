# F-2.9.11 WeChat Mini-program Mobile Counting

## Function ID
F-2.9.11

## Function Name
WeChat Mini-program Mobile Counting

## Function Description
Support mobile asset counting through WeChat mini-program, including scan counting, photo upload, real-time synchronization, and offline operation.

## Business Rules
1. Mini-program entry: Enter via WeChat search or scan code
2. Scan counting: Scan asset QR code/barcode for counting
3. Photo upload: Upload photos of asset condition, damage
4. Real-time synchronization: Count data real-time synchronized to main system
5. Offline operation: Support counting in non-network environment, synchronize after network recovery
6. Counting task: Receive counting tasks, display assets to be counted
7. Counting statistics: Real-time display of counting progress and statistics
8. Permission control: WeChat user-based permission control

## Preconditions
- WeChat mini-program published
- Asset QR code/barcode generated
- User WeChat authorized
- Counting task created

## Postconditions
- Count data collection completed
- Data synchronized to main system
- Count report generated
- Difference handling started

## Input Data
1. WeChat user identity
2. Scan data (asset number)
3. Count status (normal, damaged, lost, etc.)
4. Count photos
5. Count remarks
6. Count time
7. GPS location (optional)
8. Offline cached data

## Output Data
1. Count record
2. Asset photos
3. Count progress report
4. Data synchronization status
5. Count difference report
6. User operation log

## Validation Rules
1. WeChat user must be authorized
2. Asset number must be valid
3. Count status must be predefined value
4. Count time must be within task period
5. GPS location must be within reasonable range (if enabled)

## Error Handling
1. Scan error: Show asset number invalid or duplicate
2. Network error: Show network connection failed, enable offline mode
3. Synchronization error: Show data synchronization failure reason
4. Permission error: Show user has no counting permission

## Priority
High
