## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-TAXDEP-001 | Country must be supported | CountryCode IN SupportedCountries | TAXDEP-ERR-001 | "Country not supported" |
| PRE-TAXDEP-002 | Tax category must exist | TaxCategory IN ValidCategories(Country) | TAXDEP-ERR-002 | "Invalid tax category for country" |
| PRE-TAXDEP-003 | Depreciation method valid | Method IN ValidMethods(Country, Category) | TAXDEP-ERR-003 | "Invalid depreciation method for country" |
| PRE-TAXDEP-004 | Convention valid | Convention IN ValidConventions(Country) | TAXDEP-ERR-004 | "Invalid convention for country" |

**Pre-condition Validation Matrix:**

```
IF (Asset.TaxCountry IN SupportedCountries)
   AND (Asset.TaxCategory IN ValidCategories(Country))
   AND (Asset.TaxDepreciationMethod IN ValidMethods(Country, Category))
   AND (Asset.TaxConvention IN ValidConventions(Country))
THEN calculate tax depreciation
ELSE throw TAXDEP-ERR-00X Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-TAXDEP-001 | Tax depreciation calculated | Matches country-specific tables | Table verification |
| POST-TAXDEP-002 | Convention applied correctly | Half-year, mid-quarter applied | Period verification |

**Post-condition Verification:**

- Verify depreciation matches official tax tables
- Confirm convention application correct

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Determine Country and Tax Category
  │
  ▼
Select Depreciation Method
  │
  ├─► US MACRS 200% DB ─► Apply declining balance, switch to SL
  ├─► US MACRS 150% DB ─► Apply declining balance, switch to SL
  ├─► US Straight-Line ─► Alternative election
  ├─► Germany AfA ─► Apply table rates
  ├─► France Amortissement ─► Apply table rates
  └─► UK Capital Allowances ─► Apply annual investment allowance
        │
        ▼
    Apply Convention
      │
      ├─► Half-Year ─► 50% first and last year
      ├─► Mid-Quarter ─► Based on QOS placed in service
      └─► Full-Year ─► No convention (some countries)
            │
            ▼
        Calculate Tax Depreciation
            │
            ▼
        Apply Special Treatments
          │
          ├─► Section 179 (US) ─► Immediate deduction
          ├─► Bonus Depreciation (US) ─► Additional first year
          └─► AIA (UK) ─► Annual investment allowance
                │
                ▼
            END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-TAXDEP-001 | MACRS 200% DB | US, property class | 200% declining balance, switch | 1 |
| BR-TAXDEP-002 | MACRS 150% DB | US, alternative | 150% declining balance, switch | 2 |
| BR-TAXDEP-003 | Half-Year Convention | US, standard | 50% first and last year | 1 |
| BR-TAXDEP-004 | Mid-Quarter Convention | US, election | Based on quarter placed | 1 |
| BR-TAXDEP-005 | German AfA | Germany | Apply BMF tables | 1 |
| BR-TAXDEP-006 | French Amortissement | France | Apply administration tables | 1 |
| BR-TAXDEP-007 | UK Capital Allowances | UK | Apply annual rates | 1 |
| BR-TAXDEP-008 | Section 179 | US, eligible | Immediate deduction up to limit | 1 |
| BR-TAXDEP-009 | Bonus Depreciation | US, enacted | Additional percentage first year | 1 |
| BR-TAXDEP-010 | Year-End Placed | Mid-quarter trigger | Apply mid-quarter if >40% Q4 | 1 |

**Calculation Logic:**

```
US MACRS 200% DB:
  DB_Rate = 2.0 / Recovery_Period
  Year1 = (Cost - Salvage) × DB_Rate × 0.5 (half-year)
  YearN = MIN((BookValue - Salvage) × DB_Rate, Straight-Line)
  Switch to SL when SL > DB

Germany AfA:
  Annual_Depreciation = Cost × AfA_Table_Rate(Property_Class)

Section 179 Deduction:
  Deduction = MIN(Cost, Section_179_Limit)
  Reduced_Basis = Cost - Deduction

Bonus Depreciation:
  Additional_First_Year = (Cost - Section_179) × Bonus_Rate
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-TAXDEP-001 | Mid-quarter election | Check Q4 placed percentage | Apply if >40% Q4 |
| EC-TAXDEP-002 | Property class change | Apply new class for remaining life | Propagate change |
| EC-TAXDEP-003 | Section 179 recapture | Calculate recapture on disposal | Add to income |
| EC-TAXDEP-004 | Multi-country asset | Calculate per country | Separate books |
| EC-TAXDEP-005 | Country law change | Apply new rates prospectively | Update tables |

| UK | Quarterly | Any | Pro-rata |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-TAXDEP-001 | Mid-quarter check | >40% Q4 placed | Apply mid-quarter | TAXDEP-WRN-001 |
| BRV-TAXDEP-002 | Section 179 limit | Amount <= limit | Apply limit | TAXDEP-WRN-002 |

**Cross-Field Validation Logic:**

```
IF (Asset.TaxCountry = 'US')
   AND (Asset.PlacedInServiceQtr = 4)
   AND (Asset.Q4PlacedPercentage > 0.4)
THEN display TAXDEP-WRN-001 "Mid-quarter convention applies"
     AND apply mid-quarter rates

IF (Asset.TaxCountry = 'US')
   AND (Asset.Section179Amount > US_SECTION179_LIMIT)
THEN apply LIMIT
     AND display TAXDEP-WRN-002 "Section 179 deduction limited to $1,160,000"
```
