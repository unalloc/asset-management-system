# F-2.4.8 - Component-Level Financial

**Requirement ID:** F-2.4.8  
**Module:** Financial Management  
**Priority:** Medium

---

## Requirement Description

Provide independent financial tracking and depreciation calculation for components of complex assets. Each significant component depreciated separately per IAS 16 and ASC 360 requirements with parent asset rolling up component values.

---

## Business Rules

1. Component allocations must sum to 100% of parent asset cost
2. Each component has independent depreciation schedule
3. Parent asset net book value = sum of component net book values
4. Component replacement triggers disposal of old and capitalization of new
5. Remaining life of parent = minimum remaining life of active components

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Parent Asset | Yes | Complex asset with components |
| Component Name | Yes | Name of the component |
| Allocation Percentage | Yes | Portion of parent cost (must total 100%) |
| Component Life | Yes | Useful life of component |
| Residual Value | Yes | Expected value at end of component life |
| Depreciation Method | Yes | How component depreciates |

---

## Output Data

| Data | Description |
|------|-------------|
| Component Depreciation Schedule | Independent schedules per component |
| Parent Asset Rollup | Parent NBV = sum of components |
| Replacement Records | Old component disposal, new component capitalization |

---

## Validation Rules

1. Parent asset must be designated as a parent asset
2. Allocation percentages must sum to exactly 100%
3. Component life must be between 1 and parent asset life
4. Depreciation method must be specified for each component
5. Component life cannot exceed parent life

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full access |
| Company Controller | Define components, approve replacements |
| Division Finance | View within division |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Component definition at company level
- **Isolation**: Scoped by parent asset's Company Code |

---

## Compliance Requirements

- ASC 360 (US GAAP) component depreciation for significant parts
- IAS 16 (IFRS) depreciation of significant components separately
- SOX 404 audit trail for component changes
