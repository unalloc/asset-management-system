## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-CMP-001 | Parent asset must exist | AssetID IN (SELECT AssetID FROM Asset WHERE IsParent = TRUE) | CMP-ERR-001 | "Parent asset must be a parent asset" |
| PRE-CMP-002 | Component cost allocation must sum to 100% | SUM(AllocationPercent) = 100 | CMP-ERR-002 | "Component allocations must sum to 100%" |
| PRE-CMP-003 | Component useful life must be valid | ComponentLife >= 1 AND ComponentLife <= ParentLife | CMP-ERR-003 | "Component life must be between 1 and parent asset life" |
| PRE-CMP-004 | Component must have depreciation method | DepreciationMethod IS NOT NULL | CMP-ERR-004 | "Component must have depreciation method" |

**Pre-condition Validation Matrix:**

```
IF (Component.ParentAssetID IS NOT NULL)
   AND (Component.AllocationPercent > 0)
   AND (Component.ComponentLife >= 1)
   AND (SUM(Component.AllocationPercent) = 100)
THEN validate component
ELSE throw CMP-ERR-00X Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-CMP-001 | Component depreciation calculated | Each component depreciated independently | Schedule verification |
| POST-CMP-002 | Parent asset reflects component changes | Parent NBV = Sum(Component NBVs) | Reconciliation |

**Post-condition Verification:**

- Verify component depreciation schedules sum to parent schedule
- Confirm asset hierarchy reflects component structure

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Identify Asset Components
  │
  ▼
Determine Component Costs
  │
  ├─► Original Components ─► Allocation from parent cost
  ├─► Added Components ─► Capitalize as improvement
  └─► Replaced Components ─► Handle disposal of old
        │
        ▼
    Calculate Component Depreciation
        │
        ▼
    Track Component Status
        │
        ├─► In Service ─► Depreciate normally
        ├─► Replaced ─► Stop depreciation, record gain/loss
        └─► Disposed ─► Remove from asset
              │
              ▼
          Update Parent Asset
              │
              ▼
          END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-CMP-001 | Cost Allocation | Original components | ParentCost × AllocationPercent | 1 |
| BR-CMP-002 | Component Depreciation | Each component | Independent schedule per component | 1 |
| BR-CMP-003 | Improvement Capitalization | New component added | Add to parent cost | 1 |
| BR-CMP-004 | Replacement Disposal | Old component replaced | Calculate gain/loss | 1 |
| BR-CMP-005 | Parent NBV Calculation | Sum of components | ParentNBV = SUM(ComponentNBV) | 1 |
| BR-CMP-006 | Remaining Life Adjustment | Component replaced | Recalculate parent remaining life | 2 |

**Calculation Logic:**

```
Component Cost = ParentCost × (ComponentAllocationPercent / 100)

Component Depreciation = (ComponentCost - ComponentResidualValue) / ComponentLife

Component NBV = ComponentCost - ComponentAccumulatedDepreciation

Parent NBV = Σ(Component NBV for all active components)

Replacement Gain/Loss = DisposalProceeds - Component Net Book Value

Parent Remaining Life = MIN(Remaining life of all active components)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-CMP-001 | Component fully depreciated | Continue tracking at zero NBV | Maintain records |
| EC-CMP-002 | Component replaced | Dispose old, capitalize new | Replacement workflow |
| EC-CMP-003 | Single component | Treat as simple asset | Standard depreciation |
| EC-CMP-004 | Added component mid-life | Capitalize and depreciate | Partial period |
| EC-CMP-005 | Component exceeds parent life | Warn user | Adjust life or reject |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| Allocation = 100% | All components defined | Valid | Proceed |
| Allocation < 100% | Missing components | Incomplete | Require completion |
| Allocation > 100% | Over-allocated | Error | Reject |
| Component Life > Parent Life | Invalid | Warning | Adjust |

### 3.5 State Transitions

**State Diagram:**

```
States: DEFINED, ACTIVE, COMPONENT_REPLACED, DISPOSED

Transitions:
┌─────────────┐       ┌─────────────┐       ┌─────────────────┐
│  DEFINED    │──────►│   ACTIVE    │──────►│COMPONENT_REPLACED│
└─────────────┘       └─────────────┘       └─────────────────┘
                                                       │
       │                                                 │
       │                                                 ▼
       │                                          ┌─────────────┐
       └──────────────────────────────────────────│  DISPOSED   │
                                                   └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| DEFINED | Parent asset activated | ACTIVE | All components valid | Component depreciation begins |
| ACTIVE | Component replaced | COMPONENT_REPLACED | Replacement approved | Old component disposed |
| ACTIVE | Parent disposed | DISPOSED | Disposal approved | All components disposed |
| COMPONENT_REPLACED | New component added | ACTIVE | New component valid | Resume depreciation |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-CMP-001 | Allocation sum | SUM(AllocationPercent) = 100 | Reject | CMP-ERR-002 |
| BRV-CMP-002 | Life within parent | ComponentLife <= ParentUsefulLife | Warn | CMP-WRN-001 |

**Cross-Field Validation Logic:**

```
IF (SUM(Component.AllocationPercent) != 100)
THEN display CMP-ERR-002 "Component allocations must sum to 100%"
     AND block until corrected

IF (Component.UsefulLife > ParentAsset.UsefulLife)
THEN display CMP-WRN-001 "Component life exceeds parent life - may cause issues"
```
