## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-CIP-001 | Project code must be unique | ProjectCode NOT IN (SELECT ProjectCode FROM CIPProject) | CIP-ERR-001 | "Project code already exists" |
| PRE-CIP-002 | Budget must be positive | BudgetAmount > 0 | CIP-ERR-002 | "Budget amount must be positive" |
| PRE-CIP-003 | Start date must be valid | StartDate <= EndDate OR EndDate IS NULL | CIP-ERR-003 | "Start date must be before end date" |
| PRE-CIP-004 | Capitalization threshold met | TotalCost >= CapitalizationThreshold | CIP-ERR-004 | "Project cost below capitalization threshold" |
| PRE-CIP-005 | Budget approved | ApprovalStatus = 'APPROVED' | CIP-ERR-005 | "Project budget must be approved" |

**Pre-condition Validation Matrix:**

```
IF (CIPProject.ProjectCode IS UNIQUE)
   AND (CIPProject.BudgetAmount > 0)
   AND (CIPProject.StartDate <= CIPProject.EndDate OR CIPProject.EndDate IS NULL)
   AND (CIPProject.ApprovalStatus = 'APPROVED')
THEN activate project tracking
ELSE throw CIP-ERR-00X Error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Project Initiation
  │
  ▼
Budget Approval
  │
  ▼
Expenditure Tracking
  │
  ├─► Direct Costs ─► Capitalize
  ├─► Indirect Costs ─► Allocate by percentage
  ├─► Interest ─► Capitalize if qualifying
  └─► Overhead ─► Capitalize per policy
        │
        ▼
    Milestone Review
        │
        ├─► Pass ─► Continue
        └─► Fail ─► Issue Resolution
              │
              ▼
          Capitalization Decision
              │
              ├─► Yes ─► Create Asset, Transfer Costs
              └─► No ─► Expense, Close Project
                    │
                    ▼
                Project Close
                    │
                    ▼
                END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-CIP-001 | Direct Cost Capitalization | Cost directly attributable | Add to asset cost | 1 |
| BR-CIP-002 | Indirect Cost Allocation | Allocation basis defined | Allocate by basis | 1 |
| BR-CIP-003 | Interest Capitalization | Qualifying asset, actual interest | Add to asset cost | 1 |
| BR-CIP-004 | Overhead Capitalization | Policy allows overhead | Apply overhead rate | 2 |
| BR-CIP-005 | Asset Capitalization | Substantial completion | Create asset record | 1 |
| BR-CIP-006 | Remaining Cost Expense | Non-qualifying costs | Write off to expense | 1 |
| BR-CIP-007 | Cost Transfer Validation | Sum of parts = Total | Verify allocation | 1 |

**Calculation Logic:**

```
Capitalizable Cost = DirectCosts + AllocatedIndirectCosts + CapitalizedInterest

Interest Capitalization:
  CapitalizedInterest = WeightedAverageAccumulatedExpenditures × CapitalizationRate
  Where:
    WeightedAverage = Σ(MonthlyExpenditure × MonthsRemaining/12)
    CapitalizationRate = SpecificBorrowingRate OR WeightedAverageRate

Overhead Application:
  OverheadAmount = DirectCosts × OverheadRate
  TotalAssetCost = DirectCosts + IndirectCosts + Interest + Overhead

Asset Creation:
  Asset.AcquisitionCost = TotalCapitalizedCost
  Asset.CapitalizationDate = SubstantialCompletionDate
```

### 3.4 Edge Cases

|--------------|----------|-------------------|-------------------|
| EC-CIP-001 | Abandoned project | Expense all costs, no asset | Write-off workflow |

| EC-CIP-004 | Cost over-run | Alert, require approval | Over-run approval |
| EC-CIP-005 | Multi-asset project | Create multiple assets | Asset splitting |
| EC-CIP-006 | Long-term construction | Continue capitalization | Periodic review |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| Substantially Complete | All costs captured | Threshold met | Capitalize |
| Substantially Complete | All costs captured | Threshold not met | Expense |
| Not Complete | Any | Any | Continue tracking |
| Abandoned | Any | Any | Expense all costs |
