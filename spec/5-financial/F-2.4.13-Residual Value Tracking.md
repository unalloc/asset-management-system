## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-RESV-001 | Residual value reasonable | ResidualValue >= 0 AND ResidualValue <= AcquisitionCost | RESV-ERR-001 | "Residual value must be between 0 and acquisition cost" |
| PRE-RESV-002 | Assessment documented | MarketAssessment exists or Reason provided | RESV-ERR-002 | "Market assessment or justification required" |
| PRE-RESV-003 | Asset near end of life | RemainingLife <= 2 years OR Status = 'DISPOSED' | RESV-WRN-001 | "Asset should be near end of life for residual review" |

**Pre-condition Validation Matrix:**

```
IF (ResidualValue >= 0)
   AND (ResidualValue <= Asset.AcquisitionCost)
   AND (MarketAssessment EXISTS OR Reason IS NOT NULL)
THEN accept residual value
ELSE throw RESV-ERR-00X Error
```

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Set Initial Residual Value
  │
  ▼
Periodic Review
  │
  ├─► Annual assessment
  ├─► Trigger-based review (condition change)
  └─► Pre-disposal final assessment
        │
        ▼
    Market Comparison
      │
      ├─► Market value > Book value ─► Potential gain
      ├─► Market value < Book value ─► Potential loss
      └─► Market value = Book value ─► No gain/loss
            │
            ▼
        Disposal Decision
          │
          ├─► Proceed with disposal ─► Record actual proceeds
          ├─► Defer disposal ─► Update estimate
          └─► Recondition ─► Extend life, update residual
                │
                ▼
            Record Gain/Loss
                │
                ▼
            END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-RESV-001 | Initial Residual Value | At acquisition | % of cost based on category | 1 |
| BR-RESV-002 | Annual Review | Annual trigger | Update based on market | 1 |
| BR-RESV-003 | Disposal Gain/Loss | At disposal | Proceeds - NBV | 1 |
| BR-RESV-004 | Residual Value Floor | During depreciation | NBV cannot go below residual | 1 |
| BR-RESV-005 | Variance Analysis | Actual vs estimated | Track realization rate | 2 |

**Calculation Logic:**

```
Disposal Gain/Loss = Disposal Proceeds - Net Book Value

Where:
  Net Book Value = Acquisition Cost - Accumulated Depreciation

Realization Rate = Actual Proceeds / Estimated Residual Value × 100%

Variance = Actual Proceeds - Estimated Residual Value
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-RESV-001 | Negative NBV | Write off additional loss | Record as expense |
| EC-RESV-002 | No market for asset | Find alternative disposal | Recycle/scrap |
| EC-RESV-003 | Trade-in value | Use trade-in as proceeds | Document trade-in |
| EC-RESV-004 | Partial disposal | Allocate residual value | Pro-rata calculation |
