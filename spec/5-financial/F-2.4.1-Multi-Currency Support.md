## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-MC-001 | Currency code must be valid ISO 4217 | CurrencyCode IN (SELECT Code FROM ISO4217_Codes) | MC-ERR-001 | "Invalid ISO 4217 currency code" |
| PRE-MC-002 | Exchange rate must be positive | ExchangeRate > 0 | MC-ERR-002 | "Exchange rate must be a positive number" |
| PRE-MC-003 | Exchange rate date cannot be future | RateDate <= CURRENT_DATE | MC-ERR-003 | "Exchange rate date cannot be in the future" |
| PRE-MC-004 | Asset must have functional currency defined | FunctionalCurrency IS NOT NULL | MC-ERR-004 | "Functional currency must be defined for the asset" |

**Pre-condition Validation Matrix:**

```
IF (Asset.CompanyCode IS NOT NULL)
   AND (Asset.FunctionalCurrency IS NOT NULL)
   AND (Asset.AcquisitionCurrency IN VALID_ISO4217_CODES)
THEN proceed to currency conversion
ELSE throw MC-ERR-004 Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-MC-001 | Asset values translated | All asset values available in functional and reporting currency | Query verification |
| POST-MC-002 | Exchange gains/losses recorded | Unrealized gain/loss amounts match calculation | Reconciliation report |

**Post-condition Verification:**

- Verify all assets have both functional and reporting currency values
- Confirm exchange gain/loss calculations match GL entries

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Asset Acquisition Entry
  │
  ▼
Is AcquisitionCurrency = FunctionalCurrency?
  ├─► YES ─► Record in Functional Currency ─► END
  └─► NO ─► Fetch Exchange Rate for AcquisitionDate
        │
        ▼
    Calculate FunctionalCurrencyAmount = AcquisitionAmount × ExchangeRate
        │
        ▼
    Record Exchange Gain/Loss if historical rate differs
        │
        ▼
    END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-MC-001 | Exchange Rate Source Priority | Rate requested | Use Central Bank rate > Corporate rate > Manual rate | 1 |
| BR-MC-002 | Rounding Rule | Currency conversion | Round to currency precision (2 decimals for most, 0 for JPY) | 2 |
| BR-MC-003 | Temporal Method | Asset < 12 months old | Use historical rate on transaction date | 1 |
| BR-MC-004 | Current Rate Method | Investment property/long-term assets | Use closing rate at reporting date | 1 |
| BR-MC-005 | Realized Gain/Loss | Asset disposal | Recognize accumulated translation adjustments | 1 |
| BR-MC-006 | Unrealized Gain/Loss | Period-end revaluation | Record to OCI or P&L based on classification | 1 |

**Calculation Logic:**

```
FunctionalCurrencyAmount = AcquisitionAmount × ExchangeRate(AcquisitionDate, AcquisitionCurrency, FunctionalCurrency)

ReportingCurrencyAmount = FunctionalCurrencyAmount × ExchangeRate(ReportingDate, FunctionalCurrency, ReportingCurrency)

ExchangeGainLoss = ReportingCurrencyAmount(PeriodEnd) - ReportingCurrencyAmount(PeriodStart)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-MC-001 | Hypothecary rate not available | Use last available rate with warning | WRN-MC-001 with manual override option |
| EC-MC-002 | Dual currency asset (e.g., EUR/USD) | Split into two transaction records | Create separate line items |
| EC-MC-003 | Hyperinflationary currency | Apply IAS 29 restatement | Historical cost adjustment |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| Rate Available | Rate Date Valid | Asset Type | Action |
| YES | YES | Any | Standard conversion |
| NO | N/A | Any | Use last rate + warning |
| YES | NO (Future) | Any | Reject with MC-ERR-003 |
| YES | YES | Investment Property | Current rate method |

Transitions:
┌─────────────┐       ┌─────────────┐       ┌─────────────┐

### 6.1 Field-Level Validation

| Field Name | Data Type | Format | Range | Required | Validation Rule | Error Code | Error Message |
|------------|-----------|--------|-------|----------|-----------------|------------|---------------|
| AcquisitionCurrency | CHAR | ISO 4217 | N/A | YES | Pattern match: `^[A-Z]{3}$` | MC-VAL-001 | "Currency code must be 3 uppercase letters (ISO 4217)" |
| FunctionalCurrency | CHAR | ISO 4217 | N/A | YES | Must exist in currency master | MC-VAL-002 | "Invalid functional currency code" |
| ExchangeRate | DECIMAL | N/A | > 0 | YES | Range check | MC-VAL-003 | "Exchange rate must be greater than 0" |
| AcquisitionCost | DECIMAL | N/A | > 0 AND <= 999999999.9999 | YES | Range check | MC-VAL-004 | "Acquisition cost must be between 0 and 999,999,999.9999" |
| RateDate | DATE | ISO 8601 | <= CURRENT_DATE | YES | Date validation | MC-VAL-005 | "Rate date cannot be in the future" |

### 6.2 Business Rule Validation

| Validation Rule | Description | Condition | Action on Failure | Error Code |
|-----------------|-------------|-----------|------------------|------------|
| BRV-MC-001 | Currency code uniqueness | CompanyCode + AssetCode + Currency | Reject duplicate | MC-VAL-006 |
| BRV-MC-002 | Rate source validation | RateSource IN ('CENTRAL_BANK', 'CORPORATE', 'MANUAL') | Reject invalid | MC-VAL-007 |
| BRV-MC-003 | Acquisition date rate required | Rate must exist for AcquisitionDate | Allow with warning | MC-WRN-001 |

**Cross-Field Validation Logic:**

```
IF (Asset.AcquisitionCurrency != Asset.FunctionalCurrency)
   AND (ExchangeRate_For_Date(Asset.AcquisitionDate, Asset.AcquisitionCurrency, Asset.FunctionalCurrency) NOT EXISTS)
THEN require ManualRateApproval
     AND display WRN-MC-001 "No exchange rate available for acquisition date"
```

| MC-ERR-003 | Error | Validation | Future rate date | Exchange rate date cannot be in the future | Enter a date on or before today |

| MC-VAL-002 | Error | Validation | Invalid functional currency | Invalid functional currency code | Select a valid functional currency from the list |
