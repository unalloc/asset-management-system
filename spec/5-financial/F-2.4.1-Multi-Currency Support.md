# F-2.4.1 - Multi-Currency Support

**Requirement ID:** F-2.4.1  
**Module:** Financial Management  
**Priority:** High

---

## Requirement Description

Support asset value tracking and financial reporting in multiple currencies to meet multinational enterprise operational needs. Enable accurate asset value representation across global operations with proper currency conversion using temporal or current rate methods based on asset classification.

---

## Business Rules

1. Currency codes must be valid ISO 4217 codes
2. Exchange rates must be positive and from approved sources (Central Bank, Corporate, or Manual)
3. Temporal method used for assets less than 12 months old (historical rate on transaction date)
4. Current rate method used for investment property and long-term assets (closing rate at reporting date)
5. Realized exchange gains/losses recognized on asset disposal
6. Unrealized exchange gains/losses recorded to OCI or P&L based on asset classification

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset Code | Yes | Unique identifier for the asset |
| Company Code | Yes | Company that owns the asset |
| Functional Currency | Yes | Primary currency for financial reporting |
| Acquisition Currency | Yes | Currency in which asset was purchased |
| Acquisition Cost | Yes | Original cost in acquisition currency |
| Acquisition Date | Yes | When asset was acquired |

---

## Output Data

| Data | Description |
|------|-------------|
| Asset Values | Values in both functional and reporting currency |
| Exchange Gains/Losses | Unrealized and realized currency adjustments |
| Currency Conversion Records | Audit trail of all conversions |

---

## Validation Rules

1. Currency code must be valid ISO 4217 format (3 uppercase letters)
2. Exchange rate must be positive
3. Exchange rate date cannot be in the future
4. Functional currency must be defined for the asset
5. No exchange rate available for acquisition date requires manual rate approval

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full access to all currency functions |
| Company Controller | Manage currency conversions within company |
| Division Finance | View within division |
| Department User | View within department |

---

## Multi-Company Data Scope

- **Data Level**: Company level for assets, global for exchange rates
- **Access**: Users can only view assets within their organizational hierarchy
- **Isolation**: Company Code filters all asset currency queries |

---

## Compliance Requirements

- ASC 830 (US GAAP) foreign currency translation support
- IAS 21 (IFRS) foreign currency translation requirements
- SOX Section 404 internal controls over financial reporting
- GDPR data minimization for currency data (no personal data implications)
