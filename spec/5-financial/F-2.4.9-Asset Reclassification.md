# F-2.4.9 - Asset Reclassification

**Requirement ID:** F-2.4.9  
**Module:** Financial Management  
**Priority:** High

---

## Requirement Description

Manage asset reclassification and impairment testing to reflect true asset value. Includes impairment testing per IAS 36/ASC 360, recoverable amount calculation, impairment loss recognition, and classification changes between asset types.

---

## Business Rules

1. Recoverable amount = MAX(Fair Value - Costs to Sell, Value in Use)
2. Value in Use calculated by discounting future cash flows (DCF approach)
3. Impairment loss = MAX(0, Carrying Amount - Recoverable Amount)
4. Reversal of impairment loss allowed under IAS 36 (not for goodwill)
5. Classification changes require detailed reason documentation
6. Asset must be active or depreciating to reclassify

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset | Yes | Asset being reclassified or tested |
| Reason | Yes | Detailed explanation (minimum 10 characters) |
| New Classification | Yes | Target classification |
| Supporting Documentation | Yes | Evidence for the change |
| Recoverable Amount | Yes | Fair value or value in use |
| Carrying Amount | Yes | Current book value |

---

## Output Data

| Data | Description |
|------|-------------|
| Impairment Record | Loss amount and calculation |
| Reclassification Journal | Before and after values |
| Asset Status Update | Changed classification |
| Audit Trail | Complete change documentation |

---

## Validation Rules

1. Detailed reason required (minimum 10 characters)
2. Supporting documentation must be attached
3. Approval required for all reclassifications
4. Asset must be in Active or Depreciating status
5. Large impairments require additional approval

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full access |
| Company Controller | Approve impairments and reclassifications |
| Division Finance | View within division |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Reclassification at company level only
- **Isolation**: Cross-company reclassification not allowed |

---

## Compliance Requirements

- ASC 360 (US GAAP) impairment testing and recognition
- IAS 36 (IFRS) impairment of assets
- IAS 40 (IFRS) investment property classification
- SOX 404 internal controls over reclassification
