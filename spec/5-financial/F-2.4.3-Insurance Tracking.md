## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-INS-001 | Policy number must be unique | PolicyNumber NOT IN (SELECT PolicyNumber FROM InsurancePolicy) | INS-ERR-001 | "Policy number already exists" |
| PRE-INS-002 | Insurance amount must be positive | CoverageAmount > 0 | INS-ERR-002 | "Coverage amount must be positive" |
| PRE-INS-003 | End date after start date | EndDate > StartDate | INS-ERR-003 | "Policy end date must be after start date" |
| PRE-INS-004 | Premium must be positive | PremiumAmount >= 0 | INS-ERR-004 | "Premium amount cannot be negative" |
| PRE-INS-005 | Asset must exist | AssetID IN (SELECT AssetID FROM Asset) | INS-ERR-005 | "Asset does not exist" |
| PRE-INS-006 | Insurer must be valid | InsurerID IN (SELECT InsurerID FROM InsurerMaster) | INS-ERR-006 | "Invalid insurer" |

**Pre-condition Validation Matrix:**

```
IF (Policy.PolicyNumber IS NOT NULL)
   AND (Policy.CoverageAmount > 0)
   AND (Policy.StartDate < Policy.EndDate)
   AND (Policy.InsurerID IS NOT NULL)
THEN proceed to policy creation
ELSE throw INS-ERR-00X Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-INS-001 | Insurance policy active | Policy.Status = 'ACTIVE' | Query verification |
| POST-INS-002 | Coverage adequate | CoverageAmount >= AssetInsurableValue | Coverage analysis |
| POST-INS-003 | Claims properly recorded | All claims linked to valid policy | Audit verification |

**Post-condition Verification:**

- Verify all high-value assets (> $50,000) have active insurance
- Confirm claim payments properly documented

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
New Asset Acquired
  │
  ▼
Check Insurable Value
  │
  ├─► Value >= InsuranceThreshold ─► Prompt for Insurance Review
  └─► Value < InsuranceThreshold ─► Log and Continue
        │
        ▼
    Insurance Coordinator Reviews
        │
        ├─► Purchase New Policy ─► Link Policy to Asset
        ├─► Extend Existing Policy ─► Add Asset to Policy
        └─► Self-Insure ─► Document Decision
              │
              ▼
          Track Coverage Period
              │
              ▼
          Alert 30/60/90 Days Before Expiration
              │
              ▼
          END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-INS-001 | High-Value Alert | Asset.Value >= 50000 AND NoActivePolicy | Alert Insurance Coordinator | 1 |
| BR-INS-002 | Expiration Warning | DaysToExpiration <= 90 | Send expiration warning | 1 |
| BR-INS-003 | Coverage Gap Detection | Policy.EndDate < CURRENT_DATE + CoverageGapDays | Flag coverage gap | 1 |
| BR-INS-004 | Claim Limit Check | ClaimAmount <= CoverageAmount | Allow or reject | 1 |
| BR-INS-005 | Deductible Application | ClaimAmount - Deductible | Calculate payable | 1 |
| BR-INS-006 | Underinsurance Detection | CoverageAmount < AssetValue × 0.8 | Flag underinsurance | 1 |
| BR-INS-007 | Premium Allocation | AssetPremium = TotalPremium × (AssetValue / TotalPolicyValue) | Calculate per asset | 2 |

**Calculation Logic:**

```
Insurable Value (Replacement Cost) = OriginalCost × CostIndex(CurrentDate) / CostIndex(AcquisitionDate)

Policy Premium Allocation = TotalPolicyPremium × (AssetInsurableValue / TotalPolicyCoverage)

Claim Payable Amount = ClaimAmount - DeductibleAmount (if ClaimAmount > Deductible)

Underinsurance Factor = CoverageAmount / AssetInsurableValue
If Underinsurance Factor < 0.8, Flag for review
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-INS-001 | Policy cancelled mid-term | Pro-rate return premium, alert affected assets | Calculate return premium, require reassignment |
| EC-INS-002 | Asset disposed with active policy | Option to transfer, cancel, or assign to another asset | Workflow for policy disposition |
| EC-INS-003 | Claim exceeds coverage | Pay up to coverage limit, document shortfall | Alert for additional claims handling |
| EC-INS-004 | Multiple policies on one asset | Allow with overlap detection | Track primary/secondary |
| EC-INS-005 | Self-insured assets | Track as self-insured category | Include in risk analysis |
| EC-INS-006 | Asset transferred between companies | Insurance coverage change | Require policy re-review |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| Multiple Policies | Overlapping Coverage | Conflict | Use highest coverage, log conflict |
| Single Policy | Asset Value > Coverage | Underinsurance | Flag for review |
| No Policy | Asset Value > Threshold | Alert | Require decision |
| Claim Pending | Asset Disposal | Block | Complete or transfer claim |

**State Diagram:**

States: DRAFT, PENDING_APPROVAL, ACTIVE, EXPIRED, CANCELLED, CLAIM_PENDING, CLAIM_PAID
