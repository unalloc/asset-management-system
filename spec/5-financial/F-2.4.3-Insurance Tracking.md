# F-2.4.3 - Insurance Tracking

**Requirement ID:** F-2.4.3  
**Module:** Financial Management  
**Priority:** Medium

---

## Requirement Description

Track asset insurance information, policy management, and claims processing. Includes policy tracking with coverage amounts, premiums, and expiration dates; asset insurance linkage; claims filing and tracking; and coverage adequacy alerts for high-value assets.

---

## Business Rules

1. Assets valued over $50,000 require active insurance coverage
2. Policy expiration alerts sent 30, 60, and 90 days before expiration
3. Underinsurance flagged when coverage is less than 80% of asset value
4. Claim amount cannot exceed coverage minus deductible
5. Premium allocated to assets by proportional value (Asset Premium = Total Premium x Asset Value / Total Policy Value)

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Policy Number | Yes | Unique identifier for the insurance policy |
| Insurer | Yes | Insurance company providing coverage |
| Policy Type | Yes | Type of coverage (property, liability, etc.) |
| Coverage Amount | Yes | Maximum amount covered |
| Premium Amount | Yes | Cost of the policy |
| Deductible Amount | Yes | Amount paid by policyholder before insurance applies |
| Start Date | Yes | When coverage begins |
| End Date | Yes | When coverage expires |

---

## Output Data

| Data | Description |
|------|-------------|
| Insurance Policy Record | Complete policy details |
| Insured Asset Linkages | Which assets are covered by which policies |
| Claim Records | Filed and paid claims |
| Coverage Alerts | Warnings about expiration or underinsurance |

---

## Validation Rules

1. Policy number must be unique
2. Coverage amount must be positive
3. End date must be after start date
4. Premium cannot be negative
5. Asset must exist before linking to policy
6. Insurer must be valid and active

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full access |
| Risk Manager | Create and manage policies, file claims |
| Company Controller | View all, modify policies |
| Plant Manager | View within plant |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Policies scoped by Company Code
- **Isolation**: Cross-company insurance not allowed |

---

## Compliance Requirements

- SOX 404 internal controls over insurance documentation
- OSHA workers compensation and liability coverage tracking
- EPA environmental liability coverage for hazardous assets
- GDPR lawful basis documentation for any personal data
