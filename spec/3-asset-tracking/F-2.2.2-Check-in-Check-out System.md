# F-2.2.2 - Check-in-Check-out System

## Requirement ID
F-2.2.2

## Function Name
Check-in/Check-out System

## Function Description
Manage asset lending and return processes, recording borrower information and duration.

## Business Rules

1. Loan requires application and approval
2. Loan period is configurable
3. Automatic reminders for overdue items
4. Complete loan history recorded

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Asset ID | Yes | Target asset |
| Borrower ID | Yes | Person borrowing |
| Expected Loan Duration | Yes | Duration |
| Purpose | Yes | Reason for loan |
| Approver ID | Yes | Approval authority |

## Output Data

1. Loan ticket number
2. Loan status
3. Expected return date
4. Loan records

## Validation Rules

1. Asset must be available (not under maintenance or disposal)
2. Borrower must have valid account
3. Loan duration cannot exceed maximum limit

## Priority
High
