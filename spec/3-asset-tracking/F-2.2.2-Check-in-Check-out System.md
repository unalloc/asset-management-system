# F-2.2.2 - Check-in-Check-out System

**Requirement ID:** F-2.2.2  
**Module:** Asset Tracking & Monitoring (2.2)  
**Priority:** High

---

## Requirement Description

Manage asset lending and return processes, recording borrower information and duration. The system supports configurable loan periods, automated reminders for overdue items, complete loan history, and approval workflows for both internal employee loans and external customer/partner loans.

---

## Business Rules

1. Assets must be in Active status and not already allocated to be checked out
2. Borrowers must have active accounts and required training for the asset type
3. Loan duration must be within configured policy limits
4. High-value assets require enhanced approval workflows
5. Overdue items trigger escalation to managers
6. Lost items are flagged after extended overdue periods
7. Loan renewals require re-approval after limit reached

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Asset ID | Yes | Asset being loaned |
| Borrower ID | Yes | Employee receiving the asset |
| Loan Type | Yes | Type of loan |
| Requested Duration | Yes | Length of loan in days |
| Purpose | Yes | Business justification for loan |

---

## Output Data

| Data | Description |
|------|-------------|
| Loan Ticket | Record of the loan with unique ID |
| Loan History | Complete history of loan actions |
| Reminder Schedule | Scheduled reminders for due dates |
| Approval Record | Audit trail of approvals |

---

## Validation Rules

1. Asset must be available (Active status, not allocated)
2. Borrower must have active account
3. Borrower must have required training for asset type
4. Loan duration must not exceed policy maximum
5. Purpose description must be at least 10 characters
6. Approver must have authority for the loan type

---

## Access Control

| Role | Permission |
|------|------------|
| System Admin | Full loan management |
| Company Admin | Full management within company |
| Division Manager | Approve within division |
| Plant Manager | Approve within plant |
| Department User | Request and return own loans |

---

## Multi-Company Data Scope

- **Data Level**: Company level
- **Access**: Loans scoped to company and organizational hierarchy
- **Isolation**: Company data isolated with access controls

---

## Compliance Requirements

- **SOX**: Asset custody controls for capitalized assets
- **OSHA**: Safety equipment check-out tracking
- **GDPR**: Borrower data protected with lawful basis

---

## Priority

**High** - Essential for asset custody control and operational tracking
