### 1.1 Requirement Description

Manage asset lending and return processes, recording borrower information and duration. The system shall support configurable loan periods, automated reminders for overdue items, complete loan history, and approval workflows. This requirement supports both internal employee loans and external customer/partner loans.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-LOAN-001 | Asset must be available | `Asset.Status = 'Active' AND Asset.IsAllocated = FALSE` | LOAN-001 | "Asset is not available for checkout. Status: {status}" |
| PRE-LOAN-002 | Borrower account valid | `Borrower.Status = 'Active' AND Borrower.EmployeeID IS NOT NULL` | LOAN-002 | "Borrower account is not active or invalid" |
| PRE-LOAN-003 | Borrower has required training | `Training.IsValid(Asset.AssetType, Borrower.PersonID) = TRUE` | LOAN-003 | "Borrower lacks required training for asset type: {assetType}" |
| PRE-LOAN-004 | Loan duration within limits | `RequestedDuration <= LoanPolicy.MaxDuration` | LOAN-004 | "Requested loan duration exceeds maximum limit of {max} days" |
| PRE-LOAN-005 | Purpose provided | `Purpose IS NOT NULL AND LENGTH(Purpose) >= 10` | LOAN-005 | "Loan purpose must be at least 10 characters" |
| PRE-LOAN-006 | Approver authorized | `Approver.HasAuthority(LoanType) = TRUE` | LOAN-006 | "Approver is not authorized for this loan type" |

**Pre-condition Validation Matrix:**

```
IF Asset.Status = 'Active'
   AND Asset.IsAllocated = FALSE
   AND Borrower.Status = 'Active'
   AND Training.IsValid(Asset.AssetType, Borrower.PersonID)
   AND RequestedDuration <= LoanPolicy.MaxDuration
   AND Purpose IS NOT NULL
   AND Approver.HasAuthority(LoanType)
THEN proceed to approval workflow
ELSE throw PRE-LOAN-00X Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-LOAN-001 | Loan ticket created | Loan record with unique LoanTicketID | Database query |
| POST-LOAN-002 | Asset status updated | Asset.IsAllocated = TRUE, Asset.CurrentHolder = BorrowerID | Asset query |
| POST-LOAN-003 | Reminder scheduled | Reminder jobs created for due date and overdue | Scheduler verification |
| POST-LOAN-004 | Approval recorded | Approval record with timestamp and approver | Audit query |

**Post-condition Verification:**

- Loan record created with unique identifier
- Asset.IsAllocated flag set to TRUE
- Asset.CurrentHolder updated to BorrowerID
- Calendar reminders scheduled (3 days before, due date, 1 day overdue, 7 days overdue)
- Email/SMS notifications dispatched

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Loan Request Submitted
  │
  ▼
Validate Pre-conditions ────► FAIL ────► Reject with error code
  │
  ▼ PASS
Determine Approval Route
  │
  ├──► Standard Loan (<= 7 days, value < $1000) ────► Auto-approve ────► Issue Ticket
  │
  ├──► Extended Loan (> 7 days) ────► Manager Approval ────► Approved?
  │                                                │
  │                                                ├─► Yes ────► Issue Ticket
  │                                                └─► No ────► Reject
  │
  └──► High Value (>= $5000) ────► Multi-level Approval ────► All Approved?
                                                          │
                                                          ├─► Yes ────► Issue Ticket
                                                          └─► No ────► Reject
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-LOAN-001 | Auto-approve Standard | Standard loan criteria met | Approve immediately | 1 |
| BR-LOAN-002 | Overdue Escalation | ReturnDate < TODAY AND daysOverdue >= 7 | Escalate to manager | 1 |
| BR-LOAN-003 | Lost Item Flag | No return within 30 days of due date | Flag as potentially lost | 2 |
| BR-LOAN-004 | Renewal Limit | RenewalCount >= 2 | Require re-approval | 2 |
| BR-LOAN-005 | Late Return Penalty | daysOverdue > 0 | Apply penalty per policy | 3 |

**Calculation Logic:**

```
DaysOverdue = ReturnDate - DueDate
LoanPenalty = DaysOverdue * DailyPenaltyRate
RenewalCount = Number of times loan was renewed
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-LOAN-001 | Asset under maintenance during loan | Reject loan if Asset.Status != 'Active' | Return PRE-LOAN-001 |
| EC-LOAN-002 | Borrower employment ends with asset | Trigger immediate return workflow | Manager notification |
| EC-LOAN-003 | Asset transfer requested while on loan | Queue transfer until return | Return 409 Conflict |
| EC-LOAN-004 | Borrower loses asset | File report, apply penalty | Incident workflow |
| EC-LOAN-005 | Multiple loans same asset | Prevent concurrent loans | Serial loan enforcement |

**Edge Case Decision Matrix:**

| Asset Status | Borrower Active | Has Training | Training Valid | Outcome |
|--------------|-----------------|--------------|----------------|---------|
| Active | Yes | Yes | Yes | Approve |
| Active | Yes | No | N/A | Reject - training |
| Active | No | N/A | N/A | Reject - inactive |
| Maintenance | Yes | Yes | Yes | Reject - status |
| Disposed | N/A | N/A | N/A | Reject - not found |

### 3.5 State Transitions

**State Diagram:**

```
States: AVAILABLE, RESERVED, CHECKED_OUT, OVERDUE, RETURNED, LOST

┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│ AVAILABLE   │──────►│  RESERVED   │──────►│ CHECKED_OUT │
└─────────────┘       └─────────────┘       └─────────────┘
       │                                            │
       │                                            │
       ▼                                            ▼
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   RETURNED  │◄──────│   OVERDUE   │◄──────│ CHECKED_OUT │ (past due)
└─────────────┘       └─────────────┘       └─────────────┘
       │                                            │
       │                                            ▼
       │                                     ┌─────────────┐
       └─────────────────────────────────────►│    LOST     │
                                              └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| AVAILABLE | Reserve | RESERVED | Borrower validated | Asset locked |
| RESERVED | Check-out | CHECKED_OUT | Borrower confirms pickup | Update holder |
| CHECKED_OUT | Due date passes | OVERDUE | DueDate < TODAY | Send alert |
| OVERDUE | Return | RETURNED | Asset returned | Clear holder |
| OVERDUE | 30 days overdue | LOST | No return + investigation | Incident created |
| CHECKED_OUT | Return | RETURNED | Asset in acceptable condition | Clear holder |
| RESERVED | Cancel | AVAILABLE | Borrower cancels | Unlock asset |
