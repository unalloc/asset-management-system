# F-2.2.8 - Downtime Tracking

## Requirement ID
F-2.2.8

## Function Name
Downtime Tracking

## Function Description
Record and categorize equipment downtime and analyze downtime reasons.

## Business Rules

1. Standardized downtime reason categories
2. Downtime start/end recording
3. Production impact analysis
4. Downtime trend reporting

## Input Data

| Field | Required | Description |
|-------|----------|-------------|
| Downtime Start Time | Yes | When stopped |
| Downtime End Time | Yes | When resumed |
| Downtime Category | Yes | Reason classification |
| Description | No | Details |
| Impact Scope | No | Areas affected |

## Output Data

1. Downtime record
2. Category statistics
3. Trend analysis
4. Improvement recommendations

## Validation Rules

1. Downtime periods cannot overlap
2. Category must be valid
3. Start time must be before end time

## Priority
High (for manufacturing enterprises)

## Related User Stories
US-042
