# F-2.2.5 - Mobile Asset Verification

## Requirement ID
F-2.2.5

## Function Name
Mobile Asset Verification

## Function Description
Provide mobile applications for field technicians to verify asset location and condition with offline operation support, barcode/QR code scanning, photo attachments, and real-time synchronization.

### 1.1 Requirement Description

Provide mobile applications for field technicians to verify asset location and condition. The system shall support offline operation, barcode/QR code scanning, photo attachments, and real-time synchronization when connectivity is available. Verification results shall update the asset master and trigger exception reporting for discrepancies.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-VER-001 | Asset identifier valid | `Asset.Barcode OR Asset.QRCode OR Asset.RFIDTag = Identifier` | VER-001 | "Asset identifier not recognized" |
| PRE-VER-002 | Technician authorized | `Technician.HasVerificationRights = TRUE` | VER-002 | "Technician not authorized for verification" |
| PRE-VER-003 | Verification within schedule | `Asset.NextVerificationDate >= TODAY - GracePeriod` | VER-003 | "Verification not yet due or overdue beyond grace period" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-VER-001 | Verification recorded | Verification record with timestamp | Database query |
| POST-VER-002 | Asset condition updated | Asset.ConditionScore updated | Asset query |
| POST-VER-003 | Exception reported | Exception record created if discrepancy | Exception report |
| POST-VER-004 | Sync completed | Cloud sync confirmed (when online) | Sync status |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Scan Asset Identifier
  │
  ▼
Validate Pre-conditions ────► FAIL ────► Log error, allow retry
  │
  ▼ PASS
Display Asset Details
  │
  ▼
Capture Verification Data
  │
  ├──► Condition Rating (1-5)
  ├──► Location Confirmation (Y/N)
  ├──► Photo Attachments (optional)
  ├──► GPS Location (optional)
  └──► Notes
  │
  ▼
Evaluate Discrepancies
  │
  ├──► No Discrepancy ────► Mark Verified ────► Sync to Cloud
  │
  └──► Discrepancy Found ────► Create Exception ────► Log and Continue
                                                    │
                                                    ▼
                                              Sync to Cloud
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-VER-001 | Auto-condition update | Verification completed | Recalculate Asset.ConditionScore | 1 |
| BR-VER-002 | Verification overdue alert | Date > NextVerification + GracePeriod | Alert manager | 1 |
| BR-VER-003 | Photo compression | Photo size > 5MB | Compress before storage | 2 |
| BR-VER-004 | Offline queue limit | Queue size > 100 items | Warn user | 3 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-VER-001 | No network during sync | Queue locally, sync when online | Local storage + retry |
| EC-VER-002 | Duplicate scan | Ignore duplicate within 5 minutes | Debounce |
| EC-VER-003 | Conflict on sync | Last-write-wins with conflict log | Conflict resolution |
| EC-VER-004 | Unknown asset scanned | Create ad-hoc verification record | Exception workflow |
| EC-VER-005 | Low storage | Prevent new photos until sync | Warning + block |
