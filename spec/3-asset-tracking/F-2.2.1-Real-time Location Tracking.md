### 1.1 Requirement Description

Track asset locations in real-time using GPS, WiFi, or RFID technology with geofencing support and boundary violation alerts. The system shall maintain location history, support configurable update frequencies, and provide immediate notification of boundary violations through multiple channels (email, SMS, in-app).

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-LOC-001 | Asset must exist and be active | `Asset.Status = 'Active' AND Asset.AssetID IS NOT NULL` | LOC-001 | "Asset must exist and be in active status to update location" |
| PRE-LOC-002 | Location coordinates must be valid | `Latitude >= -90 AND Latitude <= 90 AND Longitude >= -180 AND Longitude <= 180` | LOC-002 | "Coordinates are outside valid geographic range" |
| PRE-LOC-003 | Timestamp cannot be in future | `Timestamp <= CURRENT_TIMESTAMP + 5 minutes (clock skew tolerance)` | LOC-003 | "Location timestamp cannot be in the future" |
| PRE-LOC-004 | Device info required for audit | `DeviceInfo IS NOT NULL OR SourceSystem IS NOT NULL` | LOC-004 | "Device or source system information required for tracking" |

**Pre-condition Validation Matrix:**

```
IF Asset exists AND Asset.Status = 'Active'
   AND Latitude IS VALID (-90 to 90)
   AND Longitude IS VALID (-180 to 180)
   AND Timestamp <= NOW + 5 minutes
   AND (DeviceInfo IS NOT NULL OR SourceSystem IS NOT NULL)
THEN proceed to location update
ELSE throw PRE-LOC-00X Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-LOC-001 | Location record created | LocationHistory record created with all fields | Database query |
| POST-LOC-002 | Geofence status evaluated | GeofenceStatus updated if asset crosses boundary | Alert generated |
| POST-LOC-003 | Real-time display updated | Asset.CurrentLatitude/Longitude updated | API response |

**Post-condition Verification:**

- Asset location fields (CurrentLatitude, CurrentLongitude, LastLocationUpdate) are updated
- LocationHistory record appended with immutable flag
- Geofence evaluation triggered and alerts dispatched if boundary crossed
- Event published for real-time subscribers

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Receive Location Update
  │
  ▼
Validate Pre-conditions ────► FAIL ────► Reject with error code
  │
  ▼ PASS
Update Asset Current Location
  │
  ▼
Evaluate Geofences
  │
  ├──► No Boundary Cross ────► Log location only ────► END
  │
  └──► Boundary Cross Detected
         │
         ▼
    Determine Alert Type
         │
         ├─► ENTER ────► Generate Entry Alert ────► END
         │
         ├─► EXIT ────► Generate Exit Alert ────► END
         │
         └─► DWELL_TIME_EXCEEDED ────► Generate Dwell Alert ────► END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-LOC-001 | Location Update Frequency | Configurable per asset type | Store last N locations per retention policy | 1 |
| BR-LOC-002 | Geofence Evaluation | Asset crosses geofence boundary | Trigger alert via configured channel | 1 |
| BR-LOC-003 | Stale Location Alert | No update received within configured threshold | Generate stale data warning | 2 |
| BR-LOC-004 | Location Accuracy Filter | Accuracy > configured threshold | Flag low-quality reading | 3 |
| BR-LOC-005 | Historical Data Pruning | Location age > 180 days (configurable) | Archive to cold storage | 4 |

**Calculation Logic:**

```
Distance = HAVERSINE(Lat1, Lon1, Lat2, Lon2)
DwellTime = LastExitTimestamp - LastEntryTimestamp
Speed = Distance / TimeSinceLastUpdate
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-LOC-001 | GPS signal loss during update | Use last known location with stale flag | Store with accuracy=-1, flag as estimated |
| EC-LOC-002 | Rapid geofence crossing (bounce) | Debounce within 30-second window | Suppress duplicate alerts |
| EC-LOC-003 | Location update during asset transfer | Queue update until transfer completes | Return 409 Conflict |
| EC-LOC-004 | Simultaneous updates for same asset | Last-write-wins with timestamp ordering | Use vector clock for ordering |
| EC-LOC-005 | Geofence polygon self-intersecting | Reject geofence definition | Validation error GF-002 |

**Edge Case Decision Matrix:**

| Condition | Condition | Condition | Outcome |
|-----------|-----------|-----------|---------|
| GPS Available | Accuracy <= 10m | Same geofence | Process update, no alert |
| GPS Unavailable | WiFi fallback | Same geofence | Process with warning, flag accuracy |
| No signal | Last update > 1 hour | Boundary crossed | Generate stale + boundary alert |

### 3.5 State Transitions

**State Diagram:**

```
States: UNKNOWN, TRACKED, STALE, IN_TRANSIT, DISPOSED

┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   UNKNOWN   │──────►│   TRACKED   │──────►│    STALE    │
└─────────────┘       └─────────────┘       └─────────────┘
                            │                     │
                            │                     │
                            ▼                     ▼
                     ┌─────────────┐       ┌─────────────┐
                     │ IN_TRANSIT  │       │   TRACKED   │ (update received)
                     └─────────────┘       └─────────────┘
                            │
                            ▼
                     ┌─────────────┐
                     │  DISPOSED   │
                     └─────────────┘
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| UNKNOWN | First location update | TRACKED | Valid coordinates | Asset location initialized |
| TRACKED | No update for 24 hours | STALE | Configurable threshold | Alert generated |
| STALE | Update received | TRACKED | Valid coordinates | Clear stale flag |
| TRACKED | Transfer initiated | IN_TRANSIT | Transfer workflow started | Location locked |
| IN_TRANSIT | Transfer completed | TRACKED | Destination confirmed | Unlock location |
| ANY | Disposal | DISPOSED | Disposal workflow completed | Stop tracking |
