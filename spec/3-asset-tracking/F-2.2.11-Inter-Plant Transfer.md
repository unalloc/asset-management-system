# F-2.2.11 - Inter-Plant Transfer

## Requirement ID
F-2.2.11

## Function Name
Inter-Plant Transfer

## Function Description
Manage asset transfer workflows between different plants including multi-level approvals, transportation arrangement and tracking, cost allocation, and complete documentation. Extends the base Transfer Management (F-2.2.3) with inter-plant specific workflows, cross-plant cost allocation, and transportation logistics.

## Business Purpose
Enable efficient movement of assets between plants within an organization while maintaining proper controls, approvals, and documentation. Supports multi-plant operations by providing visibility into transfer status, associated costs, and transportation logistics. Ensures regulatory compliance for hazardous materials and provides audit trail for SOX compliance.

## Core Business Rules

1. **Plant Relationship**: Source and target plants must be in the same company or same division.

2. **Different Plants**: Source and target plants must be different—cannot transfer to the same plant.

3. **Asset Eligibility**: Only active assets not currently under transfer can be transferred.

4. **Multi-Level Approval**: Inter-plant transfers require approval from multiple levels:
   - Same Division: Division Manager + both Plant Managers
   - Different Division: Both Division Managers + Company Controller

5. **Transportation Arrangement**: Transportation must be booked and mode specified before transfer proceeds.

6. **Cost Allocation**: All inter-plant transfers must have costs calculated and distributed between source and target plants.

7. **ETA Notification**: Target plant must be notified when transportation is arranged.

8. **Arrival Verification**: Assets must be verified upon arrival before transfer is complete.

9. **Inter-Company Transfer Pricing**: Transfers between different companies require documented transfer pricing.

10. **Hazmat Handling**: Assets with hazardous materials require proper permits for transport.

## User Roles and Permissions

| Role | Same Company | Same Division | Same Plant | Cross-Plant View |
|------|--------------|---------------|------------|------------------|
| System Admin | Full | Full | Full | Full |
| Company Admin | Full | Full | Full | Full |
| Division Manager | Full | Full | Own plant | Same division |
| Plant Manager | Full | Full | Full | None |
| Logistics | Full | Full | Full | Transport info only |

## Data Scope

- **Creates Data:** Users create transfer requests; System generates tickets, bookings, cost allocations
- **Views Data:** Same company/division personnel; cross-plant view depends on role
- **Modifies Data:** Plant managers own plant transfers; logistics manages transportation

## Multi-Company Hierarchy

| Dimension | Level | Creates Data | Views Data | Modifies Data |
|-----------|-------|--------------|------------|---------------|
| Legal Entity | Company | Yes | Yes | No |
| Business Unit | Division | Yes | Yes | No |
| Operations | Plant | Yes | Yes | No |

**Cross-Company Transfers:** Require multi-level approval including company administrators for both companies.

## Compliance Notes

**SOX:**
- Inter-plant transfers require proper authorization
- Transfer pricing documentation required for inter-company transfers
- Complete audit trail for all transfer-related costs

**EPA:**
- Hazardous materials tracking across plant boundaries under RCRA
- TSCA chemical equipment transfer tracking (30-year retention)
- Manifest documentation required for regulated transfers

**GDPR:**
- Employee personal data in approval chain (approvers, requestors)
- Lawful basis: Contract performance, legitimate interest

**GAMP 5 (Pharma):**
- Equipment qualification transfer between GxP facilities

## Transfer States

| State | Description |
|-------|-------------|
| DRAFT | Transfer request being prepared |
| PENDING_SOURCE_APPROVAL | Awaiting source plant approval |
| PENDING_TARGET_APPROVAL | Awaiting target plant approval |
| PENDING_TRANSIT | Approved, awaiting transportation |
| IN_TRANSIT | Assets in transit |
| ARRIVED | Assets arrived at destination |
| PENDING_RECEIVING | Awaiting receiving verification |
| COMPLETED | Transfer fully completed |
| REJECTED | Transfer rejected |
| CANCELLED | Transfer cancelled |

## Approval Routes

| Transfer Type | Approval Route |
|--------------|---------------|
| Same Division | Division Manager → Source Plant Manager → Target Plant Manager |
| Different Division | Source Division Manager → Target Division Manager → Company Controller |

## Edge Cases

| Scenario | Expected Behavior |
|----------|-------------------|
| Transportation delayed | Alert relevant parties, extend approval window |
| Asset damaged in transit | Create incident record, initiate insurance claim process |
| Transfer rejected at destination | Return workflow initiated |
| Partial transfer | Complete transfer for received assets, track missing items |

## Transportation Modes

- Truck
- Rail
- Air
- Sea
- Courier

## Cost Allocation

Transfer costs are distributed across:
- Source plant cost portion
- Target plant cost portion
- Transportation cost

High (for enterprise groups)
