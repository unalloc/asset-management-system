# F-2.2.3 - Transfer Management

## Requirement ID
F-2.2.3

## Function Name
Transfer Management

## Function Description
Manage asset transfers between locations, departments, or users with approval workflow, batch transfer support, and transfer cost tracking.

### 1.1 Requirement Description

Manage asset transfers between locations, departments, or users with full approval workflow, batch transfer support, pre and post transfer status confirmation, and transfer cost tracking. Transfers must maintain complete audit trail and support inter-company and inter-plant scenarios.

- Right-of-use asset transfers must be tracked for lease compliance

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-TRANS-001 | Assets must exist and be transferable | `Asset.Status = 'Active' AND Asset.IsUnderTransfer = FALSE` | TRANS-001 | "Asset is not transferable. Status: {status}" |
| PRE-TRANS-002 | Source location valid | `SourceLocation IS NOT NULL AND SourceLocation.Type IN ('DEPARTMENT','USER','PLANT')` | TRANS-002 | "Invalid source location" |
| PRE-TRANS-003 | Target location valid | `TargetLocation IS NOT NULL AND TargetLocation.Type IN ('DEPARTMENT','USER','PLANT')` | TRANS-003 | "Invalid target location" |
| PRE-TRANS-004 | Source != Target | `SourceLocation != TargetLocation` | TRANS-004 | "Source and target locations must be different" |
| PRE-TRANS-005 | Transfer reason required | `TransferReason IS NOT NULL AND LENGTH(TransferReason) >= 10` | TRANS-005 | "Transfer reason must be at least 10 characters" |
| PRE-TRANS-006 | Batch size within limits | `AssetCount <= BatchTransferLimit` | TRANS-006 | "Batch size exceeds maximum limit of {limit}" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-TRANS-001 | Transfer ticket created | Transfer record with unique TransferTicketID | Database query |
| POST-TRANS-002 | Asset status updated | Asset.IsUnderTransfer = TRUE (if locked during transfer) | Asset query |
| POST-TRANS-003 | Approval recorded | Approval records with timestamp | Audit query |
| POST-TRANS-004 | Asset location updated | Asset.Location = TargetLocation (after completion) | Asset query |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Transfer Request Submitted
  │
  ▼
Validate Pre-conditions ────► FAIL ────► Reject with error
  │
  ▼ PASS
Determine Approval Route
  │
  ├──► Same Plant/Department ────► Department Manager ────► END
  │
  ├──► Different Plant (same Division) ────► Division Manager ────► END
  │
  └──► Different Division/Company ────► Multi-level Approval ────► END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-TRANS-001 | Same-day transfer limit | User requests > 5 transfers/day | Require manager override | 1 |
| BR-TRANS-002 | High-value transfer flag | Asset.Value >= 50000 | Enhanced approval | 1 |
| BR-TRANS-003 | Inter-plant notification | SourcePlant != TargetPlant | Notify both plant managers | 2 |
| BR-TRANS-004 | Transfer cost calculation | Any transfer | Calculate and record cost | 2 |
| BR-TRANS-005 | Batch approval threshold | BatchCount > 10 | Split for review | 3 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-TRANS-001 | Asset on loan during transfer | Reject or queue transfer | Return loan first |
| EC-TRANS-002 | Asset under maintenance | Block transfer until complete | Maintenance completion required |
| EC-TRANS-003 | Target location capacity full | Block transfer | Require capacity confirmation |
| EC-TRANS-004 | Partial batch approval | Allow partial approval | Process approved subset |
