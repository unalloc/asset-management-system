# F-2.2.12 - Calibration Management

## Requirement ID

## Function Description
Manage calibration schedules and records for measurement and test equipment including cycle management, certificate management, overdue alerts, and measurement uncertainty tracking.

### 1.1 Requirement Description

Manage calibration schedules and records for measurement and test equipment. The system shall support calibration cycle management, calibration certificate management with document attachment, overdue alerts, and measurement uncertainty tracking. This requirement is critical for quality management systems (ISO 9001), regulatory compliance (FDA 21 CFR Part 820), and GAMP 5 pharmaceutical manufacturing.

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-CAL-001 | Equipment registered | `Equipment.IsCalibrationRequired = TRUE` | CAL-001 | "Equipment does not require calibration" |
| PRE-CAL-002 | Calibration due | `Equipment.NextCalibrationDate <= TODAY` | CAL-002 | "Calibration not yet due" |
| PRE-CAL-003 | Technician qualified | `Technician.HasCalibrationCertification(Equipment.Type) = TRUE` | CAL-003 | "Technician not qualified for this equipment type" |
| PRE-CAL-004 | Calibration procedure available | `CalibrationProcedure.Exists(Equipment.Type) = TRUE` | CAL-004 | "Calibration procedure not found" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-CAL-001 | Calibration record created | Record with pass/fail result | Database query |
| POST-CAL-002 | Certificate attached | Certificate document stored | Document query |
| POST-CAL-003 | Next date calculated | Equipment.NextCalibrationDate updated | Equipment query |
| POST-CAL-004 | Alert cleared | Overdue alert removed if passed | Alert query |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Calibration Event Triggered
  │
  ▼
Validate Pre-conditions ────► FAIL ────► Reject with error
  │
  ▼ PASS
Retrieve Calibration Procedure
  │
  ▼
Perform Calibration
  │
  ├──► Record As-Found Values
  ├──► Perform Adjustment (if needed)
  ├──► Record As-Left Values
  └──► Calculate Uncertainty
  │
  ▼
Evaluate Results
  │
  ├──► PASS ────► Update Next Date ────► Attach Certificate ────► END
  │
  └──► FAIL ────► Generate Exception ────► Quarantine Equipment ────► END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-CAL-001 | Auto-schedule | Calibration completed | Calculate next due date | 1 |
| BR-CAL-002 | Overdue alert | NextCalibrationDate < TODAY | Generate alert | 1 |
| BR-CAL-003 | Quarantine failed | Result = FAIL | Set Equipment.Status = QUARANTINE | 1 |
| BR-CAL-004 | Trend analysis | 3+ consecutive passes | Extend interval if allowed | 2 |
| BR-CAL-005 | Certificate expiration | Certificate expiry approaching | Warn user | 2 |

**Calculation Logic:**

```
NextCalibrationDate = CalibrationDate + CalibrationInterval
MeasurementUncertainty = k * sqrt(sum(squared_std_deviations))
TrendScore = MovingAverage(Last3Readings)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-CAL-001 | Equipment failed calibration | Quarantine, generate exception | Exception workflow |
| EC-CAL-002 | Technician unqualified | Reject calibration attempt | Pre-condition check |
| EC-CAL-003 | Calibration data lost | Revert to last known good | Restore from backup |
| EC-CAL-004 | Procedure updated mid-calibration | Complete with current procedure | Version control |
| EC-CAL-005 | External lab calibration | Record external certificate | Manual entry + upload |

### 3.5 State Transitions

**State Diagram:**

```
Equipment States: CALIBRATED, CALIBRATION_DUE, OVERDUE, QUARANTINE

Calibration States: SCHEDULED, IN_PROGRESS, PASSED, FAILED, CANCELLED

┌───────────────┐       ┌────────────────┐
│  CALIBRATED   │──────►│ CALIBRATION_DUE │
└───────────────┘       └────────────────┘
       │                       │
       │                       ▼
       │                ┌─────────────┐
       │                │  OVERDUE    │
       │                └─────────────┘
       │                       │
       ▼                       ▼
┌───────────────┐       ┌────────────────┐
│ QUARANTINE    │◄──────│ IN_PROGRESS   │
└───────────────┘       └────────────────┘
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
              ┌─────────────┐     ┌─────────────┐
```
