# F-2.7.4 Real-time Asset Availability Tracking

## Function ID
F-2.7.4

## Function Name
Real-time Availability Monitoring

## Function Description
Provide real-time asset availability tracking, including current status, location, user, estimated available time, supporting dynamic updates and notifications.

## Business Rules
1. Status types: Available, In Use, Under Maintenance, Reserved, Out of Service, Being Disposed
2. Real-time update: status change immediately reflected on all interfaces
3. Location tracking: asset current location (GPS, RFID, manual update)
4. User tracking: current user/responsible person information
5. Estimated available time: based on current usage end time or maintenance completion time
6. Status history: status change history for analysis and audit
7. Automatic status transition: automatically update status based on rules (e.g., maintenance complete → available)
8. Multi-dimensional view: filter and group by status, location, type, etc.

## Preconditions
- Asset status tracking enabled
- Status update mechanism ready (manual or automatic)
- User has permission to view status
- Real-time communication infrastructure available

## Postconditions
- Status information is latest and accurate
- Relevant parties receive status change notification (if needed)
- History record saved
- Dashboard data updated

## Input Data
1. Asset ID
2. New status
3. Status change time
4. Change reason
5. Location information (if changed)
6. User information (if changed)
7. Estimated available time
8. Changed by/user/system

## Output Data
1. Real-time status display
2. Status change notification
3. Status history report
4. Available asset list
5. Estimated availability schedule
6. Abnormal status alert

## Validation Rules
1. Status change must be reasonable (e.g., cannot change from "Being Disposed" to "Available")
2. Location information must be valid (existing location)
3. User must be a valid employee
4. Estimated time must be based on reliable data
5. Changer must have corresponding permissions

## Error Handling
1. Invalid status transition: Reject change and show allowed transitions
2. Invalid location: Prompt to select valid location
3. Insufficient permission: Reject change and notify administrator
4. Data inconsistency: Trigger data validation process

## Priority
High
