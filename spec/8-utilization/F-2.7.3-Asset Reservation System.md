# F-2.7.3 - Asset Reservation System

**Requirement ID:** F-2.7.3  
**Module:** Asset Utilization  
**Priority:** High

---

## Requirement Description

Provide asset reservation functionality, support advance booking of asset usage, automatically detect and resolve reservation conflicts.

---

## Business Rules

1. Assets under repair, being disposed, or retired cannot be reserved
2. User must have reservation permission
3. HARD reservation blocks all conflicting reservations
4. SOFT reservation can be overridden by higher priority
5. Priority order: EMERGENCY > EXECUTIVE > PRODUCTION > REGULAR
6. Max reservation duration enforced by asset type
7. Advance booking limit enforced (configurable days)
8. Cancellation within 24 hours of start records violation

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| ReservationNumber | Yes | Format: RSV-YYYYMMDD-SEQ |
| AssetID | Yes | UUID of asset to reserve |
| ReserverUserID | Yes | UUID of user making reservation |
| StartDateTime | Yes | Reservation start time |
| EndDateTime | Yes | Reservation end time |
| ReservationType | Yes | HARD or SOFT |
| Priority | Yes | Priority level (1-10) |

---

## Output Data

| Data | Description |
|------|-------------|
| ReservationID | System-generated UUID |
| ConflictRecord | Details if conflict detected |
| ViolationRecord | If cancellation penalty applied |

---

## Validation Rules

1. Reservation number follows format RSV-YYYYMMDD-SEQ
2. Duration must not exceed maximum for asset type
3. Days in advance must not exceed booking limit
4. Time conflict check prevents double-booking
5. Asset must be in reservable status

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all reservations |
| DepartmentManager | View department reservations |
| RegularUser | Create and manage own reservations |

---

## Multi-Company Data Scope

- **Data Level**: Company/Department
- **Access**: Reservations scoped to company
- **Isolation**: CompanyCode mandatory on all queries

---

## Compliance Requirements

- GDPR: Reservation records anonymized; minimal data collection
- ISO 55000: Asset allocation aligned with operational control

---
