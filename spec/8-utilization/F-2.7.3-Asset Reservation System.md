## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-2.7.3-001 | Asset reservable | `asset.status NOT IN ('UNDER_REPAIR','BEING_DISPOSED','RETIRED')` | ERR-PRE-273-001 | "Asset cannot be reserved (under repair, disposal, or retired)" |
| PRE-2.7.3-002 | Requester has permission | `user.reservationPermission = TRUE` | ERR-PRE-273-002 | "User does not have reservation permission" |
| PRE-2.7.3-003 | Reservation rules defined | `reservationRules.length > 0` | ERR-PRE-273-003 | "Reservation rules must be configured" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-2.7.3-001 | Reservation created | `reservation.id IS NOT NULL` | Query reservation |
| POST-2.7.3-002 | Asset status updated | `IF type='HARD' THEN asset.status = 'RESERVED'` | Query asset status |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Validate Pre-conditions
  │
  ▼
Check Availability
  │
  ├─► Available ─► Check for Conflicts
  │
  ├─► Soft Reserved ─► Apply Priority Rules
  │
  └─► Hard Reserved ─► Reject or Suggest Alternative
           │
           ▼
Priority Resolution (EMERGENCY > EXECUTIVE > PRODUCTION > REGULAR)
           │
           ▼
Create Reservation
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.3-001 | Hard reservation guarantee | `type = 'HARD'` | Block all conflicting reservations | 1 |
| BR-2.7.3-002 | Soft reservation override | `type = 'SOFT'` AND `priority > conflictingPriority` | Override conflicting reservation | 1 |
| BR-2.7.3-003 | Max reservation duration | `duration > maxDuration[assetType]` | Reject with max duration info | 1 |
| BR-2.7.3-004 | Advance booking limit | `daysInAdvance > maxAdvanceDays` | Reject with limit info | 2 |
| BR-2.7.3-005 | Cancellation penalty | `cancelled < 24h before start` | Record violation | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-2.7.3-001 | Double booking attempt | Reject later booking | Show first booking priority |
| EC-2.7.3-002 | Reservation limit exceeded | Show limit reached | Suggest different asset/time |
| EC-2.7.3-003 | No-show handling | Mark as no-show after 30min | Apply violation record |

### 3.5 State Transitions

**State Diagram:**

```
States: DRAFT, CONFIRMED, CHECKED_IN, CHECKED_OUT, CANCELLED, NO_SHOW, CONFLICT_RESOLVED

Transitions:
DRAFT ──► CONFIRMED ──► CHECKED_IN ──► CHECKED_OUT
              │              │
              ▼              ▼
         CANCELLED      CANCELLED
              │
              ▼
      CONFLICT_RESOLVED (automatic)
```

- Cross-company reservations require inter-company agreement
