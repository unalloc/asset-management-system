### 3.3 Business Logic

**Load Indicators:**
- UTILIZATION_RATE: Current utilization vs capacity
- WAIT_QUEUE: Requests waiting for asset
- USAGE_COST: Cost per department

**Decision Tree:**

```
START
  │
  ▼
Collect Load Data Across Departments/Locations
  │
  ▼
Calculate Load Indicators
  │
  ▼
Identify Imbalance
  │
  ├─► Overloaded department ─► Flag
  ├─► Underutilized department ─► Flag
  └─► Balanced ─► No action
           │
           ▼
Generate Balancing Options
           │
  ├─► Asset redeployment
  ├─► Work redistribution
  └─► Pool expansion
           │
           ▼
Apply Constraints
           │
  ├─► Movement cost
  ├─► Technical compatibility
  └─► Geographic proximity
           │
           ▼
Cost-Benefit Analysis
           │
           ▼
Generate Recommendations
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.16-001 | Imbalance threshold | `STDDEV(loadValues) > imbalanceThreshold` | Flag for review | 1 |
| BR-2.7.16-002 | Movement cost limit | `movementCost > budgetLimit` | Exclude redeployment | 2 |
| BR-2.7.16-003 | Minimum utilization target | `utilization < minimumUtilization` | Suggest redeployment | 1 |
| BR-2.7.16-004 | Geographic constraint | `distance > maxDistance` | Exclude remote redeployment | 1 |

**Calculation Formulas:**

```
Load Balance Index = 1 - (MAX(load) - MIN(load)) / AVG(load)

Total Movement Cost = SUM(AssetValue * Distance * MoveCostRate)

Balancing Benefit = (BeforeVariance - AfterVariance) / BeforeVariance * 100
```
