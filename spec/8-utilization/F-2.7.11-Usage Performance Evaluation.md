# F-2.7.11 - Usage Performance Evaluation

## Requirement ID
REQ-FAM-8-2.7.11

## Module
Asset Utilization

## Priority
Medium

## Requirement Description
Evaluate asset usage performance, including efficiency, cost-effectiveness, compliance, safety dimensions, supporting KPI tracking and improvement plans.

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.11-001 | Scoring weight validation | `SUM(weights) = 100%` | Reject if not 100% | 1 |
| BR-2.7.11-002 | KPI data quality | `dataQuality < minimumQuality` | Exclude KPI; note as N/A | 2 |
| BR-2.7.11-003 | Improvement trigger | `score < improvementThreshold` | Generate improvement plan | 1 |

**Performance Dimensions:**
- EFFICIENCY: Utilization rate, output per hour
- COST: Unit cost, ROI, cost per output
- COMPLIANCE: Policy adherence rate, violation count
- SAFETY: Incident rate, training compliance
- QUALITY: Output quality metrics, user satisfaction

**Calculation Formulas:**
```
DimensionScore = SUM(KPI_value * KPI_weight) / SUM(KPI_weight)
OverallScore = w1 * EfficiencyScore + w2 * CostScore + w3 * ComplianceScore + w4 * SafetyScore + w5 * QualityScore
where SUM(w1..w5) = 1.0
Grade: A >= 90, B >= 75, C >= 60, D < 60
```

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-2.7.11-001 | KPIs defined | `kpis.length > 0` | ERR-PRE-2711-001 | "At least one KPI must be defined" |
| PRE-2.7.11-002 | Evaluation period set | `period.isValid = TRUE` | ERR-PRE-2711-002 | "Evaluation period must be valid" |
| PRE-2.7.11-003 | Raw data available | `rawData.isComplete = TRUE` | ERR-PRE-2711-003 | "All required data for KPI calculation must be available" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-2.7.11-001 | Evaluation completed | `evaluation.id IS NOT NULL` |
| POST-2.7.11-002 | Improvement plan created (if needed) | `IF score < threshold THEN plan.id IS NOT NULL` |

**Edge Cases:**

| Edge Case ID | Scenario | Expected Behavior |
|--------------|----------|-------------------|
| EC-2.7.11-001 | Missing KPI data | Exclude from calculation; note limitation |
| EC-2.7.11-002 | New asset (< 3 months) | Exclude from evaluation; note insufficient data |
| EC-2.7.11-003 | Perfect score (100%) | Validate data; flag for review |
