# F-2.7.4 - Real-time Asset Availability Tracking

**Requirement ID:** F-2.7.4  
**Module:** Asset Utilization  
**Priority:** High

---

## Requirement Description

Provide real-time asset availability tracking, including current status, location, user, estimated available time, supporting dynamic updates and notifications.

---

## Business Rules

1. Asset tracking must be enabled for real-time updates
2. Status update mechanism required: MANUAL, AUTOMATIC, RFID, or GPS
3. Valid transitions enforced based on current state
4. AVAILABLE assets can transition to: IN_USE, RESERVED, UNDER_MAINTENANCE, OUT_OF_SERVICE
5. Work order completion auto-returns asset to AVAILABLE
6. Reservation start auto-changes status to IN_USE
7. Concurrent updates use last-write-wins with conflict detection

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| AssetID | Yes | UUID of asset |
| NewStatus | Yes | AVAILABLE, IN_USE, UNDER_MAINTENANCE, RESERVED, OUT_OF_SERVICE, BEING_DISPOSED |
| Timestamp | Yes | Update timestamp (cannot be future) |
| Source | Yes | MANUAL, AUTOMATIC, RFID, GPS, SYSTEM |
| LocationCode | No | Current location |
| Reason | No | Reason for status change |

---

## Output Data

| Data | Description |
|------|-------------|
| AssetStatusUpdateID | System-generated UUID |
| AssetStatusHistoryID | Immutable history record |
| NotificationsSent | Subscribers notified |

---

## Validation Rules

1. Status must be valid enum value
2. Timestamp cannot be more than 5 minutes in future
3. Location code must reference valid location
4. Invalid status transitions rejected with allowed transitions list
5. Stale updates (old timestamp) rejected

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all status updates |
| AssetManager | Update status for managed assets |
| PlantManager | Update within plant scope |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Status updates scoped to organizational hierarchy
- **Isolation**: CompanyCode on all status history records

---

## Compliance Requirements

- GDPR: Location tracking with consent; data minimization
- ISO 55000: Asset tracking and monitoring requirements
- ISO 55001: Asset information management
