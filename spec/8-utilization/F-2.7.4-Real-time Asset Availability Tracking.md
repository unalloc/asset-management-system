## 3. Business Rules Enhancement

### 3.1 Pre-conditions

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-2.7.4-001 | Status updated | `statusHistory.recordCreated = TRUE` |
| POST-2.7.4-002 | Notification sent | `IF subscribed THEN notification.sent = TRUE` |

### 3.3 Business Logic

**Status Types:**
- AVAILABLE
- IN_USE
- UNDER_MAINTENANCE
- RESERVED
- OUT_OF_SERVICE
- BEING_DISPOSED

**Decision Tree:**

```
START
  │
  ▼
Receive Status Update
  │
  ▼
Validate Transition (is transition allowed?)
  │
  ├─► Invalid ─► Reject with allowed transitions
  │
  ├─► Valid ─► Update Asset Status
  │
  ▼
Create Status History Record
  │
  ▼
Notify Subscribers
  │
  ▼
Update Dashboard
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.4-001 | Valid transition check | `targetStatus IN allowedTransitions[currentStatus]` | Proceed or reject | 1 |
| BR-2.7.4-002 | Auto status on maintenance complete | `workOrder.status = 'COMPLETED'` AND `asset.status = 'UNDER_MAINTENANCE'` | Set status = 'AVAILABLE' | 1 |
| BR-2.7.4-003 | Auto status on reservation start | `reservation.startTime <= NOW()` AND `reservation.status = 'CONFIRMED'` | Set status = 'IN_USE' | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior |
|--------------|----------|-------------------|
| EC-2.7.4-001 | Invalid status transition | Reject and return allowed transitions |
| EC-2.7.4-002 | Stale update | Timestamp validation; reject old updates |
| EC-2.7.4-003 | Concurrent updates | Last-write-wins with conflict detection |

### 3.5 State Transitions

**State Diagram:**

```
AVAILABLE ◄───────────────────► IN_USE
     │                              │
     │                              │
     ▼                              ▼
UNDER_MAINTENANCE ◄────► RESERVED
     │
     ▼
OUT_OF_SERVICE ◄──► BEING_DISPOSED
```

**Transition Rules:**

| Current State | Valid Transitions | Guard Conditions |
|---------------|-------------------|------------------|
| AVAILABLE | IN_USE, RESERVED, UNDER_MAINTENANCE, OUT_OF_SERVICE | None |
| IN_USE | AVAILABLE, UNDER_MAINTENANCE | Usage session ended |
| UNDER_MAINTENANCE | AVAILABLE, OUT_OF_SERVICE | Work completed |
| RESERVED | IN_USE, AVAILABLE | Reservation expired or cancelled |
| OUT_OF_SERVICE | BEING_DISPOSED | Disposal initiated |
| BEING_DISPOSED | (terminal state) | None |
