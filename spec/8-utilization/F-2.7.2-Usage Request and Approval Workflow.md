| PRE-2.7.2-004 | Approver available | `approvers.length > 0` AND `approvers.exists(a => a.isAvailable = TRUE)` | ERR-PRE-272-004 | "At least one approver must be available for the first approval step" |

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-2.7.2-001 | Request status updated | `request.status IN ('APPROVED','REJECTED','CONDITIONAL')` | Query request status |

### 3.3 Business Logic

**Decision Tree:**

```
START
  │
  ▼
Validate Pre-conditions
  │
  ├─► FAIL ─► Throw PRE Error ─► END
  │
  ▼
Determine Request Type
  │
  ├─► SINGLE_USE ─► Determine Approval Level (Level 1-2)
  │
  ├─► RECURRING_USE ─► Determine Approval Level (Level 2-3)
  │
  ├─► LONG_TERM_BORROWING ─► Determine Approval Level (Level 3-4)
  │
  └─► EMERGENCY_USE ─► Set Highest Priority, Fast Track
           │
           ▼
Determine Approval Level Based on Asset Value
  │
  ├─► Value < $5,000 ─► Level 1 (Auto-approve or Department Manager)
  ├─► Value $5,000-$50,000 ─► Level 2 (+ Finance)
  └─► Value > $50,000 ─► Level 3 (+ Management)
           │
           ▼
Parallel Approval (if applicable)
  │
  ├─► Technical Approval
  ├─► Safety Approval
  └─► Finance Approval
           │
           ▼
All Approved?
  │
  ├─► YES ─► APPROVED ─► Update Asset Status ─► END
  │
  ├─► NO (some rejected) ─► REJECTED ─► Notify Requester ─► END
  │
  └─► CONDITIONS_PENDING ─► CONDITIONAL ─► Wait for Conditions ─► Re-evaluate
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.2-001 | Auto-approval threshold | `asset.value < 5000` AND `request.duration < 24 hours` AND `request.type = 'SINGLE_USE'` | Set status = 'AUTO_APPROVED' | 1 |
| BR-2.7.2-002 | Emergency escalation | `request.type = 'EMERGENCY_USE'` | Set priority = 'CRITICAL', bypass normal approval | 1 |
| BR-2.7.2-003 | Approval timeout escalation | `approvalStep.ageInHours > timeoutHours` | Send reminder, escalate to backup approver after 24h | 2 |
| BR-2.7.2-004 | Training prerequisite | `request.requiresSafetyTraining = TRUE` AND `requester.safetyTrainingExpiry < request.startDate` | Set status = 'CONDITIONAL', require training completion | 1 |
| BR-2.7.2-005 | Parallel approval sync | `approval.type = 'PARALLEL'` | All parallel approvals must complete before proceeding | 1 |

**Calculation Logic:**

```
Approval Level Score = 
  Base Score (Request Type)
  + Value Score (Asset Value)
  + Duration Score (Usage Duration)
  + Risk Score (Asset Risk Classification)
  + Compliance Score (Compliance Requirements)

Where:
  Request Type: SINGLE_USE=1, RECURRING=2, LONG_TERM=3, EMERGENCY=0
  Asset Value: <$5K=0, $5K-$50K=1, >$50K=2
  Duration: <24h=0, 24h-7d=1, >7d=2
  Risk: Standard=0, High=1, Critical=2
  Compliance: None=0, Safety=1, Environmental=2

Level 1: Score 0-2
Level 2: Score 3-5
Level 3: Score 6-8
Level 4: Score > 8
```

### 3.5 State Transitions

**State Diagram:**

```
States: DRAFT, SUBMITTED, PENDING_PARALLEL_APPROVAL, PENDING_SINGLE_APPROVAL, APPROVED, REJECTED, CONDITIONAL, CANCELLED, EXPIRED

Transitions:
┌─────────┐       ┌───────────┐       ┌─────────────────────┐
│  DRAFT  │──────►│ SUBMITTED │──────►│PENDING_PARALLEL_    │
└─────────┘       └───────────┘       │    APPROVAL         │
                                       └─────────────────────┘
                                              │ All Approved
                                              ▼
┌───────────┐       ┌──────────┐       ┌──────────┐       ┌───────────┐
│  REJECTED │◄──────│CONDITIONAL│◄─────│PENDING_  │──────►│ APPROVED  │
└───────────┘       └──────────┘      │ SINGLE_  │  Yes  └───────────┘
                     │ Conditions Met  │ APPROVAL │
                     └──────────────────┘    │ No
                                              ▼
                                         ┌──────────┐
                                         │ REJECTED │
                                         └──────────┘

Any State ──► CANCELLED (by requester, if permitted)
DRAFT ──► EXPIRED (if not submitted within validity period)
```

**Transition Rules:**

| Current State | Event | Next State | Guard Conditions | Side Effects |
|---------------|-------|------------|------------------|---------------|
| DRAFT | Submit | SUBMITTED | All required fields complete | Notify first approver |
| SUBMITTED | Determine type | PENDING_PARALLEL_APPROVAL or PENDING_SINGLE_APPROVAL | Approval routing determined | Create approval tasks |
| PENDING_PARALLEL_APPROVAL | All parallel approved | PENDING_SINGLE_APPROVAL | All parallel tasks complete | Notify sequential approver |
| PENDING_SINGLE_APPROVAL | Approved | APPROVED | Final approval received | Update asset status |
| PENDING_SINGLE_APPROVAL | Rejected | REJECTED | Rejection received | Notify requester |
| PENDING_SINGLE_APPROVAL | Conditions set | CONDITIONAL | Conditional approval given | Notify requester of conditions |
| CONDITIONAL | All conditions met | APPROVED | Conditions logged and verified | Update asset status |
| CONDITIONAL | Condition not met | REJECTED | Timeout or condition failed | Notify requester |
| SUBMITTED,PENDING_* | Cancelled by requester | CANCELLED | Cancellation allowed for type | Release any locks |
