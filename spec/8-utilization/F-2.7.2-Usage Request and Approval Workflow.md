# F-2.7.2 - Usage Request and Approval Workflow

**Requirement ID:** F-2.7.2  
**Module:** Asset Utilization  
**Priority:** High

---

## Requirement Description

Manage complete workflow for asset usage request submission, multi-level approval, conditional approval, rejection, and feedback.

---

## Business Rules

1. Asset must be AVAILABLE or RESERVED for allocation
2. Requester must have valid account and asset usage permission
3. Approval level based on request type and asset value: <$5K=Level 1, $5K-$50K=Level 2, >$50K=Level 3
4. Segregation of duties enforced: requester cannot approve own request
5. Emergency requests bypass normal hierarchy with fast-track approval
6. Training prerequisites can trigger conditional approval
7. Parallel approvals supported (Technical, Safety, Finance simultaneously)

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| RequestNumber | Yes | Format: REQ-YYYYMMDD-SEQ |
| RequestType | Yes | SINGLE_USE, RECURRING, LONG_TERM, or EMERGENCY |
| RequesterID | Yes | UUID of requesting user |
| StartDateTime | Yes | Requested start time (future) |
| EndDateTime | Yes | Requested end time (after start) |
| Purpose | Yes | Reason for request (1-500 characters) |
| AssetID | Yes | UUID of asset requested |

---

## Output Data

| Data | Description |
|------|-------------|
| UsageRequestID | System-generated UUID |
| ApprovalTasks | Multi-level approval tasks |
| ApprovalHistory | Complete approval audit trail |
| RequestConditions | Conditional approval requirements |

---

## Validation Rules

1. Request number follows format REQ-YYYYMMDD-SEQ
2. Start time must be in future and before end time
3. Purpose must be 1-500 characters
4. Self-approval prevented
5. No time conflicts with existing reservations
6. Training requirements enforced if asset requires certification

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all requests |
| CompanyAssetManager | Approve requests within company |
| DepartmentManager | Approve within department |
| RegularUser | Submit requests |

---

## Multi-Company Data Scope

- **Data Level**: Company/Division/Plant/Department
- **Access**: Users can only submit requests within their company
- **Isolation**: CompanyCode filtering ensures company data isolation

---

## Compliance Requirements

- SOX: Segregation of duties; multi-level approval; complete audit trail
- OSHA: Conditional approval ensures safety training completion
- GDPR: Requestor data minimized; approval history retained for compliance
- GAMP 5: Conditional approval supports GxP training requirements

---
