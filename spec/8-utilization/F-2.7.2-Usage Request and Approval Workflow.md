# F-2.7.2 - Usage Request and Approval Workflow

## Function ID
F-2.7.2

## Function Name
Asset Usage Request Approval

## Function Description
Manage complete workflow for asset usage request submission, multi-level approval, conditional approval, rejection, and feedback.

## Business Rules
1. Request types: single use, recurring use, long-term borrowing, emergency use
2. Approval levels: tiered approval based on asset value, sensitivity, usage duration
3. Parallel approval: multiple departments approve simultaneously (technical, safety, finance)
4. Conditional approval: approval with conditions (training completion, safety check)
5. Auto-approval: low-risk, low-value requests auto-approved
6. Approval timeout: auto-escalation or reminder
7. Approval history: complete record of approval process, comments, attachments
8. Mobile approval: support mobile approval operations

## Preconditions
- Asset can be allocated (not under repair, not reserved, not being disposed)
- Requester has valid account and permissions
- Approval rules defined
- Approver available

## Postconditions
- Request status updated (approved, rejected, requires modification)
- Approval record saved
- Notification sent to relevant personnel
- Asset status updated (if reserved)

## Input Data
1. Requester information
2. Asset ID list
3. Usage start and end time
4. Usage purpose
5. Usage location
6. Special requirements
7. Approval rule reference
8. Attachments (business justification, risk assessment)

## Output Data
1. Usage request number
2. Approval task assignment
3. Approval status tracking
4. Approval completion notification
5. Approval history report
6. Timeout escalation record

## Validation Rules
1. Usage time must be within asset available time
2. Requester must have appropriate permissions
3. Approval rules must cover request scenario
4. Required information complete
5. Attachment format and size meet requirements

## Error Handling
1. Asset not available: Suggest alternative assets or other times
2. Insufficient permission: Reject request and notify administrator
3. Approval rule missing: Use default rule and prompt configuration
4. Time conflict: Detect conflict and suggest adjustment

## Priority
High
