# F-2.7.7 - Equipment Efficiency Analysis

## Function ID
F-2.7.7

## Function Name
Overall Equipment Effectiveness Analysis

## Function Description
Calculate and analyze manufacturing equipment Overall Equipment Effectiveness (OEE), capacity utilization, and equipment effectiveness, supporting production performance improvement.

## Business Rules
1. OEE calculation: Availability × Performance × Quality
2. Data source: integrate with MES, SCADA, PLC systems for real-time data
3. Loss classification: planned downtime, unplanned downtime, speed loss, quality loss
4. Root cause analysis: identify root causes of efficiency loss (equipment, process, personnel, materials)
5. Improvement tracking: track efficiency changes after improvement measures implemented
6. Benchmark analysis: equipment-to-equipment, factory-to-factory, industry benchmark
7. Real-time monitoring: OEE real-time dashboard, anomaly alerts
8. Report generation: shift report, daily report, weekly report, monthly report

## Preconditions
- Equipment data available (MES/SCADA integration)
- OEE calculation parameters configured (planned time, theoretical cycle time)
- Loss classification defined
- Analysis tools available

## Postconditions
- OEE indicators calculated
- Loss analysis report generated
- Improvement suggestions proposed
- Performance data updated

## Input Data
1. Equipment ID
2. Production data (running time, downtime, output quantity, defective quantity)
3. Planned time
4. Theoretical cycle time
5. Loss classification rules
6. Analysis time period
7. Benchmark data
8. Improvement measure records

## Output Data
1. OEE indicators (availability rate, performance rate, quality rate, overall OEE)
2. Loss analysis report
3. Root cause analysis
4. Improvement suggestions
5. Real-time OEE dashboard
6. Benchmark analysis results
7. Trend charts

## Validation Rules
1. Production data must be accurate and timely
2. Planned time must be reasonable (considering shifts, maintenance)
3. Theoretical cycle time must be based on engineering standards
4. Loss classification must be consistent
5. Calculation logic must comply with OEE standards

## Error Handling
1. Data integration failure: Use last known data and mark
2. Parameter missing: Use default parameters and prompt configuration
3. Calculation anomaly: Check data anomaly and recalculate
4. Benchmark data unavailable: Skip benchmark analysis

## Priority
High
