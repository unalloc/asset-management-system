# F-2.7.8 - Idle Asset Identification

## Requirement ID
REQ-FAM-8-2.7.8

## Module
Asset Utilization

## Priority
Medium

## Requirement Description
Identify long-term idle or low-utilization assets, analyze idle reasons, propose optimization solutions (redeployment, sharing, sale, disposal).

## Business Rules

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.8-001 | Idle threshold | `daysUnused >= idleThresholdDays` | Flag as idle | 1 |
| BR-2.7.8-002 | Low utilization threshold | `utilization < utilizationThreshold` | Flag as underutilized | 1 |
| BR-2.7.8-003 | Redeployment priority | `asset.isRedeployable = TRUE` AND `demand EXISTS` | Suggest redeployment | 2 |
| BR-2.7.8-004 | Disposal check | `asset.age >= usefulLife` AND `maintenanceCost > threshold` | Suggest disposal | 1 |

**Idle Definitions:**
- TIME_IDLE: Days unused / Total days in period >= threshold (default 30 days = 5% utilization)
- UTILIZATION_IDLE: Utilization < utilization threshold (default 5%)

**Calculation Formulas:**
```
Idle Cost = (DepreciationRate / 365) * IdleDays + MaintenanceCost + SpaceCost
Optimization Score = (Redeployment Benefit - Cost) / Implementation Cost
```

**Pre-conditions:**

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-2.7.8-001 | Usage data complete | `usageData.period >= idleThresholdDays` | ERR-PRE-278-001 | "Usage data must cover at least idle threshold period" |
| PRE-2.7.8-002 | Idle threshold defined | `idleThresholdDays > 0` | ERR-PRE-278-002 | "Idle threshold must be defined" |
| PRE-2.7.8-003 | Optimization templates available | `templates.length > 0` | ERR-PRE-278-003 | "At least one optimization template required" |

**Post-conditions:**

| Post-condition ID | Description | Success Criteria |
|-------------------|-------------|-------------------|
| POST-2.7.8-001 | Idle report generated | `report.id IS NOT NULL` |
| POST-2.7.8-002 | Optimization workflow initiated | `workflow.status = 'INITIATED'` |

**Edge Cases:**

| Edge Case ID | Scenario | Expected Behavior |
|--------------|----------|-------------------|
| EC-2.7.8-001 | New asset flagged idle | Exclude assets < 90 days old |
| EC-2.7.8-002 | Seasonal demand pattern | Account for seasonal patterns in analysis |
| EC-2.7.8-003 | Assets under warranty | Exclude from disposal consideration |
