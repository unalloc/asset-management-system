### 3.3 Business Logic

**Idle Definitions:**
- TIME_IDLE: Days unused / Total days in period >= threshold (default 30 days = 5% utilization)
- UTILIZATION_IDLE: Utilization < utilization threshold (default 5%)

**Decision Tree:**

```
START
  │
  ▼
Scan Assets for Idle Criteria
  │
  ▼
For Each Idle Asset
  │
  ▼
Analyze Idle Reason
  │
  ├─► Demand Change
  ├─► Technical Obsolescence
  ├─► Configuration Issue
  ├─► Improper Location
  └─► Other
           │
           ▼
Calculate Idle Cost
  │
  ▼
Generate Optimization Solutions
  │
  ├─► Redeploy
  ├─► Share Pool
  ├─► Sell
  ├─► Dispose
  └─► Rent
           │
           ▼
Cost-Benefit Analysis
           │
           ▼
Workflow Initiation (if approved)
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.8-001 | Idle threshold | `daysUnused >= idleThresholdDays` | Flag as idle | 1 |
| BR-2.7.8-002 | Low utilization threshold | `utilization < utilizationThreshold` | Flag as underutilized | 1 |
| BR-2.7.8-003 | Redeployment priority | `asset.isRedeployable = TRUE` AND `demand EXISTS` | Suggest redeployment | 2 |
| BR-2.7.8-004 | Disposal check | `asset.age >= usefulLife` AND `maintenanceCost > threshold` | Suggest disposal | 1 |

**Calculation Formulas:**

```
Idle Cost = (DepreciationRate / 365) * IdleDays + MaintenanceCost + SpaceCost

Optimization Score = (Redeployment Benefit - Cost) / Implementation Cost
```
