# F-2.7.6 Asset Utilization Calculation

## Function ID
F-2.7.6

## Function Name
Utilization Calculation and Analysis

## Function Description
Calculate and analyze asset utilization, including time utilization, capacity utilization, value utilization, supporting multiple calculation methods and benchmark comparison.

## Business Rules
1. Utilization types: time utilization (actual use time/available time), capacity utilization (actual output/design capacity), value utilization (generated value/asset value)
2. Calculation methods: based on sensor data, manual report, system logs
3. Benchmark setting: industry benchmark, historical benchmark, target benchmark
4. Analysis dimensions: by asset, department, time period, asset type
5. Trend analysis: utilization change trends, seasonal patterns
6. Root cause analysis: reasons for low utilization (excessive maintenance, insufficient demand, configuration issues)
7. Optimization suggestions: propose optimization suggestions based on analysis results
8. Report generation: regular utilization reports, anomaly alerts

## Preconditions
- Usage data available (sensor, log, manual input)
- Available time defined (considering shifts, maintenance windows)
- Benchmark data available
- Analysis model configured

## Postconditions
- Utilization indicators calculated
- Analysis report generated
- Optimization suggestions proposed
- Performance indicators updated

## Input Data
1. Asset ID
2. Usage data (start time, end time, output, value)
3. Available time definition
4. Benchmark data
5. Analysis parameters
6. Report period
7. Alert thresholds
8. Optimization objectives

## Output Data
1. Utilization indicators
2. Analysis report
3. Trend charts
4. Root cause analysis
5. Optimization suggestions
6. Anomaly alerts
7. Performance benchmark results

## Validation Rules
1. Usage data must be accurate and reliable
2. Available time definition must be reasonable
3. Benchmark data must be comparable
4. Calculation method must be consistent
5. Analysis parameters must be within reasonable range

## Error Handling
1. Data missing: Use estimated value and mark uncertainty
2. Definition conflict: Manual review to determine correct definition
3. Calculation error: Recalculate and verify logic
4. Benchmark not comparable: Adjust benchmark or use alternative benchmark

## Priority
High
