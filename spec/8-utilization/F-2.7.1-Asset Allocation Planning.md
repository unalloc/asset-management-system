**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-2.7.1-001 | Plan Period Alignment | `plan.startDate` | `plan.startDate = budgetPeriod.startDate` | 1 |
| BR-2.7.1-002 | Utilization Maximization | `optimization.objective = 'MAX_UTILIZATION'` | `MAXIMIZE SUM(allocatedHours) / SUM(availableHours)` | 1 |
| BR-2.7.1-003 | Idle Minimization | `optimization.objective = 'MIN_IDLE'` | `MINIMIZE SUM(idleDays) * idleCostRate` | 1 |
| BR-2.7.1-004 | Workload Balancing | `optimization.objective = 'BALANCE'` | `MINIMIZE STDDEV(departmentUtilization)` | 2 |
| BR-2.7.1-005 | Budget Constraint | `allocationCost > budgetAmount` | `WARN_USER` AND `requireOverrideReason` | 1 |

**Calculation Logic:**

```
Asset Allocation Score = w1 * UtilizationScore + w2 * IdlePenalty + w3 * BalanceScore + w4 * CostScore

Where:
  UtilizationScore = ActualAllocation / TargetAllocation
  IdlePenalty = IdleDays * DailyIdleCost
  BalanceScore = 1 - (STDDEV(departmentLoads) / MEAN(departmentLoads))
  CostScore = 1 - (TotalCost / BudgetLimit)

Constraint: All allocations must satisfy:
  - Department demand <= Asset capacity
  - Total cost <= Budget limit (with warning threshold at 90%)
  - Technical compatibility requirements met
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-2.7.1-001 | Budget period crosses fiscal year | Split allocation into two sub-plans aligned with fiscal years | Create linked sub-plans with carry-forward logic |
| EC-2.7.1-002 | Conflicting department demands | Prioritize based on strategic importance and historical allocation | Apply priority weights; manual review if conflict unresolved |
| EC-2.7.1-003 | Asset pending disposal during plan period | Exclude from available pool or include with reduced life | Flag with disposal date; calculate remaining useful allocation |
| EC-2.7.1-004 | Zero demand from a department | No allocation to that department; rebalance across others | Validate demand data; redistribute if capacity available |
| EC-2.7.1-005 | Emergency procurement needed | Generate procurement recommendation with justification | Create procurement task linked to allocation plan |
