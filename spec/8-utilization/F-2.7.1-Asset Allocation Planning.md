# F-2.7.1 - Asset Allocation Planning

**Requirement ID:** F-2.7.1  
**Module:** Asset Utilization  
**Priority:** High

---

## Requirement Description

Support annual and quarterly asset allocation planning, optimizing asset configuration based on business demand forecasts, budget constraints, and asset availability.

---

## Business Rules

1. Plan period must align with budget period (annual or quarterly)
2. Multi-level approval required: Department → Finance → Management
3. Optimization objectives: MAX_UTILIZATION, MIN_IDLE, BALANCE workload
4. Budget constraint: cost > 90% of budget triggers warning
5. Total cost must not exceed budget limit (requires override with reason)
6. Asset availability must be validated before allocation
7. Utilization score calculated from actual vs target allocation

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| PlanCode | Yes | Format: APL-YYYYMM-SEQ |
| PlanName | Yes | Plan name (1-200 characters) |
| PlanType | Yes | ANNUAL or QUARTERLY |
| StartDate | Yes | Plan start date |
| EndDate | Yes | Plan end date |
| TotalBudget | Yes | Budget amount (positive decimal) |
| CompanyCode | Yes | Company identifier |

---

## Output Data

| Data | Description |
|------|-------------|
| AllocationPlanID | System-generated UUID |
| AllocationMatrix | Asset-department allocation details |
| ApprovalTasks | Multi-level approval tasks |
| MonitoringIndicators | KPIs for tracking |

---

## Validation Rules

1. Plan code must follow format APL-YYYYMM-SEQ
2. Start date must be today or future, before end date
3. Total budget must be between 0 and 999,999,999.99
4. Company code must reference valid company
5. No duplicate plan code per period
6. Annual plan must start on fiscal year boundary
7. Quarterly plan must start on quarter boundary

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all allocation plans |
| CompanyPlanner | Create plans within company scope |
| DepartmentManager | View and approve within department |
| FinancialAnalyst | View allocation data |

---

## Multi-Company Data Scope

- **Data Level**: Company/Division/Plant/Department
- **Access**: Users can only create/view plans within their company
- **Isolation**: CompanyCode filtering ensures data isolation

---

## Compliance Requirements

- SOX: Multi-level approval for internal controls; segregation of duties enforced
- GDPR: Allocation plans anonymized for reporting; no personal data
- ISO 55000: Asset management planning aligned with organizational objectives
