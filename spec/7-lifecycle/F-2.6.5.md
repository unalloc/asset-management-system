# F-2.6.5 Borrowing/Return System

## Function ID
F-2.6.5

## Function Name
Asset Borrowing Return

## Function Description
Provide complete process management for asset borrowing, returning, and reservation, supporting temporary asset allocation and tracking.

## Business Rules
1. Borrowing requires application and approval, with different approval levels based on asset value and sensitivity
2. Support reserved borrowing (advance booking) and instant borrowing
3. Borrowing period can be set, with automatic overdue reminders
4. Return requires asset condition inspection, recording damage or loss
5. Support batch borrowing and returning
6. Complete borrowing history, traceable
7. Integrate with employee access/identity system, automatically associate borrower
8. Support mobile borrowing application and approval

## Preconditions
- Asset status is "Available" or "In Stock"
- Borrower has valid employee account
- Borrowing approval process is defined
- Asset is not under repair or reserved

## Postconditions
- Asset status updated to "Borrowed"
- Borrowing record created
- Approval record saved
- Borrowing notification sent

## Input Data
1. Asset ID or asset list
2. Borrower ID
3. Borrowing start date and time
4. Expected return date and time
5. Borrowing purpose
6. Approver ID (if required)
7. Borrowing location
8. Special requirements

## Output Data
1. Borrowing application number
2. Approval status
3. Borrowing confirmation
4. Borrowing reminder (before start)
5. Return reminder (before due)
6. Borrowing history record

## Validation Rules
1. Borrowing start time cannot be earlier than current time
2. Borrowing end time must be after start time
3. Asset must be available at specified location
4. Borrower must have valid borrowing permission
5. Borrowing period cannot exceed maximum allowed period

## Error Handling
1. Asset not available: Show available substitute assets or suggest other times
2. Borrower without permission: Reject application and notify administrator
3. Approver invalid: Prompt to select valid approver
4. Borrowing conflict: Show conflicting assets and times

## Priority
High
