## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset ABC classification defined | Check ClassificationRules exists | CC-101 | "ABC classification not defined" |
| PRE-002 | Risk level assessed | Check RiskLevels exist | CC-102 | "Risk levels not defined" |
| PRE-003 | Counting history exists | Check CountingHistory exists | CC-103 | "Counting history not available" |

### 3.3 Business Logic

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | ABC Frequency | Classification = 'A' | Count Monthly | 1 |
| BR-002 | ABC Frequency | Classification = 'B' | Count Quarterly | 1 |
| BR-003 | ABC Frequency | Classification = 'C' | Count Semi-annually | 1 |
| BR-004 | Risk Adjustment | RiskLevel = 'HIGH' | Double frequency | 2 |
| BR-005 | Variance Adjustment | PastVariance > Threshold | Increase frequency | 2 |

**Calculation Logic:**
```
CountingFrequency = 
  BaseFrequencyByClass(Asset.Class)
  * RiskMultiplier(Asset.RiskLevel)
  * VarianceMultiplier(Asset.VarianceHistory)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Asset reclassified | Adjust frequency | Recalculate based on new class |
| EC-002 | New asset without history | Use default frequency | Flag for review |
| EC-003 | Count missed | Catch-up count | Schedule immediately |
