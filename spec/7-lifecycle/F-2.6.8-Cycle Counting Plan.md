# F-2.6.8 - Cycle Counting Plan

**Requirement ID:** F-2.6.8  
**Module:** Asset Lifecycle Management  
**Priority:** Medium

---

## Requirement Description

Support cycle counting plan based on ABC classification, risk level, or other rules to achieve continuous inventory accuracy and variance prevention.

---

## Business Rules

1. ABC classification determines base counting frequency: A=Monthly, B=Quarterly, C=Semi-annually
2. HIGH risk assets have frequency doubled
3. Assets with past variance > threshold have increased frequency
4. Asset reclassification triggers frequency recalculation
5. New assets without history use default frequency
6. Missed counts trigger immediate catch-up scheduling

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| Class | Yes | A, B, or C classification |
| ValueThreshold | Yes | Value boundary for classification (positive decimal) |
| CountFrequency | Yes | MONTHLY, QUARTERLY, SEMI_ANNUAL, or ANNUAL |
| AssetID | Yes | UUID of asset for schedule |
| RiskLevel | No | LOW, MEDIUM, or HIGH |

---

## Output Data

| Data | Description |
|------|-------------|
| ScheduleID | System-generated UUID |
| NextCountDate | Calculated next counting date |
| Frequency | Final calculated frequency |
| FrequencyAdjustmentID | UUID if frequency changed |

---

## Validation Rules

1. Value threshold must be positive
2. Frequency must be valid format (MONTHLY, QUARTERLY, SEMI_ANNUAL, ANNUAL)
3. ABC classification must be defined before scheduling
4. Risk levels must be defined before scheduling

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to classification rules and schedules |
| AssetManager | Manage ABC classification rules |
| PlantManager | View cycle count schedules |

---

## Multi-Company Data Scope

- **Data Level**: Company/Division/Plant
- **Access**: Classification rules managed at company level; schedules auto-generated by system
- **Isolation**: CompanyCode filtering on classification rules

---

## Compliance Requirements

- SOX: Continuous counting supports annual count requirement
- US GAAP: ASC 360 - asset existence verification support
- ISO 55000: Plan-Do-Check-Act cycle with continuous verification
