## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset suitable for refurbishment | Check Asset.Status = 'AVAILABLE' OR 'RETIRED' | REF-101 | "Asset not eligible for refurbishment" |
| PRE-002 | Refurbishment budget approved | Check Budget.Approved = TRUE | REF-102 | "Refurbishment budget not approved" |
| PRE-003 | Redeployment requirement clear | Check RedeploymentRequest exists | REF-103 | "Redeployment requirement not defined" |

### 3.3 Business Logic

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Cost Cap | RefurbishmentCost > 0.6 * NewAssetCost | Warn, may not be worthwhile | 1 |
| BR-002 | Life Extension | PostRefurbishmentLife > CurrentRemainingLife * 1.2 | Confirm benefit | 1 |
| BR-003 | Warranty Reset | Refurbishment complete | Set new warranty period | 2 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Hidden damage found | Assess additional cost, re-approve | Pause, reassess |
| EC-002 | Refurbishment fails QA | Rework or disposal | Conversion workflow |
