## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Supports asset allocation and utilization |
| **ISO 55001 Physical Asset Management** | Yes | Asset custody and tracking requirements |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Asset utilization best practices |

**ISO 55000 Alignment Explanation:**
Asset borrowing and return management supports ISO 55000's emphasis on asset utilization and value realization. Proper tracking of temporary asset allocations ensures assets are not idle and maintains chain of custody for asset accountability.

**Alignment Gaps:**
- Missing linkage to asset utilization reporting
- No explicit requirement for asset condition at return comparison to assignment

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset status is "Available" or "In Stock" | Check Asset.Status IN ('AVAILABLE', 'IN_STOCK') | BOR-101 | "Asset not available for borrowing" |
| PRE-002 | Borrower has valid employee account | Check Employee.EmployeeID exists AND Employee.Status = 'ACTIVE' | BOR-102 | "Borrower not valid employee" |
| PRE-003 | Borrowing approval process defined | Check ApprovalWorkflow exists for 'BORROWING' | BOR-103 | "Borrowing workflow not configured" |
| PRE-004 | Asset not under repair or reserved | Check Asset.Status NOT IN ('UNDER_REPAIR', 'RESERVED') | BOR-104 | "Asset under repair or reserved" |

**Pre-condition Validation Matrix:**
```
IF (Asset.Status IN ('AVAILABLE', 'IN_STOCK')
   AND Employee IS VALID AND ACTIVE
   AND ApprovalWorkflow EXISTS FOR 'BORROWING'
   AND Asset.Status NOT IN ('UNDER_REPAIR', 'RESERVED'))
THEN proceed to borrowing
ELSE throw PRE-xxx Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Asset status updated to "Borrowed" | Asset.Status = 'BORROWED' | Asset record query |
| POST-002 | Borrowing record created | BorrowingRecord created with timestamps | Database query |
| POST-003 | Approval record saved | ApprovalRecord with approver, timestamp | Approval audit |
| POST-004 | Borrowing notification sent | Notification sent to borrower, owner | Notification log |

**Post-condition Verification:**
- Asset status change logged in audit trail
- Borrowing record linked to asset and borrower
- Notification sent and logged

### 3.3 Business Logic

**Decision Tree:**
```
START
  │
  ▼
Borrowing Request
  │
  ▼
Check Pre-conditions
  │
  ├── Fail ──► Return Error ──► END
  │
  └── Pass ──► ▼
  │
  ▼
Determine Approval Level
  │
  ├── Low Value (< $1K) ──► Auto-Approve ──► ▼
  │
  ├── Medium Value ($1K-$10K) ──► Department Manager ──► ▼
  │
  └── High Value (> $10K) ──► VP Approval ──► ▼
  │
  ▼
Reserve Asset
  │
  ▼
Issue Asset to Borrower
  │
  ▼
Record Condition (if applicable)
  │
  ▼
Asset.Status = 'BORROWED'
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Auto-Approval | Asset.Value < 1000 AND Borrower.History = 'GOOD' | Auto-approve | 1 |
| BR-002 | Tiered Approval | Asset.Value >= threshold | Route to appropriate approver | 1 |
| BR-003 | Overdue Check | DueDate < TODAY AND Status = 'BORROWED' | Create overdue alert, block new requests | 2 |
| BR-004 | Blacklist Check | Borrower.InBlacklist = TRUE | Reject borrowing request | 1 |
| BR-005 | Reservation Conflict | Asset already reserved for dates | Block, show conflict | 1 |

**Calculation Logic:**
```
ApprovalLevel = 
  IF Asset.Value < 1000 THEN 'AUTO'
  ELSE IF Asset.Value < 10000 THEN 'DEPARTMENT'
  ELSE 'VP'
  
AvailableForBorrowing = 
  Asset.Status IN ('AVAILABLE', 'IN_STOCK')
  AND NOT EXISTS (ActiveReservation WHERE AssetID = Asset.ID AND DateRange OVERLAPS)
  AND NOT Borrower.InBlacklist
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Asset overdue | Block new borrowing, send alerts | Create escalation workflow |
| EC-002 | Borrower on blacklist | Reject request, notify admin | Return error with reason |
| EC-003 | Borrower leaves company with asset | Create alert, HR integration | Trigger asset return process |
| EC-004 | Asset damaged at return | Create damage assessment, assess fees | Block return until resolved |
| EC-005 | Reservation conflict | Show alternative dates/assets | Suggest alternatives |
| EC-006 | Emergency borrowing | Simplified approval, complete later | Auto-approve with conditions |
