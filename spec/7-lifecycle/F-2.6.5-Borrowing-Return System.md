# F-2.6.5 - Borrowing-Return System

**Requirement ID:** F-2.6.5  
**Module:** Asset Lifecycle Management  
**Priority:** Medium

---

## Requirement Description

Provide complete process management for asset borrowing, returning, and reservation, supporting temporary asset allocation and tracking.

---

## Business Rules

1. Asset must be in AVAILABLE or IN_STOCK status for borrowing
2. Borrower must be valid active employee
3. Tiered approval based on asset value: <$1K auto-approve, $1K-$10K department manager, >$10K VP approval
4. Overdue borrowers are blocked from new borrowing requests
5. Blacklisted borrowers cannot borrow assets
6. Reservation conflict check prevents double-booking
7. Asset status changes to BORROWED when issued, back to AVAILABLE on return
8. Condition documentation at borrowing and return

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| AssetID | Yes | UUID of asset to borrow |
| BorrowerID | Yes | UUID of borrowing employee |
| RequestedStartDate | Yes | Borrowing start date (must be future) |
| RequestedEndDate | Yes | Borrowing end date (must be after start) |
| Purpose | Yes | Reason for borrowing (max 500 chars) |

---

## Output Data

| Data | Description |
|------|-------------|
| BorrowingRecordID | System-generated UUID |
| ApprovalLevel | AUTO, DEPARTMENT, or VP |
| AssetStatus | Updated to BORROWED |
| DueDate | Calculated return due date |

---

## Validation Rules

1. Requested start date must be in the future
2. End date must be after start date
3. Purpose is required
4. Asset must be AVAILABLE or IN_STOCK
5. Borrower must be active employee
6. Borrower must not be blacklisted
7. No overlapping reservations for the same dates

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all borrowing operations |
| AssetManager | Approve and manage borrowings |
| DepartmentManager | Approve borrowings within department |
| Employee | Request to borrow assets |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Users can only borrow assets within their organizational hierarchy
- **Isolation**: CompanyCode filtering on borrowing records

---

## Compliance Requirements

- SOX: Asset custody changes tracked with full audit trail
- OSHA: Equipment operator assignment tracked for certification compliance
- GDPR: Borrower personal data minimized (employee ID and department only)
