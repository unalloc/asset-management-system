# F-2.6.6 Temporary Asset Borrowing Management

## Function ID
F-2.6.6

## Function Name
Temporary Asset Borrowing Management

## Function Description
Manage complete process of temporary asset borrowing, extension, early return, damage compensation, supporting borrowing cost calculation and performance analysis.

## Business Rules
1. Borrowing can be extended, requires re-approval (if exceeding original period by 50%)
2. Early return can be automatically processed, releasing asset
3. Damage or loss during borrowing requires compensation process
4. Borrowing cost can be calculated by time, asset value, department budget
5. Borrowing performance analysis: borrowing frequency, average borrowing duration, on-time return rate
6. Support cross-department borrowing cost settlement
7. Borrowing blacklist management: multiple violations limit borrowing permission
8. Emergency borrowing process: simplified approval, supplementary procedures later

## Preconditions
- Borrowing application approved
- Asset assigned to borrower
- Borrowing cost calculation rules defined

## Postconditions
- Borrowing status updated
- Borrowing cost record
- Borrowing performance data updated
- Asset availability updated

## Input Data
1. Borrowing application number
2. Actual borrowing start time
3. Actual return time
4. Asset condition inspection result
5. Damage description (if applicable)
6. Compensation amount (if applicable)
7. Extension application (if needed)
8. Borrowing cost allocation department

## Output Data
1. Borrowing completion record
2. Damage compensation order
3. Borrowing cost calculation report
4. Borrowing performance indicators
5. Borrowing blacklist update
6. Asset status update notification

## Validation Rules
1. Actual borrowing time must be within borrowing period (unless extension approved)
2. Damage description must include photos or evidence
3. Compensation amount must be based on predefined compensation standards
4. Borrowing cost must be allocated to valid cost center
5. Asset must go through condition inspection at return

## Error Handling
1. Borrowing record does not exist: Return error "Borrowing record invalid"
2. Insufficient damage evidence: Require additional evidence or transfer to investigation process
3. Compensation standard not defined: Prompt administrator to configure standard
4. Cost center invalid: Prompt to select valid cost center

## Priority
Medium
