# F-2.6.2 - Asset Receiving and Inspection

**Requirement ID:** F-2.6.2  
**Module:** Asset Lifecycle Management  
**Priority:** High

---

## Requirement Description

Provide standard process for asset receiving, quality inspection, and acceptance confirmation to ensure received assets meet procurement specifications and quality requirements.

---

## Business Rules

1. PO must exist and not be fully received before receiving can proceed
2. Three-way match support: PO, receipt, invoice verification
3. Inspection required before asset creation (QUALIFIED, PENDING_REPAIR, or REJECTED)
4. Inspector must have valid certification for the asset type
5. Safety equipment inspection required at receipt before commissioning
6. Partial receipts update PO received quantity and create backorder
7. Over-receipt (receiving more than ordered) requires PO amendment or manual override

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| PONumber | Yes | Purchase order number to receive against |
| ReceivedQty | Yes | Quantity received (positive decimal) |
| LocationCode | Yes | Receiving location identifier |
| InspectorID | Yes | UUID of inspector |
| InspectionDate | Yes | Date of inspection (cannot be future) |
| Result | Yes | QUALIFIED, PENDING_REPAIR, REJECTED |
| Findings | No | Text description of inspection findings |

---

## Output Data

| Data | Description |
|------|-------------|
| ReceivingRecordID | System-generated UUID |
| AssetReceivingRecord | Receipt details linked to PO |
| InspectionReport | Inspection results with findings and recommendations |
| SupplierPerformance | Updated quality score for supplier |
| ReturnOrderID | UUID if items rejected (for returns) |

---

## Validation Rules

1. PO number must exist in system
2. PO must not be fully received
3. Received quantity must be positive
4. Inspector must have INSPECTOR role for the asset type
5. Inspection date cannot be in the future
6. Rejection requires a reason to be provided
7. Quantity variance > 5% triggers warning

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all receiving operations |
| WarehouseManager | Create receiving sessions, submit inspections |
| QualityInspector | Submit inspection results |
| ReceivingOperator | Read-only access to receiving operations |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant
- **Access**: Receiving can only be performed at user's assigned plant
- **Isolation**: CompanyCode filtering on all receiving records

---

## Compliance Requirements

- SOX: Receiving records support asset recognition and three-way match
- OSHA: Safety equipment inspection documentation required at receipt
- ISO 55000: Asset handover criteria with quality inspection
- GDPR: Inspector personal data minimized (name only)

---

High
