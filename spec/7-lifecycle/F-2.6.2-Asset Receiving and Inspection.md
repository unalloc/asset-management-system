# F-2.6.2 - Asset Receiving and Inspection

## Function ID
F-2.6.2

## Function Name
Asset Receiving Inspection

## Function Description
Provide standard process for asset receiving, quality inspection, and acceptance confirmation to ensure received assets meet procurement specifications and quality requirements.

## Business Rules
1. Support batch receiving and single receiving modes
2. Basic quality check required at receiving (appearance, quantity, model)
3. Support rejection, partial receiving, conditional receiving
4. Inspection result determines asset status (qualified, pending repair, returned)
5. Inspection personnel must have corresponding qualification certification
6. Inspection records must be saved and associated with asset files
7. Support attachment upload (inspection reports, photos, certificates)

## Preconditions
- Purchase order exists and not fully received
- User has "Asset Receiving" permission
- Inspection standards are configured

## Postconditions
- Asset receiving status updated
- Inspection record saved
- Supplier performance data updated

## Input Data
1. Purchase order number
2. Actual received quantity
3. Inspection result (qualified/unqualified)
4. Inspector
5. Inspection date
6. Inspection remarks
7. Attachment files
8. Rejection reason (if applicable)

## Output Data
1. Asset receiving record
2. Inspection report
3. Supplier performance score update
4. Purchase order receiving status update
5. Non-conforming product handling suggestions

## Validation Rules
1. Received quantity cannot exceed purchase order unreceived quantity
2. Inspection date cannot be earlier than purchase order date
3. Inspector must have valid qualification
4. Unqualified asset must include reason
5. Required fields complete

## Error Handling
1. Purchase order does not exist: Return error "Purchase order invalid"
2. Quantity exceeds limit: Show maximum receivable quantity
3. Qualification invalid: Block inspection submission and notify administrator
4. Attachment too large: Suggest compression or splitting

## Priority
High
