## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Supports asset handover criteria between lifecycle stages |
| **ISO 55001 Physical Asset Management** | Yes | Clause 6.4.2 - Asset creation/acquisition |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Handover documentation requirements |

**ISO 55000 Alignment Explanation:**
This requirement supports ISO 55000 asset management by ensuring proper asset handover from procurement to operations. It includes quality inspection, documentation of asset condition, and formal acceptance confirmation - all key elements of ISO 55001's asset creation and handover requirements.

**Alignment Gaps:**
- Missing requirement to link inspection results to asset master for future maintenance planning
- No explicit linkage to maintenance strategy determination based on inspection results

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Purchase order exists and not fully received | Check PO exists AND PO.ReceivedQty < PO.OrderedQty | RCV-101 | "Purchase order does not exist or fully received" |
| PRE-002 | User has "Asset Receiving" permission | Check UserPermissions contains 'ASSET_RECEIVING' | RCV-102 | "User lacks asset receiving permission" |
| PRE-003 | Inspection standards are configured | Check InspectionStandards exists for asset type | RCV-103 | "Inspection standards not configured for asset type" |
| PRE-004 | Company code is valid | Check Company.CompanyCode is not null | RCV-104 | "Company code required for receiving" |
| PRE-005 | Receiving location exists | Check Location.LocationCode is valid | RCV-105 | "Invalid receiving location" |

**Pre-condition Validation Matrix:**
```
IF (PO EXISTS AND PO.ReceivedQty < PO.OrderedQty)
   AND (User has 'ASSET_RECEIVING')
   AND (InspectionStandards EXISTS for PO.ItemType)
   AND (Company.CompanyCode IS NOT NULL)
   AND (Location.LocationCode IS VALID)
THEN proceed to receiving workflow
ELSE throw PRE-xxx Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Asset receiving status updated | PO.ReceivedQty updated, status reflects partial/full | PO record verification |
| POST-002 | Inspection record saved | Inspection record created with timestamp | Database query |
| POST-003 | Supplier performance data updated | Supplier score recalculated | Supplier score report |

**Post-condition Verification:**
- Receipt quantity added to PO.ReceivedQty
- Inspection record linked to PO and Asset (when created)
- Supplier score updated based on inspection results

### 3.3 Business Logic

**Decision Tree:**
```
START
  │
  ▼
Receive Asset at Dock
  │
  ▼
Scan/Enter PO Number
  │
  ▼
Quantity Verification
  │
  ├── Qty Matches PO ──► Basic Quality Check ──► Detailed Inspection ──► Result Determination
  │                                                  │
  │                                                  ├── QUALIFIED ──► Accept Asset ──► Create Asset Record
  │                                                  │
  │                                                  ├── PENDING_REPAIR ──► Create Repair WO ──► Update PO Status
  │                                                  │
  │                                                  └── REJECTED ──► Create Return Order ──► Notify Supplier
  │
  └── Qty Does Not Match ──► Log Variance ──► Partial Receipt Option ──► Customer Confirmation
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Full Receipt Processing | ReceivedQty = PO.OrderedQty | Mark PO as 'FULLY_RECEIVED', create asset | 1 |
| BR-002 | Partial Receipt Processing | ReceivedQty < PO.OrderedQty | Update PO.ReceivedQty, create backorder | 1 |
| BR-003 | Over-Receipt Handling | ReceivedQty > PO.OrderedQty | Block, require PO amendment or manual override | 2 |
| BR-004 | Inspection Result Classification | Inspection completed | Set Asset.Status based on result | 1 |
| BR-005 | Supplier Quality Score | Inspection completed | Update Supplier.QualityScore | 2 |
| BR-006 | Auto-Assignment Inspector | PO item type known | Assign qualified inspector based on asset category | 3 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | PO fully received but inspection failed | Block asset creation, create repair work order | Status = 'INSPECTION_FAILED' |
| EC-002 | Inspector qualification expired | Block inspection, require reassignment | Notify administrator, auto-suggest alternative |
| EC-003 | Receiving at unanticipated location | Allow with override reason, update PO location | Log location variance |
| EC-004 | Batch receiving items with different inspection requirements | Process individually or group by inspection class | Create multiple inspection records |
| EC-005 | Network interruption during mobile receiving | Enable offline mode, auto-sync when restored | Queue transactions locally |

**Edge Case Decision Matrix:**

| PO Status | Receipt Qty | Inspection Result | Action |
|-----------|-------------|-------------------|--------|
| PARTIAL | < Ordered | QUALIFIED | Update PO, create asset |
| PARTIAL | < Ordered | REJECTED | Update PO, create return |
| FULL | = Ordered | QUALIFIED | Close PO, create asset |
| FULL | > Ordered | ANY | Block, require amendment |
