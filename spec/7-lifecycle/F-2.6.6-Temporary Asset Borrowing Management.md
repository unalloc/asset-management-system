# F-2.6.6 - Temporary Asset Borrowing Management

**Requirement ID:** F-2.6.6  
**Module:** Asset Lifecycle Management  
**Priority:** Medium

---

## Requirement Description

Manage complete process of temporary asset borrowing, extension, early return, damage compensation, supporting borrowing cost calculation and performance analysis.

---

## Business Rules

1. Extension duration > 50% of original duration requires re-approval
2. Condition at return compared to condition at issue for damage detection
3. Damage triggers damage assessment workflow and compensation order
4. Early return calculates prorated fee
5. Borrowers with damaged returns or >30 days overdue added to blacklist
6. Borrowing costs calculated by department with chargeback

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| BorrowingRecordID | Yes | UUID of active borrowing |
| ExtensionDays | Yes | Days to extend (1-180) |
| Reason | Yes | Reason for extension |
| ReturnCondition | Yes | GOOD, DAMAGED, or WORSE |
| DamageDescription | No | Description of damage if applicable |

---

## Output Data

| Data | Description |
|------|-------------|
| ExtensionID | System-generated UUID |
| CompensationOrderID | UUID if damage compensation required |
| UpdatedDueDate | New due date after extension |
| BorrowingCostRecord | Cost calculations if applicable |

---

## Validation Rules

1. Extension days must be 1-180
2. Compensation amount must be non-negative
3. Borrowing must be approved before extension
4. Asset must be issued to borrower before return
5. Damage description required if condition is DAMAGED or WORSE

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all extensions and damage processing |
| DepartmentManager | Approve extensions within department |
| AssetManager | Process returns and damage assessments |

---

## Multi-Company Data Scope

- **Data Level**: Company/Division/Plant
- **Access**: Damage compensation processed at division level; blacklist at company level
- **Isolation**: CompanyCode filtering on compensation records

---

## Compliance Requirements

- SOX: Damage compensation approval with full audit trail
- OSHA: Return inspection documentation for equipment condition
- GDPR: Damage photo retention minimized (only necessary evidence)
