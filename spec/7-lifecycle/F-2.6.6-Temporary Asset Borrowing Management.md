## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Borrowing application approved | Check BorrowingRecord.Status = 'APPROVED' | TMP-101 | "Borrowing application not approved" |
| PRE-002 | Asset assigned to borrower | Check Asset.Status = 'BORROWED' | TMP-102 | "Asset not assigned to borrower" |
| PRE-003 | Cost calculation rules defined | Check CostRules exists for company | TMP-103 | "Cost calculation rules not defined" |

### 3.3 Business Logic

**Decision Tree:**
```
START
  │
  ▼
Process Return
  │
  ▼
Asset Condition Inspection
  │
  ├── Condition = Good ──► ▼
  │                 │
  │                 ▼
  │           Calculate Costs
  │                 │
  │                 ▼
  │           Close Borrowing Record
  │                 │
  ├── Condition = Damaged ──► ▼
  │                      │
  │                      ▼
  │              Damage Assessment
  │                      │
  │                      ▼
  │              Calculate Compensation
  │                      │
  │                      ▼
  │              Create Compensation Order
  │
  └── Early Return ──► ▼
                      │
                      ▼
                Calculate Early Return Fee
                      │
                      ▼
                Update Cost Record
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Extension Approval | ExtensionDuration > OriginalDuration * 0.5 | Require re-approval | 1 |
| BR-002 | Damage Assessment | Condition at return < Condition at issue | Trigger damage workflow | 1 |
| BR-003 | Cost Calculation | Borrowing completed | Calculate borrowing cost | 2 |
| BR-004 | Early Return Fee | ReturnDate < DueDate | Calculate prorated fee | 2 |
| BR-005 | Blacklist Update | ReturnDamaged = TRUE OR OverdueDays > 30 | Add to blacklist | 1 |

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Borrower disputes damage | Third-party assessment | Mediation process |
| EC-002 | Cost center closed | Allocate to default or parent | Override process |
| EC-003 | Missing condition at issue | Use default condition | Flag for review |
| EC-004 | Emergency extension | Auto-approve with notification | Simplified workflow |
