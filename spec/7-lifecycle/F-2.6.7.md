# F-2.6.7 Physical Inventory Counting

## Function ID
F-2.6.7

## Function Name
Physical Inventory Counting

## Function Description
Support regular or ad-hoc physical asset inventory counting, including counting plan creation, counting task assignment, mobile counting, variance analysis, and adjustment handling.

## Business Rules
1. Support annual full counting and monthly/quarterly cycle counting
2. Counting plan can be customized: by location, asset type, department, etc.
3. Mobile counting support: scan barcode/RFID, take photos, add remarks
4. Variance automatically identified: overage, shortage, location error
5. Variance handling workflow: investigation, approval, adjustment
6. Real-time counting progress monitoring
7. Support pause and resume counting
8. Counting report automatically generated

## Preconditions
- Counting plan created and approved
- Counting personnel assigned and trained
- Mobile device configured with counting application
- Asset labels (barcode/RFID) readable

## Postconditions
- Counting result recorded
- Variance handling workflow initiated
- Asset record updated (if needed)
- Counting report generated

## Input Data
1. Counting plan ID
2. Counting area/scope
3. Counting personnel ID
4. Counting start time
5. Actual asset location
6. Asset condition
7. Variance remarks
8. Photo evidence

## Output Data
1. Counting task assignment
2. Real-time counting progress
3. Variance list
4. Counting completion report
5. Adjustment suggestions
6. Audit tracking record

## Validation Rules
1. Counting personnel must be in authorized personnel list
2. Counting time must be within plan time range
3. Asset must belong to counting scope
4. Variance must be confirmed before submission
5. Photo evidence must be clear and identifiable

## Error Handling
1. Label cannot be read: Manually enter asset number and mark issue
2. Network connection interrupted: Offline mode support, auto-sync later
3. Counting scope error: Prompt personnel to confirm scope
4. Personnel without permission: Reject counting submission

## Priority
High
