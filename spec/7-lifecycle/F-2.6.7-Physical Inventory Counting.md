## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Physical verification requirement |
| **ISO 55001 Physical Asset Management** | Yes | Clause 8.2 - Asset tracking verification |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Physical count best practices |

**ISO 55000 Alignment Explanation:**
ISO 55001 requires organizations to track their assets and verify asset existence and condition. Physical inventory counting is essential for maintaining accurate asset records and supporting the asset management system's reliability.

**Alignment Gaps:**
- Missing requirement for automated count scheduling based on risk
- No linkage to asset criticality for count frequency determination

### 2.2 US/EU Regulatory Compliance

| Regulation | Applies | Compliance Status | Evidence/Mapping |
|------------|---------|-------------------|------------------|
| **GDPR** (EU) | Yes | Compliant | INV-001: Counter personal data minimized |
| **SOX** (US) | Yes | Gap | INV-002: Annual physical count required for financial controls |
| **OSHA** (US) | Yes | Compliant | INV-003: Safety equipment verification during counts |
| **EPA** (US) | No | N/A | Not applicable |
| **WEEE Directive** (EU) | No | N/A | Not end-of-life |
| **IFRS 16** (EU/US) | No | N/A | Not lease-related |
| **US GAAP** (US) | Yes | Compliant | INV-004: Asset existence verification |
| **GAMP 5** (Pharma) | No | N/A | Not pharmaceutical |

**Regulatory Compliance Details:**

**SOX Compliance (US):**
- Section 404 requires annual physical verification of fixed assets
- Count results must support internal control assertions
- Discrepancies must be investigated and resolved

**US GAAP (ASC 360):**
- Physical inventory counts required for asset existence assertion
- Count must cover all locations where assets reside
- Discrepancies require adjustment with appropriate approval

### 2.3 Gap Analysis

| Gap Category | Gap Description | Severity | Remediation Effort |
|--------------|-----------------|----------|-------------------|
| SOX | Annual count cycle not explicitly required | High | Small |
| SOX | Count approval workflow for adjustments | Medium | Medium |

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Counting plan created and approved | Check CountingPlan.Status = 'APPROVED' | INV-101 | "Counting plan not approved" |
| PRE-002 | Counting personnel assigned and trained | Check Personnel.Trained = TRUE | INV-102 | "Counting personnel not trained" |
| PRE-003 | Mobile device configured | Check Device.Configured = TRUE | INV-103 | "Mobile device not configured" |
| PRE-004 | Asset labels readable | Check Assets have Labels = TRUE | INV-104 | "Asset labels not available" |

### 3.3 Business Logic

**Decision Tree:**
```
START
  │
  ▼
Create Counting Plan
  │
  ▼
Assign Counting Tasks
  │
  ▼
Mobile Counting Session
  │
  ▼
Scan Asset Label
  │
  ├── Match Found ──► Record Count ──► ▼
  │                              │
  │                              ▼
  │                        Verify Location
  │                              │
  │                              ▼
  │                        Record Condition
  │
  └── Match Not Found ──► Flag for Investigation
  │
  ▼
Complete Counting
  │
  ▼
Variance Analysis
  │
  ├── No Variance ──► Close Count ──► Update Asset Records
  │
  ├── Minor Variance ──► Investigate ──► Approve Adjustment ──► Update Records
  │
  └── Major Variance ──► Full Investigation ──► Management Approval ──► Update Records
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Label damaged | Manual entry with photo | Flag for label replacement |
| EC-002 | Network failure | Offline mode with queue | Auto-sync when restored |
| EC-003 | Asset found at wrong location | Update location, flag variance | Record as found |
| EC-004 | Asset not in system | Create addition record | Investigate before adding |
| EC-005 | Count abandoned | Save partial progress | Resume later |
