# F-2.6.7 - Physical Inventory Counting

**Requirement ID:** F-2.6.7  
**Module:** Asset Lifecycle Management  
**Priority:** High

---

## Requirement Description

Support regular or ad-hoc physical asset inventory counting, including counting plan creation, counting task assignment, mobile counting, variance analysis, and adjustment handling.

---

## Business Rules

1. Counting plan must be created and approved before counting begins
2. Counting personnel must be trained and assigned to tasks
3. Asset labels (barcode/RFID) must be readable for scanning
4. Minor variance: investigate and approve adjustment
5. Major variance: full investigation with management approval required
6. Network failure during mobile counting: offline mode with auto-sync
7. Assets found at wrong location: update location and flag variance

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| PlanNumber | Yes | Unique plan identifier |
| CountingType | Yes | FULL, PARTIAL, or AREA |
| Scope | Yes | Description of assets included |
| StartDate | Yes | Planned start date |
| AssetID | Yes | Asset being counted |
| CountedDate | Yes | Date of counting (cannot be future) |
| FoundLocation | Yes | Where asset was found |
| Condition | Yes | Asset condition at count time |

---

## Output Data

| Data | Description |
|------|-------------|
| PlanID | System-generated UUID |
| TaskID | UUID for counting task |
| VarianceID | UUID if variance detected |
| AdjustmentID | UUID for approved adjustments |

---

## Validation Rules

1. Counted quantity must be non-negative
2. Count date cannot be in future
3. Counting plan must be approved before counting
4. Personnel must be trained for counting tasks
5. Major variance (>threshold%) requires approval before adjustment

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all counting operations |
| PlantManager | Create and approve counting plans |
| InventoryClerk | Perform counting tasks |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Counting plans created at plant level; adjustments require division approval
- **Isolation**: CompanyCode filtering on all count data

---

## Compliance Requirements

- SOX: Annual physical count required for financial controls; discrepancies must be investigated
- US GAAP: ASC 360 - asset existence verification
- OSHA: Safety equipment verification during counts
- ISO 55000: Physical verification supporting asset tracking requirements
