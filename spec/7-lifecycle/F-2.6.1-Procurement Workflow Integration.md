## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Aligns with asset management planning and acquisition processes |
| **ISO 55001 Physical Asset Management** | Yes | Supports asset creation and handover criteria |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Follows lifecycle-based approach |

**ISO 55000 Alignment Explanation:**
This requirement supports ISO 55000 asset management planning (Clause 6.3) by ensuring proper acquisition processes. It supports the asset lifecycle from planning through operation, ensuring assets are properly transitioned into service with complete documentation. The procurement integration ensures traceable asset creation aligned with organizational objectives.

**Alignment Gaps:**
- No explicit linkage to asset management objectives (should tie PO approval to asset policy)
- Missing documented handover criteria between procurement and operations

### 2.2 US/EU Regulatory Compliance

| Regulation | Applies | Compliance Status | Evidence/Mapping |
|------------|---------|-------------------|------------------|
| **GDPR** (EU) | No | N/A | No personal data processed in procurement integration |
| **SOX** (US) | Yes | Gap | PROC-001: PO approval chain must be documented for financial controls |
| **OSHA** (US) | No | N/A | No safety equipment procurement in scope |
| **EPA** (US) | No | N/A | No environmental compliance in procurement |
| **WEEE Directive** (EU) | No | N/A | Not applicable at procurement stage |
| **IFRS 16** (EU/US) | No | N/A | Lease procurement not in scope |
| **US GAAP** (US) | Yes | Gap | PROC-002: Capitalization threshold must align with company policy (typically $5,000) |
| **GAMP 5** (Pharma) | No | N/A | Not pharmaceutical industry |

**Regulatory Compliance Details:**

**SOX Compliance (US):**
- Section 404 requires internal controls over financial reporting
- PO approval authority limits must be enforced in system
- Segregation of duties: requestor cannot approve own requisition
- Audit trail of PO creation, modification, approval required

**US GAAP (ASC 360):**
- Assets meeting capitalization threshold ($5,000 standard) must be tracked
- Allocated costs (freight, installation) must be included in asset cost
- Partial receipts must be tracked against PO

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Procurement system interface is configured | Check InterfaceConfig.Status = 'ACTIVE' | PROC-101 | "Procurement interface not configured" |
| PRE-002 | User has "Procurement Integration" permission | Check UserPermissions contains 'PROC_INTEGRATION' | PROC-102 | "User lacks procurement integration permission" |
| PRE-003 | Asset classification system is defined | Check AssetClassifications exists and is not empty | PROC-103 | "Asset classification system not defined" |
| PRE-004 | Company code is valid | Check Company.CompanyCode is not null | PROC-104 | "Company code required for procurement integration" |

**Pre-condition Validation Matrix:**
```
IF (InterfaceConfig.Status = 'ACTIVE') 
   AND (User has 'PROC_INTEGRATION')
   AND (AssetClassifications NOT EMPTY)
   AND (Company.CompanyCode IS NOT NULL)
THEN proceed to procurement workflow
ELSE throw PRE-xxx Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Procurement data synchronized to asset system | PO data appears in AssetProcurement table | Database query verification |
| POST-002 | Asset record associated with purchase order | Asset.PurchaseOrderID is not null | Asset record lookup |
| POST-003 | Financial information automatically transferred | Asset.AcquisitionCost = PO.TotalAmount | Financial reconciliation report |

**Post-condition Verification:**
- Asset record creation verified via AssetCreatedEvent
- PO linkage verified via foreign key constraint
- Financial sync verified via accounting voucher generation

### 3.3 Business Logic

**Decision Tree:**
```
START
  │
  ▼
Receive PO from ERP
  │
  ├── PO.Status = 'APPROVED' ──► Validate PO Data ──► Create Asset Record
  │                                                     │
  │                                                     ▼
  │                                           Link Asset to PO
  │                                                     │
  │                                                     ▼
  │                                           Trigger Receiving Process
  │
  └── PO.Status ≠ 'APPROVED' ──► Log and Wait ──► END
```

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | PO to Asset Conversion | PO fully received AND PO.Status = 'APPROVED' | Create Asset record with PO details | 1 |
| BR-002 | Price as Original Value | Asset created from PO | Set Asset.AcquisitionCost = PO.UnitPrice * PO.Quantity | 1 |
| BR-003 | Multi-Supplier Tracking | Same item procured from multiple suppliers | Create supplier price history record | 2 |
| BR-004 | Contract Association | PO has associated contract | Link Asset to Contract.ContractID | 3 |

**Calculation Logic (if applicable):**
```
Asset.AcquisitionCost = PO.UnitPrice * PO.Quantity 
                       + PO.FreightCost 
                       + PO.InstallationCost 
                       + PO.OtherAllocatedCosts

Asset.CapitalizationDate = MAX(PO.ReceiptDate, Asset.CommissioningDate)
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | Partial PO receipt | Create CIP record, not full asset | Create CIP asset, link to PO line |
| EC-002 | PO cancelled after asset created | Reverse asset creation, create reversal entry | Status = 'CANCELLED', financial reversal |
| EC-003 | Price variance at receipt | Flag for review, allow override with reason | Create variance record, require approval |
| EC-004 | Supplier not in master | Reject sync, request master data entry | Return error, halt sync until resolved |
| EC-005 | PO received before asset classification set | Default to UNCLASSIFIED, require update | Create asset with temp class, enforce update |

**Edge Case Decision Matrix:**

| PO Status | Receipt Status | Asset Exists | Outcome |
|-----------|----------------|--------------|---------|
| APPROVED | FULL | NO | Create Asset |
| APPROVED | PARTIAL | NO | Create CIP |
| APPROVED | FULL | YES | Update Asset |
| PENDING | ANY | ANY | Wait for approval |
| CANCELLED | ANY | YES | Reverse Asset |
