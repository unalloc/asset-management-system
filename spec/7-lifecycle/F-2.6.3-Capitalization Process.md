## 2. Western Market Alignment Assessment

### 2.1 ISO 55000 Alignment

| Criterion | Assessment | Notes |
|-----------|------------|-------|
| **ISO 55000 Asset Management System** | Yes | Supports asset lifecycle transition from CIP to active |
| **ISO 55001 Physical Asset Management** | Yes | Clause 6.4.2 - Asset creation and handover |
| **ISO 55002 Asset Management Systems - Guidelines** | Yes | Financial documentation requirements |

**ISO 55000 Alignment Explanation:**
Capitalization is a critical handover point in the asset lifecycle, transitioning assets from "in construction" to "in service." This requirement ensures proper documentation, cost accumulation, and formal handover criteria are met before assets enter operational service - aligning with ISO 55001's asset creation requirements.

**Alignment Gaps:**
- Missing requirement for documented handover checklist
- No linkage to asset management objectives during capitalization

**Regulatory Compliance Details:**

**US GAAP (ASC 360) Compliance:**
- Asset cost includes all costs necessary to bring asset to intended use
- Capitalization must begin when asset is "ready for its intended use"
- Depreciation begins when asset is placed in service
- Threshold: typically $5,000 (configurable per company)

**IFRS 16 / IAS 16 Compliance:**
- Cost model: Carrying amount = Cost - Accumulated Depreciation
- Component depreciation for significant parts
- Depreciation methods must be disclosed

**SOX 404 Compliance:**
- Capitalization approval authority must be enforced
- All capitalized assets must have proper supporting documentation
- Segregation of duties: preparer ≠ approver

## 3. Business Rules Enhancement

### 3.1 Pre-conditions

| Pre-condition ID | Description | Validation Logic | Error Code | Error Message |
|------------------|-------------|------------------|------------|---------------|
| PRE-001 | Asset receiving completed | Check Asset.ReceivingStatus = 'COMPLETED' | CAP-101 | "Asset receiving not completed" |
| PRE-002 | CIP record exists or asset in construction status | Check CIP.Amount > 0 OR Asset.Status = 'IN_CONSTRUCTION' | CAP-102 | "No CIP record or construction in progress" |
| PRE-003 | User has "Capitalization" permission | Check UserPermissions contains 'CAPITALIZATION' | CAP-103 | "User lacks capitalization permission" |
| PRE-004 | Accounting period is open | Check Period.Status = 'OPEN' | CAP-104 | "Accounting period is closed" |
| PRE-005 | Asset meets capitalization criteria | Check Asset.ReadinessStatus = 'READY' | CAP-105 | "Asset not ready for capitalization" |

**Pre-condition Validation Matrix:**
```
IF (Asset.ReceivingStatus = 'COMPLETED')
   AND ((CIP.Amount > 0) OR (Asset.Status = 'IN_CONSTRUCTION'))
   AND (User has 'CAPITALIZATION')
   AND (Period.Status = 'OPEN')
   AND (Asset.ReadinessStatus = 'READY')
THEN proceed to capitalization
ELSE throw PRE-xxx Error
```

### 3.2 Post-conditions

| Post-condition ID | Description | Success Criteria | Verification Method |
|-------------------|-------------|-------------------|---------------------|
| POST-001 | Asset status changes to "In Use" | Asset.Status = 'IN_USE' | Asset record verification |
| POST-002 | Depreciation calculation starts | Depreciation.Schedule created | Depreciation schedule report |
| POST-003 | Accounting voucher generated | Voucher.VoucherNumber generated | Financial system query |
| POST-004 | CIP amount cleared | CIP.Amount = 0 | CIP record verification |

**Post-condition Verification:**
- Asset status change triggers depreciation workflow
- Accounting voucher links to asset master record
- CIP closure triggers asset readiness notification

**Business Rules Specification:**

| Rule ID | Rule Name | Condition | Action | Priority |
|---------|-----------|-----------|--------|----------|
| BR-001 | Full Capitalization | CIP.Amount = Asset.CapitalizationAmount | Clear CIP, set asset to IN_USE | 1 |
| BR-002 | Partial Capitalization | CIP.Amount > Asset.CapitalizationAmount | Partial clear CIP, keep residual | 2 |
| BR-003 | Threshold Check | Asset.CapitalizationAmount < Company.Threshold | Expense instead of capitalize | 1 |
| BR-004 | Depreciation Start | Asset.Status = 'IN_USE' | Create depreciation schedule | 1 |
| BR-005 | Depreciation Method Assignment | Asset.Category assigned | Set Depreciation.Method from category | 2 |
| BR-006 | Useful Life Assignment | Asset.Category assigned | Set Depreciation.UsefulLife from category | 2 |

**Calculation Logic:**
```
Asset.CapitalizationAmount = CIP.AccumulatedCost 
                           + Procurement.FreightCost 
                           + Procurement.InstallationCost 
                           + Procurement.OtherAllocatedCosts

Asset.CapitalizationDate = Asset.ReadinessDate (when "ready for intended use")

Depreciation.StartDate = Asset.CapitalizationDate

IF Asset.CapitalizationAmount >= Company.CapitalizationThreshold THEN
  Asset.IsCapitalized = TRUE
  Depreciation.MonthlyAmount = (Asset.CapitalizationAmount - Asset.SalvageValue) / (Asset.UsefulLife * 12)
ELSE
  Asset.IsCapitalized = FALSE
  Asset.IsExpensed = TRUE
```

### 3.4 Edge Cases

| Edge Case ID | Scenario | Expected Behavior | Handling Approach |
|--------------|----------|-------------------|-------------------|
| EC-001 | CIP amount exceeds asset cost | Create variance record, require investigation | Log variance, require override approval |
| EC-002 | Accounting period closed mid-capitalization | Block, require period reopening or adjustment | Return error, suggest alternative period |
| EC-003 | Asset partially ready for use | Support partial capitalization (phased usage) | Create multiple asset records or components |
| EC-004 | Depreciation rule not configured | Block capitalization until depreciation policy set | Return error, enforce policy configuration |
| EC-005 | Capitalization amount = 0 | Block capitalization, require cost entry | Return error |
| EC-006 | Retroactive capitalization date | Allow with justification and additional approval | Require VP approval for backdating |

**Edge Case Decision Matrix:**

| CIP Amount | Threshold | Ready Status | Period | Action |
|------------|-----------|--------------|--------|--------|
| > 0 | Above | Yes | Open | Capitalize |
| > 0 | Below | Yes | Open | Expense |
| > 0 | Above | No | Open | Block |
| > 0 | Above | Yes | Closed | Block |
| 0 | N/A | Any | Any | Block |
| < 0 | Any | Any | Any | Block, investigate |
