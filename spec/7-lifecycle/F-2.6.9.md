# F-2.6.9 Asset Disposal Workflow

## Function ID
F-2.6.9

## Function Name
Asset Disposal Process

## Function Description
Manage complete process of asset disposal including retirement, sale, donation, and disassembly, supporting approval workflow and compliance checking.

## Business Rules
1. Disposal methods: retirement, sale, donation, exchange, disassembly, loss
2. Disposal approval based on asset value and sensitivity tiered approval
3. Asset must complete cleanup before disposal (data clearing, hazardous material handling)
4. Sale disposal requires market value assessment, support auction or negotiated sale
5. Donation disposal requires recipient qualification assessment and tax impact
6. Disposal records must be complete, including reason, method, date, amount
7. Asset status updated after disposal, financial processing (impairment, gain)
8. Disposal supporting documents archived (disposal report, contract, receipt)

## Preconditions
- Asset meets disposal conditions (retired, obsolete, surplus)
- Disposal assessment completed
- User has "Asset Disposal" permission
- Compliance check passed

## Postconditions
- Asset status changes to "Disposed"
- Financial record updated (impairment loss or disposal gain)
- Disposal record saved
- Space/inventory released

## Input Data
1. Asset ID list
2. Disposal method
3. Disposal reason
4. Assessed value
5. Expected disposal date
6. Approver list
7. Compliance check result
8. Disposal document attachments

## Output Data
1. Disposal application
2. Approval status tracking
3. Disposal completion confirmation
4. Financial voucher number
5. Disposal supporting documents
6. Asset status update notification

## Validation Rules
1. Disposal method must be applicable to asset type
2. Assessed value must be reasonable (reference market value or residual value)
3. Approval process must comply with company policy
4. Compliance check must pass (environmental, safety, data security)
5. Disposal date cannot be earlier than current date

## Error Handling
1. Asset not disposable: Return reason (e.g., still under warranty, open work orders)
2. Approver invalid: Prompt to select valid approver
3. Compliance check failed: Show rectification requirements
4. Assessed value abnormal: Prompt for reassessment or provide justification

## Priority
High
