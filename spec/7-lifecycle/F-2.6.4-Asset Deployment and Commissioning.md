# F-2.6.4 - Asset Deployment and Commissioning

**Requirement ID:** F-2.6.4  
**Module:** Asset Lifecycle Management  
**Priority:** High

---

## Requirement Description

Manage complete process of asset installation, commissioning, acceptance, and handover including location assignment, responsible person designation, and commissioning records.

---

## Business Rules

1. Asset must be capitalized or eligible for direct commissioning before deployment
2. Multi-phase commissioning: Installation → Commissioning → Trial Operation → Formal Operation
3. Safety inspection sign-off required before equipment energization (OSHA compliance)
4. Location capacity check: deployment location must have available capacity
5. Responsible person must be assigned before commissioning
6. Commissioning personnel must have required certifications
7. All required safety permits must be obtained before Phase 2
8. Maintenance plan activated upon successful commissioning completion

---

## Input Data

| Data | Required | Description |
|------|----------|-------------|
| AssetID | Yes | UUID of asset to deploy |
| TargetLocationCode | Yes | Where asset will be installed |
| ResponsiblePersonID | Yes | UUID of responsible person |
| PlannedStartDate | Yes | Planned deployment start date (cannot be past) |
| PhaseResults | Yes | Results of each commissioning phase |

---

## Output Data

| Data | Description |
|------|-------------|
| SessionID | System-generated UUID for deployment session |
| AssetStatus | Updated to IN_USE upon completion |
| LocationCode | Final installation location |
| CommissioningReportID | UUID of commissioning report with signoffs |
| MaintenancePlanActivationID | UUID of activated maintenance plan |

---

## Validation Rules

1. Asset must be in CAPITALIZED or IN_USE status
2. Target location must have available capacity
3. Responsible person must be valid employee
4. Planned start date cannot be in the past
5. Personnel must have required certifications for commissioning
6. All safety permits must be obtained
7. All required signoffs must be collected before final acceptance

---

## Access Control

| Role | Permission |
|------|------------|
| SystemAdmin | Full access to all deployments |
| PlantManager | Initiate and approve deployments at plant level |
| MaintenanceManager | Manage commissioning phases |
| OperationsSupervisor | Complete commissioning phases |

---

## Multi-Company Data Scope

- **Data Level**: Company/Plant/Department
- **Access**: Deployment initiated at plant level; final acceptance requires plant/division manager
- **Isolation**: CompanyCode filtering on deployment records

---

## Compliance Requirements

- OSHA: Lockout/Tagout procedures verified before energization; operator training records required
- US GAAP: ASC 360 - placed in service date = commissioning completion date
- ISO 55000: Asset handover to operations with documented acceptance criteria
- ISO 55001: Commissioning checklist and criteria documentation

---
